M15 リリースノート(詳細)

システムの必要条件
System Requirements

Windows:
Windows 95/98、Windows NT 4.0、Windows 2000、Pentium 133 MHz、32 MB の RAM
あなたはシステムへ msvcirt.dll をインストールしなければなりません。
代替手段: Netscape 6 PR1をインストールしますと、そのdllがインストールされます。 その後に M15をインストールして下さい。 (Bug 27601)
Mac OS:
Mac OS 8.5/8.6/9、PowerPC 200 MHz、PowerPC 604 または G3、32 MB の RAM と仮想メモリ(または 48 MB の実メモリ)
あなたは QuickTimeLib をマシンにインストールしておかなければなりません。
Linux:
Red Hat Linux 6.1、Pentium 133 MHz、32 MB の RAM
もし SuSe Linux 6.2 で Netscape 6 を動かしているのであれば、ここから libjpeg.so.62(jpegsrc.v6b.tar.gz) をインストールしなければなりません。

インストールする前に読んでください
READ THIS Before You Install

AOL 4.0 かそれ以前のバージョン と Mozilla を同時に動かすことは 問題になるかも知れません。AOL 5.0 と Mozilla を動かす事による問題は報告されていません。

もし起動時にクラッシュしたならば、あなたの Mozilla のレジストリを削除してください。 Mozillaのレジストリーは M9から全てのプラットフォームで変わっています。

Mac OS:
システムフォルダー内のPreferencesフォルダーを開いて、 "Mozilla Registry" ファイルを削除してください。
Windows:
Windowsディレクトリーを開いて、"mozregistry.dat" を削除してください。
Linux:
~/.mozillaディレクトリーを開いて、"registry"ファイルを削除してください。

さらに詳しく知りたい方はここをご覧ください。 Win32にインストールしている場合には インストール手順にある起動時のクラッシュに関する追加情報を読んでください。

空のディレクトリーにインストールするか、あるいはインストールする以前に、前のビルドを削除してください。以前のマイルストーンや製品に上書きでインストールすると問題を引き起こすかもしれません。 ソフトウェアが開発中である間はマイルストーン間での不整合が生じるかもしれません。

glibc 2.0を用いた (RedHat 5.2 のような) Linux システムでは、thread safeではない dl関数 (dlopen(), dlsym()) の問題があります。 これは M15ビルドの起動時にランダムにクラッシュすると思われます。 もしこの問題が発生したならば、アプリケーションを何度か起動し直してください。 ftp.mozilla.orgにポストされたものは RedHat 6.0(それはglibc 2.1を使用して、この問題を持っていません)でビルドされます。より詳しい情報は Bug 8849 を参照してください。

Java Plugin 1.3は使用しないでください。 それは最近の Mozillaのビルドでは動作せず、クラッシュします。 *npjava*.* というファイルが、bin/pluginsディレクトリーや NS4.x のプラグイン・ディレクトリーに無いことを確認してください。 この問題は現在取り組まれています。 このバグに関して詳しくは Bug 5369 をご覧ください。

Proxiesはこのリリースで動作します。 設定の詳細に関しては Bug 8559 をご覧ください。

M15 リリース・ビルドをインストールする
Installing the M15 Release Build

インストールの手順 -- Unix

  1. "mozilla" という名のディレクトリーを作成し、その mozilla ディレクトリーに tar.gz ファイルを移動して下さい。

    2. mkdir mozilla
    mv mozilla*.tar.gz mozilla

  2. mozilla ディレクトリーに移動して、アーカイブファイルを展開(解凍)して下さい。すると "package"というディレクトリーが作成されます。

    3. cd mozilla
    gzip -dc mozilla-i686-pc-linux-gnu-M15.tar.gz | tar -xvf -

  3. "package" ディレクトリに移動して下さい。

    4. mv mozilla*.tar.gz ../
    cd package

  4. mozilla を起動スクリプトで起動して下さい。

    5. ./mozilla

重要: Seamonkey を Solarisで実行するには、Mozilla on Solaris FAQ を参照してください。 ここにはビルドと実行に必要な Solaris 2.6パッチのドキュメントがあります。

Unixでメールを使用する手順

既に使用している 4.xのプロファイルから移行することもできます。また、あたらしいプロファイルを作ってアカウントの設定（Account Setup）を使ってサーバー情報を設定することも出来ます。

インストールの手順 -- Win32

WinZipが必要となるでしょう。WinZipは http://www.winzip.com(http://www.download.com)から入手可能なシェアウェアです。

  1. mozilla-win32.zip のリンクを一回クリックしてください。ファイルは デフォルトのフォルダーにダウンロードされます。

    2.  
  2. mozilla-win32.zipファイルをクリックするとWinZipが起動します。

    3.  
  3. WinZipのシェアエウェアの条件の "I Agree(了解)" をクリックした後に、 WinZipのツールバーにある Extract(展開)ボタンをクリックしてください。 "bin"フォルダーにアプリケーションと関連ファイルが展開されます。

    4.  
  4. プログラムを起動するには bin フォルダーの Mozilla を選択してください。

    5.  
    • ブラウザとメールを使うには "mozilla.exe"を実行してください。

      •  
    • gecko レイアウトエンジンだけを動かすには "viewer.exe" を実行してください。

注釈:

移行していない 4.xのプロファイルも 5.0のプロファイルも既にあり、そして mozilla コマンドを実行しても Seamonkeyが起動しない場合は、mozilla -installerコマンドを 実行してプロファイル・マネージャーを立ち上げ、プロファイルを選択してください。

Win32 でメールを使用する手順

既に使用している 4.xのプロファイルから移行することもできます。また、あたらしいプロファイルを作ってアカウントの設定（Account Setup）を使ってサーバー情報を設定することも出来ます。

インストールする手順 -- Mac OS

もし StuffIt ExpanderTM がインストールされており、使用しているブラウザーがそれを使うように設定されている場合:

  1. mozilla-mac.sea.binのリンクを一回クリックしてください。ファイルはデフォルトのフォルダー（通常はデスクトップ）にダウンロードされ、StuffItTM が起動されて mozilla-mac.sea に変換されます。 それから StuffItTM は mozilla-mac.sea を開いて mozilla-macフォルダーに変換してくれます。

    2.  
  2. mozilla-mac フォルダーを開いて "mozilla." ファイルをダブルクリックしてください。 (注意: この段階で .bin と .sea ファイルは不要になります。）

注意: もし手順の一つ目でブラウザーから何をするか問い合わせられた場合は、StuffItTMの設定がされていないのが原因です。 その場合には保存(Save)をクリックして下記の 手順 3番 に進んでください。

もしも StuffIt ExpanderTM がインストールされていないか、あるいはブラウザーが適切に設定されていない場合:

  1. StuffItTMがハードディスクのどこかにあることを確認してください。

    2.  
  2. mozilla-mac.sea.bin のリンクをクリックしてください。

    3.  
  3. "Save this link as...(リンクを名前をつけて保存)" を選択し...

    4. ファイルダイアログが現われて、ファイルをどこに保存するか選択することができます。(例えばデスクトップ)

    注意: Macのインストーラーのアクセスパスにはスラッシュ (/) を使わないでください。 スペースは使用しても構いませんが、その他の特別な文字は避けてください

  4. Save(保存)ボタンをクリックしてください。

    5.  
  5. 新しく作られたアイコン "mozilla-mac.sea.bin" をダブルクリックしてください。

    6.  
  6. StuffItTM が起動し、同時に "mozilla-mac.sea" ファイルと "mozilla-mac." という名前のフォルダが作成されます。

    7.  
  7. mozilla-mac フォルダーを開いて "mozilla." ファイルをダブルクリックしてください。 (注意: この段階で .bin と .sea ファイルは不要になります。）

Macintosh でメールを使用する手順

既に使用している 4.xのプロファイルから移行することもできます。また、あたらしいプロファイルを作ってアカウントの設定（Account Setup）を使ってサーバー情報を設定することも出来ます。

一般的な問題
General Issues

（XP Toolkit 構成部品を操作するHTML なども含み）インターフェイスを構成する部品の中には、マウスを押してコンテキストメニューを出す（右手用マウスでの右クリック）ときに、まともに応答しないものもあります。 (Bug 30497)

M15 の Mac OS 版には、検索機能がありません。 Bug 34490 と Bug 35032 が検索を妨げています。

Windowsについて　Windows

動作中のウィンドウ、すなわち何らかのプラットフォームにおいて利用者入力をまとめる部分は、何かの拍子に、他のウィンドウの下になることがあります。最上位のウィンドウがキーボードやマウスに反応しない状態では、下のほうに動作中のウィンドウが隠されていることがあり得ます。このときは、動作中のウィンドウを手前に持ってきてやれば解決します。 (Bug 28459)

よく知られている状況としては、Web ページを読み込んだときに、 GFX スクロールバーがちゃんと再描画されず、そのスクロールバーがあった位置に影が残るというものがあります。 (Bug 34218)

ブラウザウィンドウの上部にあるパーソナルツールバーを折りたたむと、ツールバーはまったく見えなくなり、ツールバーを引っ張り上げようにも手がかりがなくなります。これを回復するには、新しいブラウザウィンドウを開くか、ブラウザを再起動します。 (Bug 35181)

(Linux 版のみ)プログレス・メータのアニメーション（床屋さんのクルクル回転する看板のようなアニメーション）はときどき、その領域からはみ出して描かれます。特に、ページ読み込み中にブラウザウィンドウを拡大・縮小すると、これがよく起きるようです。 (Bug 32331)

(Linux 版のみ)スクロールバーの操作が効かないことがあります。対策としては、新しいブラウザウィンドウを開きます。新しいウィンドウのほうのスクロールバーは動作するはずです。 (Bug 26658)

(Linux 版のみ)タスクメニューのウィンドウ一覧を押したり、コンポーネントアイコンを押したり、タスクバーの "Go To Window" アイコンを押したりしてウィンドウを手前に持ってくることができません。ウィンドウを手前に持ってくるには、そのウィンドウ自体をクリックしてください。 (Bugs 8002, 28626, と 26082)

(Windows 版のみ) Ctrl-タブ キーを押しても、開いているウィンドウは切り替えられません。タスクメニューを開いて、行先のウィンドウを選んでください。 (Bug 30864)

キーボード・ショートカット　Keyboard Shortcuts

左および右方向のカーソルキーは、横方向のスクロールをしません。代わりに横スクロールバーを使ってください。 (Bug 7404)

キーボード・ショートカットの実装は、完全ではありません。切り取り・コピーなどの編集用ショートカットには、特定の対話ウィンドウでは動かないものがあります。ナビゲーション用のキーボード・ショートカットは全然動きません。 メニューアイテム用のキーボード・ショートカットにも、いくつか動かないものがあります。 (Bugs 31233, 2674, 22529, 29348, と 20298)

(Linux 版のみ) メニューアクセスキーは今のところ、できません。 (Bugs 22515 と 24413)

初期設定　Preferences

上下方向のカーソルキーを押しっぱなしにして設定パネルを巡回していると、クラッシュの恐れがあります。キーを押しっぱなしにしないで、もうちょっとゆっくり操作してください。 (Bug 29882)

設定画面(Preferences window)から抜けると、もう mozilla は active application ではなくなっています。 mozillaの画面を押して、mozillaを手前に持ってきてください。 (Bug 7862)

フォント設定ダイアログボックス(Font Preferences dialog box): language font を選択すると、クラッシュします。セリフ・サンセリフ・モノスペースフォントを選ぶポップアップウィンドウは、空のままです。12ポイントより小さな活字を選べません。

プログラム設定画面(Applications Preferences dialog)は、まだありません。 (Bug 10958)

メニュー　Menus

context menusの「全て選択」が動きません。代わりに「編集」メニューの「全て選択」を使ってください。 (Bug 28508)

(Linux 及び Windows 版のみ) context menuから「コピー」を選ぶと、その menu が画面に残ってしまいます。これを消すには、ウィンドウの外を押してください。 (Bug 28773)

マウスホイール・スクロール　Mousewheel scrolling

Unix:
あらかじめ X サーバを設定しておかないといけません。 XFree86だと、XF86ConfigファイルのPointer項目に、"ZAxisMapping 4 5" というのを1行追加します。同じく Protocol行も、使用中のマウスに合わせてあることを確かめてください。
Windows:
Logitech の wheel mouseを使っていて、Logitech の em_exec プログラムが走っているなら、ホイールのスクロールは Mozillaには効かないことがあります。この問題の対策は（マウスドライバの更新に影響され得るので） Bug 20618 に記してあります。
Mac OS:
Mac OS版には、まだ(USB 用)マウスホイールスクロールが実装されていません。

プロフィール
Profiles

重要: 全てのプロフィールに関する情報はレジストリファイルに蓄えられ、レジストリファイルは諸君が Mozilla の初期のバージョン(Apprunner)を使っていた際も作られている。新しいビルドをチェックするときには、必ず削除かリネームを、次の項目についてインストール、ビルド、及び新しい実行ファイルを行う前にしなさい:

Windows
mozregistry.dat ( c:¥winnt か c:¥windows、あるいは相当する場所)
Mac OS
Mozilla Registry (System Folder:Preferences)
Linux
registry (~/.mozilla/)

プロフィールを蓄えるためのフォルダの選択がどのプラットフォームでも起動しない -- デフォルトフォルダが常に使われる。

Linux と Mac OS:
カスタムフォルダを選ぶことが出来るが、プルフィールは未だデフォルトのフォルダの中に蓄えられる (Bug 33752)。
Windows:
カスタムフォルダボタンを押した時に、何も起きず、諸君はそのプロフィールをデフォルトのディレクトリに保存しなければならない (Bug 34071)。

Windowsではデフォルトのプロフィールのためのプロフィールディレクトリは exe の場所から <3 レベル上のディレクトリ>¥Users50 であり MacOSでは :Documents:Mozilla:Users50: であり、そしてLinuxでは ~/.mozilla である。

プロフィールの移行の確認ダイアログが長く狭い (Bug 34200)。

スペースが入っている新しいプロフィールファイルを作るときには、その名前をクオーテーションマークで閉じなければならない (例えば, "My Profile")。

プロフィールマネージャはコマンドラインの引数で起動できる : -ProfileManager。そのプロフィールマネージャのユーザインターフェースが現れ、それは新しいプロフィールの作成と、存在するプロフィールの削除とリネームをサポートしている。どの作業を終えるときも、プロフィールマネージャを終わるためには Exit ボタンをクリックしなさい。諸君は Mozilla を選択されたプロフィールで Start Communicator ボタンをクリックすることで Mozilla をスタートさせることが出来る。また、ユーザは 4.5の Communicatorプロフィールを移行することができる。

プロフィールを、作成、管理するための様々なコマンド:

# mozilla -P <Profilename>
Mozillaを <Profilename> で起動する。
# mozilla -CreateProfile "<Profilename> <ProfileDir>"
プロフィールを作成し (つまり, <Profilename>) 特定の場所にディレクトリを作成する。つまり, <Profiledir> そして Mozilla を起動する。
# mozilla -CreateProfile <Profilename>
プロフィールとプロフィールディレクトリをデフォルトのプロフィールの場所に作る。
# mozilla -ProfileWizard
プロフィール作成ウィザードを起動し新しいプロフィールを作る。プロフィール作成ウィザードは新しいプロフィールをデフォルトの場所に作るか、自分で選択した場所に作らせる。プロフィールの処理の最後には、作成されたプロフィールが現在のものとしてマークされ、Mozilla がそのプロフィールで起動する。このウィザードは現在は二つのスクリーンである。 Mail と News 関連の設定/情報は、別のアプリケーションで集められる。
# mozilla -ProfileManager
プロフィールマネージャを起動する。ユーザはプロフィールを選び、Start Communicaor ボタンをクリックして選ばれたプロフィールで Mozilla を起動する事が出来る。現在のプロフィール(普通は最後に使われたプロフィールである)があらかじめ選択されて現れる。
# mozilla -SelectProfile
プロフィールセレクタを起動する。ユーザはプロフィールをそれら表示された者の化から選ぶか、"manage profiles" をクリックして、プロフィールマネージャに到達し、上に述べたような処理を行うことが出来る。
# mozilla
複数のシナリオがある:
 プロフィールの数 = 0 --> プロフィールウィザードを起動する。
 プロフィールの数 = 1 --> そのプロフィールでのディレクトリでMozillaを起動する。
 プロフィールの数 > 1 --> プロフィールマネージャを起動する

4.5のプロフィールを移行するためには、Mozilla をコマンドラインオプション -installer で起動しなさい。 そのプロフォールがプロフィールマネージャのリストボックスの中にアイコンを伴って選択される。ユーザは移行されたプロフィールを選択肢、Start Mozilla button をクリックすることが出来る。もし移行がうまく行われていたなら、プロフィールはもはや表示の際にアイコンを伴って表示される、5.0のプロフィールとして扱われる。現在は、 prefs.js ファイルとメール・ニュースフォルダーは移行の過程でコピーされる。ユーザは移行されたプロフィールに関して、再び名前を付て Communicator を起動することは出来ない。

Java

M15 は Java 2 Standard Edition (J2SE) Runtime Environment v1.3 のプレリリースバージョンを含んでいる。M15 をインストールする事はそのマシンに存在する J2SE を無効にするか不安定にする。

M15 はバンドルされた特別な J2SE のプレリリースバージョンでのみ動作する。ファイナルリリースの J2SE v1.3, や他の過去のバージョンの J2SE は動作しない

HTTPS はこのリリースで J2SE をサポートしていない

基本的な LiveConnect だけが M15 でサポートされている。

256色で M15 の J2SE を走らせると、ちらつきの問題の原因となる。

アプレットが現在のページのブラウザの中で動いていると、ブラウザウインドウを閉じることは例外を投げる原因となる。なぜならブラウザはアプレットを適切に停止しないからだ。この問題はブラウザが閉じられたときに現在のページでもしアプレットが見えなければ起こらない。 (Bug 12153)

同時にアプレットとプラグインを同じページで多く走らせ過ぎると、ブラウザが起動しているアプレットやプラグインのインスタンスを停止させるか終了させようとする。しかしアプレットやプラグインは適切にクリーンアップされない。

(Mac OS) M15 でアプレットを走らせるためには、Mac OS Runtime for Java (MRJ) version 2.2 を持っている必要がある。もし Mac OS version 8.5 から 8.6 へのアップグレードであれば、MRJ もアップグレードが必要である。アップデートのためには Apple ソフトウェアダウンロードページに行きなさい。 (Bug 13848)

alignパラメータを持ったアプレットはブラウザのクラッシュの原因となる。 (Bug 8290)

もし Sun Microsystems の最新の JRE 1.2 をインストールしているなら、Java プラグインが存在しているときにスタートアップの際にクラッシュする。これを修正するには、次の dlls をプラグインフォルダから取り除きなさい。それは 4.5 (あるいはそれ以降のバージョン) の Communicator directory にある: npjava32.dll, npjava12.dll と npjava11.dll.

Linux では、Java はまだ実装されていない。

Linux の小さなフォント
Small Fonts on Linux

もし自分の環境でフォントが小さすぎたら、Mozilla でこの設定を利用可能にすることが出来るかもしれない。 設定ファイルに次の行を加えなさい。 (~/.mozilla/Personal/prefs.js):


 user_pref("browser.screen_resolution", 120);

これは dpi を120にセットする。もし接待が zero (0) であると、Mozilla は X server の dpi の値を使う。もし値がセットされていなければ、デフォルトの 96 の値を使う。

X server の DPI の値をセットするための情報は http://www.mozilla.org/unix/dpi.htmlにある。

もし DPI を正しくセットした後もフォントがまだ小さすぎる(あるいは大きすぎる)ならフォントサイズを次のようにして修正することが出来る:

setenv GECKO_FONT_SIZE_FACTOR 1.5

後の方のもののみがフォントに影響を与える。スクリーンの解像度の設定が 全ての絶対サイズの特定に影響を与える (例えば. CSS "pt" のポイント数).
