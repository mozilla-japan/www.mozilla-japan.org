Mozilla で一番おもしろい特徴 トップテンは何ですか？

  • パイプライン化 - ダイヤルアップ、ブロードバンド接続のどちらでも、ブラウザのパフォーマンスを改善します
  • タブブラウジング - ウィンドウを切り替える必要なしに、複数のページを切り替えることができます
  • ポップアップ ブロッキング - これ以上言う必要がありますか？
  • Bookmark キーワード - ネットサーフィンや検索を簡単にします
  • 利用者のカスタマイズ可能性 - Mozilla の設定を自分が満足できるように微調整できます
  • ページ表示 （ズーム と代替スタイルシート） - 悪い Web デザインの犠牲者となることを止めます
  • テーマと外観 - Mozilla の外観と印象をカスタマイズします
  • チャット - 組み込みのチャットクライアントで IRC が始められます
  • 拡張性 - アドオンは Mozilla は基本的なブラウザよりずっと優れているいうことを意味します
  • 多様なプラットフォームのサポート - 異なる箱の上に同じトカゲ （Mozilla は現在、日々の基礎として 12 の異なるプラットフォーム向けにリリースされており、さらに多くのプラットフォーム向けにコンパイルできます）

 

Mozilla 1.0 はどんな言語をサポートしていますか？

Mozilla 1.0 は多言語文字データ （Unicode） について確かなサポートをしており、38 の言語で利用可能なはずです。 最新の情報については、http://mozilla.org/projects/l10n/mlp_status.html をご覧ください。

Mozilla コードの本質であるオープンソースのおかげで、だれでも Mozilla アプリケーションを他の言語に自由に翻訳できます。 これは、人々が自分たちが選択した言語や方言でブラウズできるようにし、地域の関心のあるコンテンツに集中できることから、Mozilla Organization はこれを支援しています。

地域化したアプリケーション・コンテンツへの渇望は、地域化した Mozilla アプリケーションを開発している、非常に多くの自主的なコミュニティのメンバーに見るとができます。 Mozilla.org は Mozilla 地域化プロジェクト を通じてそれらの努力をサポートしています。 それは多くの地域化プロジェクトに対して、指導やコミュニティ、連絡情報を提供しています。

 

エンドユーザ Mozilla 製品と Netscape 6/7 との違いは何ですか？

Mozilla プロジェクトのコードが Netscape 6 やそれ以降の製品ラインの主要部分を形成しています。

Mozilla はオープンソース・プロジェクトであるため、だれでも 自由に Mozilla のコードを利用したり、新製品の制作のために他のコードと結合することができます。 Netscape は Mozilla プロジェクトに対し、従業員の仲間の作業結果を寄付しています。 その作業結果は、Mozilla コードベースの作成のために他の多くの貢献者の作業結果と結合されます。

そのため Netscape は Mozilla コードベースの大部分 - でも全部ではありません - を受け取り、それを Netscape 6/7 製品の制作に利用しています。 Netscape は Mozilla の主要部分に、AOL インスタントメッセンジャー、Java バーチャルマシン、プラグインといった、機能のセットを追加しています。 Netscape はまた、そのロゴやその他のブランドアイテムを追加し、デフォルト URL を設定したりしています。

Netscape 6/7 は消費者を想定した Mozilla コードベースのカスタマイズ製品ですが、それに対して Mozilla そのものは開発コミュニティをターゲットにしています。

 

Mozilla は他のブラウザとどう比較できますか？

Mozilla は幅広いプラットフォームを越えた最高の整合性を提供します。 これは、IT 部門が複合プラットフォームをサポートする必要があるときや、Windows ユーザーが Linux を試してみたいと思うときに意味のあるものとなります。

詳細な比較表については、http://www.vorstrasse91.com/moztips/browser_compare.html へどうぞ。 最近の Mac のみの比較 （新しい Gecko ベースの Mac OS X 専用ブラウザ Chimera は入っていません） もできました。

 

Mozilla 1.0 はどんなインターネット・スタンダードをサポートしているのですか？

確立した開発途上のスタンダードのサポートが Mozilla の主な設計目標の 1 つです。 それらのスタンダードは、World Wide Web Consortium (W3C)、ECMAScript として JavaScript の規格化を引き受けている ECMA、IETF などといった、信頼のある規格団体によって制定されています。

Mozilla 1.0 がサポートしているスタンダードは次のものが含まれます :

  • HTML 4.01 と XHTML 1.0/1.1
  • CSS1、CSS2 と CSS3 の一部
  • DOM1、DOM2 と DOM3 の一部 （baseURI、ロード、いくつかのネームスペース処理方法）
  • MathML
  • SVG
  • P3P
  • XML 1.0
  • XSLT
  • XML におけるネームスペース
  • XML Base （現在はリンクがあるだけです。:visited などの CSS プロパティは利用できません）
  • XLink （シンプルな XLinks のみ）
  • XML Documents と Associating Style Sheets
  • XPath 1.0
  • FIXptr
  • RDF
  • SOAP

さらに、その他のベンダーが出している、いくつかの特別な スタンダードと機能もサポートしています。 例 : XML-RPC、XMLHTTPRequest、document.load()

これらの技術についてもっと知りたいときは、Mozilla における XML を参照してください。

 

Mozilla はどんな種類のデバイスで利用可能ですか？

Mozilla の心臓部分にある、スタンダードに準拠した Gecko レンダリングエンジンは高速で有能です。 ハンドヘルドコンピュータのように小さなデバイスでも実行でき、完全なオペレーティング環境の基礎として役立つこともできます ・・・ そのような、すべてのものです。
