組み込み概要

コメント／提案（フィードバックなどは英語で送ってください）

誰がこの文書を読むべきでしょうか？

ウェブ・ブラウザとして Mozilla / Gecko を組み込むことに興味があるのでしたらこの文書を読むべきです。

必要な前提知識：

Mozilla コンポーネントを組み込むために（Gecko、"ウェブブラウザ" として組み込みたいならば）、XPCOM 、C++、コマンドラインベースのビルド方法（例えば gmake や nmake など）についての実用的な知識が必要となります。具体的な内容として、Mozilla コンポーネント（関数呼出しの詳細は nsComponentManagerUtils.h を参照してください）を作成するためには Mozilla のコンポーネントマネージャをよく知っている必要があります。基本の組み込みコンポーネントを使用しているサンプルアプリケーションを用意しました。Mozilla のコンポーネントとモジュールアーキテクチャの概要についてはこの文書をチェックしてください。

序文：

Mozilla は各種のドキュメントを表示するために他のアプリケーションウィンドウの中へ組み込むことができます。組み込みの仕方にはいろいろなレベルがありますが、この文書は Gecko（Mozilla コンテンツ表示コンポーネントの集合）の組み込みに焦点をあてています。Geckoが現在私たちのエネルギーの大半を集中させている対象ですから．．．。

私たちは現在 Gecko の組み込みに使用するインターフェイスとライブラリの公式なセットを提供しようとしています。このセットはまだ完全には確定していませんし、"正式" とされてもいませんので、現時点では自己責任でご利用下さい。

あなたのアプリケーションにドキュメントを表示するためのウインドウ、または "領域" を備える場合、いろいろなケースが考えられます。Gecko はテキストファイル、html、xml から画像にいたるまで各種のドキュメントを表示する機能を提供します。あなたはローカルディスクからヘルプドキュメントを表示したいのかもしれませんし、あなた独自のウェブブラウザを構築したいのかもしれません。Gecko はそのような機能を提供します。

Gecko とは何でしょう?

Gecko とは、ウインドウへドキュメントの内容を完全にレンダリングするために使われる Mozilla コンポーネントの集合を表す名前です。具体的には Gecko は Mozilla のネットワーキング、ジャバスクリプト、レイアウト、XPCOM、NSPRと構文解析コンポーネントで構成されます。私たちは、あなたがウインドウへドキュメントを表示することができるように Geckoを "組み込む" ための各種のインターフェイスを構築しているところです。

Gecko の組み込み:

Gecko を組み込むにはいろいろな方法があります。あなたが Linux 上でGTKを使っているならば、mozilla が提供する widget を組み込んでいる GTK を使うことができます。あなたが Windows 上で ActiveX を使っているならば、mozilla が提供する ActiveX コントロールを使うことができます。Gecko を組み込んでいる多くの例はここにあります。予めパッケージされた "widgets" のうちの１つがあなたの要求に合わないならば、あなたは直接 Gecko を組み込むことができます。その場合、あなたは若干のコードを記述して、これらの XXX ヘッダファイルのインクルードし、これらの XXX ライブラリーをリンクする必要があります。

Gecko の最も一般的な組み込みのシナリオは、あなたのアプリケーションにブラウジング機能を提供することでしょう。；あなたは、他の人のように、自分自身のウェブブラウザを作成しているのかも知れません。

あなたは、どんな URL を読み込みたいかを Gecko に教えることができます（URL を使って）。あるいは１つ前か１つ先のページを読み込みたいかを Gecko に教えることができます。またはドキュメントの読み込みを中止することができます。あなたは、Gecko にバインドしたウィンドウの大きさを変え、Gecko のコンテンツエリアの大きさも連動して変えさせることが出来ます。あなたは、視覚的な警告または通知を提示する、あるいは提示しないようにすることができます。あなたは、ディスクにドキュメントを保存したり、プリンターへドキュメントを印刷することができます。

組み込みにかかわる最も重要な二つのコンポーネントを考えてみましょう。Gecko と、組み込まれるアプリケーションです（ここでは、組み込まれるアプリケーションの例として、Dojo という名前のウェブブラウザとしましょう）。
通常、Dojo は Gecko を制御しています。Dojo は、Gecko ウェブブラウザコンポーネントをそのウインドウのコンテンツエリアにバインドします。Dojo は、どんな URL を読み込みたいかを Gecko に教えることができます（URL を使って）。あるいは１つ先か１つ前のページへナビゲートさせたり、Dojo のコンテンツエリア（それは現に Gecko のこと）からある物をクリップボードへコピーすることができます。上記の相互作用は、Dojo が Gecko を制御している場合について述べています。しかし、もっと豊富な機能を加えるには、Gecko も同様に Dojo を制御することが出来なければなりません。

Dojo が待ち受け状態であるなら、Gecko はページ読み込みステータス情報（例えばダウンロードバイト数）を提供したり、警告やコールバック通知をすることが出来ます（これらは通例として（"DNSが見つかりません"のような）ダイアログを表示させますが、Dojo は常に完全な制御下にあります）。Gecko はよく知られるポップアップ広告を表示するために JavaScript で window.open() （訳者注：原文は window.Open() とありましたので修正しました）を呼びたいということを Dojo に教えることができます。

上記の相互作用は、Dojo と Gecko が行うことができる役目を述べています。組み込むことを検討しているのでしたら、Dojo と Gecko との "境界" について説明することは助けになるでしょう。この境界は Dojo 自体のウィンドウ矩形と Gecko（の占有領域）との文字通りの境界と見ることも出来ますし、Dojo と Gecko の論理的境界と見なすことも出来ます。関数呼び出しを行うのに（あるいはその逆に Gecko が Dojo の関数呼び出しを行うのも同様に）使用される言語は C++ です。Mozillaは、XPCOMと呼ばれているオブジェクトモデルを提供します（それは COM と非常に似ており、クロスプラットフォームでちゃんと使用できるようにほんの幾つかの追加処理が為されています）。その結びつけによって、Gecko は XPCOM コールによって駆動されます、そして Gecko は Dojo を駆動するために XPCOM を使用します。 XPCOM は二つをくっつける為の糊と考えることが出来ます。これはつまり、少なくとも Dojo は XPCOM コールを Gecko のインターフェイスとすることができるということです。そして更に、Dojo は XPCOM インターフェイスを（ベースレベルのようなクラス）実装することもできるということです。

以下の説明の図は、ビジュアルに XPCOM が組み込みアプリケーションをGecko（その逆も同様）へ結び付ける際に行う役割について示します。

技術的には、Gecko を組み込む為に XPCOM について知っておく必要はありませんが（XPCOMがバイナリ互換で設計されていることを考慮に入れてください）、より上手く処理できるようにする為に XPCOM のライブラリ、ヘッダやインターフェイスを活用しようとするのはいいことです。

スレッド：

Gecko は、メインのスレッドでその作業のほぼ全てを行います。組み込まれた状態での "メイン" スレッドは Gecko コンポーネント（例えば nsIWebBrowser）を生成するスレッド／プロセスです。Gecko はそれが所有するワーカースレッドを生成します（例えば、ソケット I/O とファイル I/O のスレッド）。Geckoは "メイン" のスレッド以外からは組み込みアプリケーションへ呼び出しを行いません。あなたの計画において、それ以外のスレッドに対してプッシュ或いは何らかの応答を返す Gecko が必要となるのであれば、呼び出しや引数の管理をする責任があるのは [あなた] です。XPCOM は COM/DCOM が提供するような正式なスレッド分離モデルは提供しません。

詳細：:
組み込みアプリケーションが使用する "ルート／メイン" の Gecko コンポーネントは、nsIWebBrowser です。nsIWebBrowserインタフェースとウィンドウの間には１対１の対応が存在します。あなたが作成するウインドウについては nsIWebBrowser を使って作成する必要があるでしょう。あなたは Mozilla コンポーネントマネージャ XXX を使用することで nsIWebBrowser を作成します。

1. nsIWebNavigation->LoadURI(...) を使うと、Gecko コンテンツエリアに URL の読み込みを開始することができます。
2. nsIWebNavigation->GoBack() を使うと、１つ前のドキュメントを読み込むことができます。.
3. nsIWebNavigation->GoForward() を使うと、１つ先のドキュメントを読み込むことができます。.
4. nsIWebNavigation->Stop() を使うと、ドキュメントの読み込みを停止することができます。
5. nsIBaseWindow->SetSize(...) を使うと、Gecko のサイズを変更することができ、サイズ変更に合わせてそのコンテンツエリアを Reflow させることが出来ます。
6. あなたのウィンドウ内に Gecko コンテンツエリア（Mozilla "ウェブブラウザ"）を "組み込む" ために、あなたはウィンドウを作成して、そのコンテンツエリアがウィンドウに含まれることを nsIWebBrowser コンポーネントにより Gecko へ伝える必要があります。あなたは自分自身に Gecko を "バインド" するために nsIWebBrowser：SetContainerWindow(...) メソッドを使用します。この方法は、このように "バインド" をすることによって Gecko は（nsIWebBrowserChrom として）自身が含んでいるウィンドウを知ることができます。
（訳者注：上記の説明文の元の原文の内容で、GoBack() と GoForward() 、Stop() の部分の説明が入れ替わっていましたので訳文では修正しました。）
