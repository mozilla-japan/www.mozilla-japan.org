何かする必要がありますか?

あなたはすでにビルトをダウンロードしました。あなたがしなければならないことは、 ブラウザーとメール/ニュースリーダとしてそれを日常的に使うことです。

バグを投稿する
投稿するツール
SeaMonkey には新しく「問題のあるWebサイトを報告するツール」が備えられています。 (カスタムインストールオプション で選択してインストールできる). 「ヘルプ」 メニューから、 「問題のあるWebサイトを報告する」を選択して、あなたが遭遇している問題について 報告ができるので情報を提供してください。問題のあるサイトを私たちに報告していただくことによって、 我々は問題を修正するためにウェブマスターと働きかけでき、ウェブサイトを互換性を持つようにできます。
Talkback
Talkback ウィンドウが表示されたら、必要事項を書いてレポートを送信してください。 Talkback のレポートは私たちに、どんなクラッシュが最も深刻で、どのくらいの頻度でそれが発生するのかについて 様々なデータを提供してくれます。あなたはそれらを OK をクリックするだけで私たちに提供出来ます。
Bugzilla
私たちのアプリケーションのバグは Bugzilla に報告されています。 あなたがバグに遭遇したらまず ここをチェック して同じような問題が報告されていないか確認してください。 もしなければ、バグの投稿方法 を元にして投稿してください。
QA（品質管理）

SeaMonkey QA の ページ もあります。 QA の方法、開発の協力のしかたなどが書かれています。多少の HTML の知識が必要ですが、コードの書き方を知っている必要はありませんし、SeaMonkey をより知りたいと 望んでいる人々にとって良いページです。 手助けしていただけるなら BugAThon をまずご覧ください。

ドキュメントの筆者募集

わたしたちは ユーザ、Web開発者、SeaMonkeyと他のMozillaプロジェクトに取り組んでいる開発者のために ドキュメント を必要としています。まだまだドキュメントは不足しています。関連するドキュメントができましたら 報告してください。 あなたが発見したことをぜひ文章にしてください。

問題があるWebサイトを報告してください

あなたの好きなウェブサイトは SeaMonkey や他の Mozilla ベースのブラウザで適切に表示しますか? SeaMonkey やそのほかのブラウザで閲覧できるような規格に準拠したウェブサイトをウェブ開発者に 働きかけることにより、Mozilla を助けることができます。

また、 Mozilla Tech Evangelism プロジェクトというものもあります。あなたの好きなブラウザが サイトを正しく表示できないときは ぜひ報告してください。それらの働きかけも Mozilla のためになります。

コミュニティー

SeaMonkey 開発者は、IRC に滞在しています。あなたが Chatzilla をインストールしたなら、 簡単に コンタクトできるでしょう。 もしくは他の IRC クライアントで irc.mozilla.org:6667 に接続してください, #mozillazine ( MozillaZine が提供しているフレンドリーなチャンネルなどがあります。)
　 開発者向けの #seamonkey チャンネルもありますが、このチャンネルは開発者向けです。したがって、そこで質問をしないでください。ここに参加するにはまず ニュースグループ を読んでください。
