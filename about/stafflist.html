mozilla.org スタッフメンバー

Mitchell Baker - 「トカゲ」世話役主任 (mitchell@mozilla.org)

Mitchell は mozilla.org の全般的な問題解決担当、代弁者、政策仲裁人です。彼女は Mozilla を利用している企業やプロジェクトとともに広く仕事をしています。これは、mozilla.org の経過説明や、貢献者のニーズの聴取、市販ソフトウェア開発の世界とオープンソース開発技術の統合の模索を含んでいます。

Mitchell はまた、mozilla.org がどうプロジェクトを管理していくべきかというコンセンサスを進めるために、多くの時間を費やします。これは彼女がマゾヒスティックな一面を持っているのかもしれないということを暗示しています。彼女は Mozilla プロジェクトをより分かりやすくすることを夢見ています。

mozilla.org スタッフに参加する前、Mitchell は Netscape で製品開発と知的財産保護に関係するすべての法律上の問題について責任のある弁護士をしていました。その時期に彼女は Netscape / Mozilla Public Licenses を書きました。

Chris Blizzard (blizzard@mozilla.org)

Chris は Mozilla の様々な部分をハックしています。Mozilla のストレート Xlib ポートは、大部分が彼の過失です。彼はまた、本当に手助けが必要で、人々が本当に親切に頼んだときに、gtk ポートをハックしています。彼は Web DAV サポートやその他の楽しいネットワーク機能の追加のようなことを夢見ています。彼はまた、Mozilla プロジェクトが取ることのできる方向や、私たちが何を成し遂げられるかについて夢見ています。

Chris は Linux とオープンソフトをずっと前から使っています。ユーザとして出発し、独学でプログラミングを習得し、今では時間があるときは様々なプロジェクトをハックしています。彼はソースが公開されて以来、Mozilla のコードで仕事をしています。これまで彼は、システム管理者、Web ジョッキー、データベースプログラマー、プロジェクトマネージャとしての役割を果たしてきました。

Scott Collins (scc@mozilla.org)

Scott は、Mozilla プロジェクトの中心的技術者であり、テクニカルエバンジェリスト、連絡係です。あなたはすでに XPCOM、nsCOMPtr、Mozilla の ストリング に関する彼の仕事、あるいは、彼のテクニカルレクチャー (例えば、C++ 移植性 [スライド、ビデオ] Mozilla のストリング [スライド、ビデオ] XPCOM [ビデオ] その他、大学での講義、カンファレンス、全米のユーザグループ会議の場で) を通じて、彼のことをよく知っているかもしれません。Mozilla をハックしたいけれど、どこから始めればいいか分からない、どうやって参加すればいいか分からない、あるいは、Mozilla の技術的な詳細について、自分たちのグループに説明してくれる人を探している、そんなときは Scott が助けてくれます。

Asa Dotzler (asa@mozilla.org)

Asa は私たちのコミュニティの品質を異常なまでに擁護しています。彼は、私たちの品質プログラムにターボ加速をかけるため、そして私たちのボランティアの QA とテストコミュニティをさらに発達させるために mozilla.org に参加しました。Mozilla をテストしてバグを報告する新しい貢献者たちを手伝っていないときは、Mozilla マイルストーンのための要求を明確にし、それをリリースするために drivers@mozilla.org とともに働いています。

Brendan Eich (brendan@mozilla.org)

Brendan は Mozilla のアーキテクチャと技術的な方向性に責任があります。彼は、モジュールオーナーの委任、ソースベースのアーキテクチャ的な問題の所有、Mozilla プロジェクトの方向を概説した「 ロードマップ 」の記述を引き受けてきました。

Brendan は JavaScript を作り出し、Navigator 4.0 を通して仕事をし、国際的な標準化へそれを押し進めました。Netscape の前、彼は SGI でオペレーティングシステムやネットワークコードを書いていました。MicroUnity では、マイクロカーネルと DSP コードを書き、GNU C コンパイラである gcc へ初めて MIPS R4K を移植しました。

Dawn Endico (endico@mozilla.org)

Dawn はツール細工師で Web マスターです。Mozilla のソースコードについての作業による LXR の習得に加えて、Dawn はこのサイトのドキュメンテーションを整理し、更新し続ける仕事をしています。

以前の生活で Dawn は、プロダクション・アーティストとして働いたり、デジタル・アートワークを展示したり、商用 Web サイトをデザインしたり、システム管理者として働いていました。

David Hyatt (hyatt@apple.com)

David の Greatest Hits (vol.1) には、XUL / XBL / Camino (以前は Chimera と呼ばれていました) / Mozilla Firebird ブラウザ が含まれるでしょう。

Gervase Markham (gerv@mozilla.org)

Gerv は 24 歳で、mozilla.org の最年少メンバーです。スタッフになる前、彼はボランティアとして数年間プロジェクトの仕事をしていました。2001 年 mozilla.org の最初の実習生になりました。

ライセンスプロジェクトの責任を持っていて、多くの時間を Bugzilla でのハックに費やしています。また、フットボール (つまり、サッカー) を楽しみ、上に書いたようなことをしていないときは、北ロンドン・エンフィールドにある 企業 のために 共同製作ソフト を書いています。

Myk Melez (myk@mozilla.org)

Myk はツール細工師で、時々ドキュメントを書いたりもしています。Bugzilla でハッキングし、世界がそのために多くなりすぎたとき、時々それをなだめすかして生き返らせるほかに、彼は必要ならば (Doctor のような) 新しいツールを書くために喜んで立ち上がります。彼は、毎日の作業と将来へ向けての壮大なビジョンを調和させるために最善を尽くしています。

mozilla.org の前、Myk は ForumZilla を書きました。その前は Web アプリケーション開発者として UC Santa Cruz で働き、その前は社会学の実地の分野で学位を得るために多くの時間と金とエネルギーを費やし、その前はキーボードサポートをビデオゲームにハックするために、スポーティーな 0.3MHz Apple IIe でプログラミング (どうやらこれらの部分の周りの共通のテーマ) を独習していました。

Daniel (Leaf) Nunes (leaf@mozilla.org)

Leaf は mozilla.org のリリースエンジニアで、先頭に立ってビルドの問題を捕らえます。彼は、Mozilla のソースが何か狩りに適したものへコンパイルできるかどうかといった、極めて些細なことについて頭を悩ますことに多くの時間を費やします。これは、彼がビルドシステムや開発者へのソース (とテスト実行) の準備のような詳細について悩んでいるということを意味しています。

Leaf は UC Santa Cruz と Fabrik のベテランで、彼のユーモアのセンスと同情は、午前 3 時に「そうキミがツリーを壊したのさ」という目覚ましコールを受けるみんなから非常に評価されています。

Seth Spitzer (sspitzer@mozilla.org)

Seth は Mozilla メール & ニュース クライアントに関する作業をおこなっていますが、その他の分野についても怖いもの知らずです。(ただし、一部のもの (レイアウトエンジン、JavaScript) に関しては、彼も怖がっています)

彼は XRE に何かが起きる、特に Mozilla Firebird と Mozilla Thunderbird が飛び立つと、いまだに信じています。

mozilla.org スタッフアソシエーツ メンバー

Peter Bojanic (peter@bojanic.ca)

Peter は、開発経験とビジネスの慧眼の捕らえどころのない組み合わせを持つ、エンジニアリングタイプです。彼は Mozilla テクノロジーを促進させ、技術的な用語を役員に翻訳して彼らの考えを読み、技術者たちにはビジネス用語を説明します。Peter は Linux と Mozilla をベースとした Penzilla オペレーティング環境を開発している OEone Corporation に勤めています。彼は、彼らの VP ソフトウェア開発で、住み込みのジェダイ (訳注: 映画 Star Wars の登場人物) です。

Frank Hecker (hecker@mozilla.org)

Frank Hecker は mozilla.org の政策ガリ勉家を自任し、アマチュアのドキュメント編集者です。その他のことでは、彼は Mozilla リライセンシング FAQ ドキュメント や mozilla.org ライセンスポリシー、Mozilla のセキュリティバグの取り扱いに関する方針、Mozilla 暗号化 FAQ の作成を手伝いました。Frank は、ソースコード公開の利益に関して、Netscape の経営をうまく売った人たちの 1 人で、「販売後のサービス」の精神で、彼はその後ずっと mozilla.org でボランティアをしています。

Marcia Knous (knous@mozilla.org)

Marcia は技術的に分類するのが難しい人物です。それは、彼女に技術的なバックグラウンドがあまりなく、人々の目に止まるような表立ったことをしていないためです。

彼女の mozilla.org における作業は、すべての CVS アカウントの処理や、物事がうまく行かないときの調停役など、多くの事務的な雑務を取り扱っています。Marcia は、Mozilla に関わる前、娯楽産業や法律関係の仕事をしており、現在でもフリーの著述業に手を出しています。最近、彼女は主に、現在の不安を「年の功」で扱うことを夢見ています。

Dan Mosedale (dmose@mozilla.org)

Dan は、しばらくの間、mozilla.org のツール (Bugzilla、Despot、ライセンス、統計スクリプテージ) と同様に、Mozilla の断片 (LDAP サポート、LDAP タイプダウン オートコンプリート、上品に振る舞うための UNIX Quantify の準備) について作業をしています。

mozilla.org に参加する前、Dan は Mozaic Communications 社のシステム管理者でした。そこから、彼は Netscape での IS アーキテクチャの仕事に転じ、Netcenter の Web サイトを世界的に広げるため、そこで使われている DNS サーバを共同で書き、Netscape のメール−ニュース ゲートウェイコードを拡張、維持しました。

Mike Shaver (shaver@mozilla.org)

Shaver はカナダ人の象徴です。日中彼は Mozilla 開発コミュニティに身を乗り出し、人々が何のために Mozilla のコードを利用しているのか、Mozilla のコードでどんな作業をしているのか、何がうまくいって何がだめなのかということに耳を傾けます。夜になると、彼はコードをハックし、時々重要なところを修正しています。たいてい、彼は、およそ同等の分量で、ガイダンスを発表することと T シャツに集中しています。

Mike はフリーソフト (特に Linux) 界のベテランで、Netscape 内では声が大きくて頑固なフリーソフトの擁護者です。Netscape の前、Mike は Ingenia (現在は Software Kinetics の 1 部門) というカナダの小さなコンサルティング会社で CTO を務めていました。
