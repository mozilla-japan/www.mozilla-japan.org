Lightning 0.3 の新機能

Lightning 0.3 は Mozilla Thunderbird  に対するカレンダー拡張機能の最新リリースです。私たちは全てのユーザの皆さんに、この最新バージョンへのアップグレードをお勧めいたします。

  • 電子メールで受け取ったイベントの基本処理が可能となり、簡単に iMIP イベントをカレンダーに追加できるようになりました。
  • ネットワークカレンダーの再読み込みができるようになりました。
  • 予 定表表示が深夜に次の日へと切り替わるようになりました。
  • 24 時間ビューは 数時間だけでなく、一日全体を表示できるようになりました。
  • アラームの信頼性が増し、イベントやタスクを確 実に通知できるようになりました。
  • ローカライゼーションサポートの改 善
  • 信頼性が大幅に向上し、データを失う可能性が大幅に減少しました。
  • 多数のパフォーマンス上の向上

リリースデータ：2006 年 10 月 11 日

ダウンロードとインストール

Lightning 0.3 ダウンロード

mozilla.org は、Windows、Linux そして Mac OS X に対応した Lightning をさまざまな言語で提供しています。 Lightning 0.3 を入手するには、こちらからダウンロードできます。

提供されていない他のシステムや言語のビルドについては、この文書の最後にある提供されているビルド項目をご覧ください。

Lightning のインストール

Lightning は Mozilla Thunderbird 1.5.x または来る Mozilla Thunderbird 2.0 で拡張機能として動作します。

重要： 古いカレンダー拡張機能を使っている場合、必ずプロファイルディレクトリにある ICS ファイルをバックアップしてください。 そして Lightning をインストールする前にカレンダー拡張機能をアンインストールしてください。Lightning をインストール後、あなたの古いイベントやタスクを、バックアップした ICS ファイルからインポートできます。

  1. ダウンロードするにはダウンロードリンクを右クリック（Mac では Control - クリック）して「名前を付けてリンク先を保存」を選択して、ハードディスクに .XPI ファイルを保存してください。
  2. Mozilla Thunderbird で、「ツール」メニュー下の「拡張機能マネージャ」（Thunderbird 1.5）か「アドオン」（Thunderbird 2.0）を開いてください。
  3. 「インストール」ボタンをクリックして、ステップ 1 でダウンロードした .XPI ファイルを見つけて選択してください。そして「OK」をクリックしてください。
  4. Mozilla Thunderbird を再起動してください。

即知の問題

このリストは Lightning 0.3 での即知の問題の一部です。このリリースの他の重要な問題は、こちらのページにリストされています。 新しいバグを報告する前に両方のリストをご覧ください。

全てのシステム

  • Thunderbird 1.5 をインストールしたときに、Lightning は Apple ical.app ユーザから送られた iMIP 招待状を読めません。 しかしながら、この動作は Thunderbird 2.0 では正確に動作します。 (bug 351997)
  • UTC 【訳注：協定世界 時】で表現される一日中のイベントは時々、Ligtning 予定表ペインで不正確な日付を 表示します。 (bug 353307)
  • Lightning は Thunderbird 1.5 の Windows 日本語バージョンで使うことができません。 (bug 330121)

Sun Java System Calendar Server (WCAP) サポートを統合した Lightning

WCAP サポートを統合した Lightning の機能強化されたビルドを利用できます。 こ ちらから入手してください。WCAP サーバーに対抗する Lightning を使うのを望まない限り、 通常の Lightning の標準ビルドをおそらく、あなたは探すべきでしょう。これがなぜ別々に出荷されているのかについての詳細は こ ちらのニュースグループへの投稿をご覧ください。

機能強化についての更に進んだ情報は Calendar:WCAP Guide【訳注：英語】を参照してください。

提供されているビ ルド
