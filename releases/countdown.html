1.0 へのカウントダウン

わたし達は 1.0 のゴールライン目前に来ています。Mozilla 1.0 をこれまででベストなものにするため、mozilla.org は Mozilla 1.0 リリ−ス候補 1（"RC1"）を作り テストとフィードバックを広く求めることにしました。このテストの結果によって Mozilla 1.0 ファイナルリリースに進むか、RC2 をはさむか を判断します。

Mozilla 1.0 そのものがリリースされる前に 1.0 関連の若干のステップを経ます。混乱を避けるために その概要を以下に示します:

  • あらゆるマイルストーンと同様、わたし達は CVS ツリーにブランチを作り、最終的な変更を加え リリース候補 1 のリリース準備を行います。
  • わたし達のゴールは 4月8日の週に このブランチを作ることです。このブランチに現れる user agent は "Mozilla 1.0 RC1" になります。ブランチの作成は 1.0 のリリースを意味するのではなく、リリース候補 1 が準備されたことを意味します。この user agent はわたし達の 1.0 リリースではなく、リリース候補を示すのです。
  • Mozilla 1.0 RC1 ブランチに最終調整がほどこされると、わたし達はそれを ダウンロード、テスト、フィードバックできるように リリースします。それは user agent の一部として "Mozilla 1.0" を含む "Mozilla 1.0 RC1" の実行可能バイナリが mozilla.org から利用可能になるということです。念を押しておきますが、これは Mozilla 1.0 のリリースを意味するのではなく リリース候補が利用可能になるということです。

あなたは RC1 をダウンロードして わたし達にフィードバックを送ることで ファイナルリリースのクォリティに貢献することができます。間もなく Mozilla 1.0 のお祝いができることを わたし達は楽しみにしています。
