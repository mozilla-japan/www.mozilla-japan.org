文書を寄稿する

原著者: Dawn Endico; 更新者: Alec Flett と fantasai

　Mozilla プロジェクトへの文書の寄稿は、ほとんど誰もができることです。ひとつ手伝いたいとお考えであれば、 mozilla.org にある文書をより良いものにする様々な方法について、以下の文章を参照してください。

ご支援方法

文書の寄稿はとても簡単なことです。実際の文書を寄稿することも出来ますし、現存する文書や作業中の文書の内容をご検証いただくことも、文書全般の体系化や整理をお手伝いしていただくことも可能です。さらに、古い文書は、今もなお適切で正確である事を期するために、継続して改訂されて行くべきです。ご支援頂けるためのさらに様々な方法を以下に述べます：

  • 問い合わせをされたり、ソースコードを読むことで、何か気付かれたことがあれば、その内容を説明する文書を書くことを検討するか、明確に伝えるためにコードにコメントを入れて拡張させて下さい。他の人たちも、たぶんあなたと同じような疑問を持つでしょう。あなたが発見された内容を関連するニュースグループへメールし、フィードバックを返してもらって下さい。mozilla.org にあなたの文書を公表する準備ができた時は、自分の仕事を登録してもらうサポートを求めて下さい。
  • 現存する文書の間違いをチェックいただき、報告して下さい。修正までしていただければ一層望ましいです。
  • もし、あなたが知りたい内容をカバーする文書がないことに気付かれたなら、欠けている文書を説明するバグを登録して下さい。
  • 知恵の宝庫としてニュースグループをあたって下さい。現行の文書に追加されるべき解説を見つけることがあるかもしれません。同じ質問が繰り返しされていれば、FAQ へ追加してもらって下さい。
  • 文書化に関する公開バグ の一覧表を確認してバグ修正につながるような興味深い内容をご指摘下さい。

管理の経過

ほとんどの文書は Bugzilla によって管理されます。私たちが新文書の検証を要求や宣言するために、さらに現存する文書についての問題点を報告するために、バグを登録する プロダクトは Documentation です。

Bugzilla の Documentation の各バグは、問題の文書に関係する読者によって、構成する三部のどれか一つへ属します：

Mozilla 開発者
Mozilla のコーディングそれ自体に従事する人たちにとって大変役に立つ文書です。これらは、ドキュメント固有のインターフェイスであるユーザーマニュアル (nsCOMPtr ユーザーマニュアルなど) および、どのように開発者が別製品に Mozilla を埋め込む 可能性があるかを説明している文書などを含みます。この部分はまた、ソースコードのコメント、特に LXR 向けのコメント を範囲に収めることが可能です。
Web 開発者
この部分は HTML、CSS、HTTP などについての情報を含みます。情報はコンテンツ制作者、ウェブサイトの管理者、システム管理者および、アプリケーション開発のプラットフォームとして Mozilla を使用している人たちを対象にしています。XUL のアプリケーションを書いているコンテンツ製作者もいますので、XUL の文書もここに入ります。
ユーザー
Mozilla のヘルプメニュからアクセス可能な文書、およびサイト上のすべてのエンドユーザー・サポート文書。

バグが登録されると、誰かが文書を書いたり修正したりした上で、mozilla.org のウェブサイトツリーに自分でチェックインするか、適切な権限をもっていて代理でチェックインしてくれる人にお願いする必要があります。

表示の手引き

もしあなたが、mozilla.org のために文書を作成しておられるならば、私たちの 表示の手引き に準拠するようにして下さい。Mozilla をまだ使えないという理由で私たちのサイトを読む人が多数います。従って、あなたの文書が様々なブラウザで読むことが出来るようにして下さい。

CVS

CVS は私たちのウェブサイトで文書管理に使用されます。現在の文書に変更を加えたい時は、CVS を使って 文書をチェックアウトし cvs diff -u によってパッチを作成するか、変更中の文書の最下部にある「このページを編集する」をクリックして Doctor を使ってパッチを作成することが可能です。

パッチが完成したら、Bugzilla のバグ報告書へパッチを添付して文書の所有者宛てにファイルを送り、レビューを依頼してください （アクセス権を持っていない場合は、変更のチェックインも依頼してください）。文書に所有者名が表示されていなければ、ページの最下部にあるリンク「文書の履歴」をクリックして、最近ページを改訂している人物を見つけ見直しを依頼します。これら全てが上手くいかない時は、 webmaster@mozilla.org へメールをして下さい。

ソースコード以外にも優れた実績をお持ちで、特に、文書を寄稿されて公式な所有者と維持管理者になっておられる場合は、レポジトリー (repository) へのアクセスが認められます。アクセスするためには、CVS Contributor Form を送る必要があります。私たちのドキュメントツリーに固有な CVS 情報用の mozilla.org Content and CVS をお読み下さい。文書 CVS によるソースコード は、あなたがお使いのプラットフォーム用の CVS の入手場所を説明し、CVS 文書へのポインターを備えています。

レビュー要件

mozilla.org の文書化におけるレビュー要件はとても簡単です。文書への大きな変更を登録する前に、所有者の同意を得て頂きたいと思います。また 文書化表現の手引き で定められたガイドラインに従って下さい。あなたは、fantasai に自分の文書を一通り確認することを依頼すれば、HTML&CSS コードを完璧に検証してもらうことが出来ます。

developer.mozilla.org での検証の請求はもっと厳格です。新しいファイルの登録の前に、ファイルの位置の選定について fantasai の了承を得なければなりません。developer.mozilla.org 上のすべての文書は、文書化表現の手引き の規則に忠実であることが求められます。

法律に関わる事項

私たちは、著作権の在り方について取り組んでいるところです。将来は、情報の共有化を奨励する著作権の方針を制定するつもりです。それはできれば、Open Publication Licence や Linux Documentation Project のように、著者が自分自身の著作権を保持しながら、複製と再配布に関する特権を（特定の条件のもとで）包括的に拡大する、と言うものです。

文書化についてあなたのご意見をお待ちしております。 documentation newsgroup へご意見を送って下さい。
