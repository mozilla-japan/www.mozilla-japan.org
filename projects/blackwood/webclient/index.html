概要

綱領: 

webclient プロジェクトは、包括的なウェブブラウジング機能を与える、最高のブラウザに依存しない Java API を提供することを狙っています。この機能は以下の機能を含みますが、これらに限定されません: 内容のレンダリング、移動、履歴の仕組み、進行の通知。実際に実装される機能は基礎をなすブラウザの実装に依存します。 

mozilla.org 前史

    このプロジェクトは Kirk Baker <kbaker@eb.com> と Ian Wilkinson <iw@ennoble.com> によって開始されたものでもともと RaptorCanvas と呼ばれていました。1999年の第一四半期に、AOL/Netscape/Sun の契約が成立し、Sun でのグループは Java と Mozilla とをより良く結びつけるために blackwood プロジェクトを開始しました。blackwood のサブプロジェクトの一つは mozilla webshell に Java API を提供することで、 Krik と Ian が既に成し遂げていました。だから、 webclient の開発で Sun は Krik と Ian に協力しました。

必要条件

    webclient をビルドし使用するのに求められる必要条件はどの webclient のマイルストーンを使いたいのかによって変わります。更なる情報を得るためには /projects/blackwood/webclient/release-notes/ でリリースノートのページを見てください。

コードの入手とビルド

コードの入手

    あなたは最新の webclient をビルドすることも、マイルストーンリリースをビルドすることも出来ます。マイルストーンリリースについて、更なる情報を得るためには /projects/blackwood/webclient/release-notes/ を見てください。

 

コードのビルド

    webclient のビルドに関しては、 /projects/blackwood/webclient/release-notes/m3-detail.html#read_this にあるリリースノートでの説明に従ってください。

Webclient 開発者へのガイド

    開発者へのガイド は Java アプリケーションへ Mozilla ブラウザを統合するため Webclient API を使うプログラマが使うことを意図しています。

Webclient 実装へのガイド

    実装へのガイド は Mozilla.org Open Source Public License のもとでの Webclient API の充実への貢献を願うオープンソースコミュニティーのメンバーが使うことを意図しています。

webclient の使用

    どのように基本的な webclient の機能を使用するかを説明する java-webclient ディストリビューションのブラウザの実例があります。このクラスは便利であるようにここの [1] リファレンスに含まれます。

    あなたは webclientについて Codestock にあるウェブ上のプレゼンテーションを見たいと思うかもしれません。http://www.mozilla.org/projects/blackwood/webclient/codestock/index.htm

    基本的に、 java.awt.Canvas のサブクラスとして BrowserControlCanvas を作り、それをあなたの表示の階層に追加します。

     

        // ブラウザを作成する
        BrowserControlCanvas browser  = 
                       BrowserControlCanvasFactory.newBrowserControlCanvas();
        add(browser,      BorderLayout.CENTER);
    そして BrowserControl インターフェイスを通して組み込みの Mozilla を操作することが出来ます。
public void actionPerformed (ActionEvent evt) {
    String command = evt.getActionCommand();
    
    try {
        if (command.equals("Back")) {
            if (browserControl.canBack()) {
                browserControl.back();
                int index = browserControl.getHistoryIndex();
                String newURL = browserControl.getURL(index);
                
                System.out.println(newURL);
                
                urlField.setText(newURL);
            }
        }
        else if (command.equals("Forward")) {
            if (browserControl.canForward()) {
                browserControl.forward();
                int index = browserControl.getHistoryIndex();
                String newURL = browserControl.getURL(index);
                
                System.out.println(newURL);
                
                urlField.setText(newURL);
            }
        }
        else if (command.equals("Stop")) {
            browserControl.stop();
        }
        else {
            browserControl.loadURL(urlField.getText());
        }
    }
    catch (Exception e) {
        System.out.println(e.toString());
    }
} // actionPerformed()
API ドキュメントはここ [2] から手に入れられます。

要求仕様

    webclient の要求仕様は http://www.mozilla.org/projects/blackwood/webclient/RequirementsSpec.html で手に入れられます。私たちは要求仕様へのフィードバックを歓迎します。ニュースグループ netscape.public.mozilla.java に全てのフィードバックを投稿してください。件名: webclient spec を使ってください。

必要条件の分析

    webclient にとって、必要条件の分析は要求仕様を入手し、何が今日のブラウザの API 技術で実際にビルドできるかを理解する作業となるでしょう。webclient の必要条件の分析のドキュメントは http://www.mozilla.org/projects/blackwood/webclient/RequirementsAnalysis.html で手に入れられます。

設計

    この節は設計文書へのリンクを含んでいます。現在、設計文書は mozilla を基にした webclient の実装のためにあります。それらは http://www.mozilla.org/projects/blackwood/webclient/design/spec-compliant-design.html で見ることが出来ます。

貢献する方法

    webclient に貢献できるいくつかの方法があります。mozilla の貢献者になることの詳細について知る必要がある場合、mozilla の実装に取り組むことが出来ます。あなたは mozilla への貢献についてのこのドキュメント、http://www.doczilla.com/development/extmoz.html を読みたいと思うかもしれません。

    私は mozilla の実装に集中しています。そのため、もし助けになるものが欲しいならば、問題のページまたは pending（未解決） grep ページを熟読してください。

    あなたは別のブラウザの実装の周りで webclient をラップすることに取り組むことができ、これは価値のある作業です。現在、IE についてはラップする準備が整い、fizzilla については始めるための準備を進めています。

    もしこれらの貢献の選択肢のいずれかにでも興味を持ったら、下のアドレス宛てで私に連絡してください。

20030929

ここに開発計画を示します。最初、私は以下のすべてを GNU/Linux RedHat 7.2 で行います。あとで Windows に取り組みます。

  • ブックマークに関する最初のテストを終える。

  • BrowserControlCanvas を親ウィンドウに加えたり、表示したり、隠したり、リサイズすることなどができることを示す、ユニットテストとコアコードを書く。

  • BrowserControlCanvas を親ウィンドウに加えたり、一連のウェブページに移動できることを示す、ユニットテストとコアコードを書く。

  • mozilla/java/webclient/src_moz/gtk 内のいやな「スタブ」の残りを削除する。私はもはやこれには JDK 1.3 が必要ではないことがわかりました。

  • ワークスペースにタグを付ける。

  • 私がコミュニティーから助けを受けるためにドキュメントとビルドの説明を更新する。

私にはこれに貢献する非常に限られた時間しかないので、私は出来るだけ早く最終段階へ到達しようと思います。人々が貢献できるように現在のやり方を説明しようと試みるに値しません。私は誰かに助けてくれるよう頼む前に、それを相当きれいにしなければなりません。

2002年 5月 7日

私は win32 で大体動く webclient を作りました。以下の特徴がテストされ、よく動きました。

  • ファイル->新しいウィンドウ

  • ファイル->閉じる

  • 表示->文字列としてページのソースを表示

  • 検索->検索

  • 検索->次を検索

  • 編集->すべて選択

  • 編集->コピー

  • 履歴->戻る

  • 履歴->進む

  • 履歴-> <移動の回数>

  • ブックマーク->ブックマークの管理

    ブックマークウィンドウがポップアップし、ダブルクリックすることで webclient がそのページに行くことが出来ます。

  • ストリーム->ファイルからストリームを読み込む...

  • ストリーム->ランダムな HTML 入力ストリームを読み込む

  • マウスオーバーイベントが機能する

  • DOMViewer が機能する

  • 移動ボタンが機能する。

これには、ブックマーク->現在のページを追加する、ブックマーク->新しいフォルダに現在のページを追加する、プロファイル->プロファイルを作成する、及び プロファイル->プロファイルを削除する、が残っています。最初の二つは易しいはずで、二番目の二つでは、それらの実装は現在完全に活動の停止した BlackConnect に基づいたものですから、たぶん取り出すだけでしょう。

win32 上の Webclient の状態に満足したら、それが linux 上でも動くようにします。
