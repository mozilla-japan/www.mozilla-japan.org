The Lightning Project

The Lightning Project is a redesign of the Calendar component. Its goal is to tightly integrate calendar functionality (scheduling, tasks, etc.) into Mozilla Thunderbird.

Some of our future goals are (none of this is implemented right now):

  • Better Thunderbird integration
    • email<->task linkage
    • Integration with the Thunderbird address book
    • iMIP support (meeting invitations)
  • Improved CalDAV support
  • Cached calendar support for offline calendar use
  • Device synchronization

Download Lightning

Lightning 0.1 RC2 (Release Candidate)

Pick the appropriate XPI file for your platform, save it to your hard disk from your browser, and then install it in Thunderbird 1.5 or 1.5.0.*. Please read the Lightning 0.1 release notes before you use this release candidate.

  • Windows
  • Linux
  • MacOS X

Lightning nightly builds (development builds)

Nightly Lightning builds are now available for development and testing.

  • Trunk builds (for Thunderbird 3.0 candidates)
    • Windows
    • Linux
    • Mac
  • Mozilla 1.8 branch builds (for Thunderbird 1.5 and 2.0 candidates)
    • Windows
    • Linux
    • Mac

Please be aware, that the use of nightly developer builds has some risks associated with it. Don't use them with prodcution data.

  • There are KNOWN DATALOSS BUGS in the calendaring code.
    • Don't trust these builds with important calendaring or mail data
    • Always make backups (one possible strategy for Calendaring data backups is described at Calendar:WebDAV testing harness)
  • Use these extension XPIs with the Thunderbird builds next to them. Note, however, that any trunk Thunderbird build of similar vintage is likely to work.
    • Both are built automatically from the CVS trunk or Mozilla 1.8 branch every night, so they may randomly be broken if the development tree is not in a good state
    • Please report bugs at Bugzilla

Building Lightning

Detailed steps to build Lightning from the Mozilla cvs code base can be found at the mozilla.org wiki.
