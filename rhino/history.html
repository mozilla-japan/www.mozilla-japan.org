Rhino の経歴

Rhino の名前の由来は JavaScript について書かれたオライリー社発行書籍の表紙になった動物からきています。

Rhino プロジェクトは1997年の秋に Netscape によって開始されました。当時、Netscape 社は全てを Java で書かれた Navigator を企画しており、Java で書かれた JavaScript 実行環境が必要でした。 Netscape 社が、Javagator と呼ばれる、その Navigator の作業を中止した時、Rhino はどうにかして中止という斧から逃れました。（噂では幹部社員が Rhino の存在を忘れていた。） その後、二、三の主要な会社（Sun Microsystems 社を含む）が Rhino のライセンスを取得し、自分達の製品に使う為に Netscape 社に資金を払った為に、Rhino の開発は続行されました。現在、Rhino は Netscape 社のサーバー製品の一部として予定されています。

本来、Rhino は、全ての JavaScript コードを、生成されたクラス・ファイルの Java バイト・コードにコンパイルしていました。これは最良の性能（ JIT 上で動いている時、頻繁に JavaScript の C 実行性能を打ち負かしました。）を実現しましたが、以下の二つの不具合に悩まされました。一つは、Java バイト・コードを生み出すことや、生成されたクラスをロードする事はとても重い作業の為、コンパイル時間が長くかかるという事。そして、もう一つは、殆どの JVM は使用しないクラスや、クラス・ファイルをロードする結果として抑留された文字列を、厳密には収集しない為、その実行環境はメモリー・リークを起こしやすいという事でした。

その為、1998年の秋には、Rhino には interpretive mode (解釈モード) が足されました。クラス・ファイル生成コードは選択となり、動的にロードされるパッケージとなりました。コンパイル作業は早くなり、スクリプトが使われなくなった時は、他の Java オブジェクトの様に、スクリプトは収集される事ができるようになりました。

Rhino は1999年の四月に mozilla.org のサイトでリリースされました。本来、Rhino クラス・ファイルの生成はリリースを控えるはずでしたが、Rhino のライセンスは、現在、クラス・ファイルの生成を含む Rhino を構成する全部分を、オープン・ソースとしてリリースされることに承諾しました。オープン・ソースの世界にリリースされることによって、Rhino は様々なユーザーと、コードに貢献する人々の増加を獲得しました。
