質問の答え

The Answers

全般

General

Sunbird と Mozilla Calendar との違いは何ですか？
Mozilla Calendar は、 Mozilla、 Mozilla Firefox や Mozilla Thunderbird といった、Mozilla 関連製品の拡張機能であるのに対し、 Sunbird は全く同じ機能を提供する、単一のプログラムの形をとっています。 両方のベースコードは同じなので、機能も同様なのです。 そのため、同じ バグ も存在しますし、同様に修復もされます。
また、プロジェクトが違います： URL を開くのに、 Mozilla または Firefox を使用します。 Email アラームは、 Mozilla または Thunderbird を使用します。

What's the difference between Mozilla Calendar and Sunbird?
Mozilla Calendar is the calendar extension for Mozilla products such as Mozilla, Mozilla Firefox and Mozilla Thunderbird. Sunbird is the standalone form of the same application. Both use the same base code so their functionality is the same and they share the same bugs and bug fixes. A couple features currently depend on the underlying product: Open URL works with Mozilla and Firefox. Email alarm works with Mozilla and Thunderbird.

Mozilla Calendar をインストールするのに必要な条件は何ですか？
Mozilla のバージョンが1.0 以降である事が必要です。 最新のナイトリービルドでも動くはずです。 Calendar をインストールするには、 ダウンロードとインストールへ。 1.0以降の Mozilla リリースなら、すべてのバージョンでカレンダーをダウンロードする事が出来ます。 もしエラーが発生する場合は、他の拡張機能より先にカレンダーをインストールしてみてください。
What do I need to install Mozilla Calendar?
You need a current version of Mozilla, either 1.0 or higher. It should work in the latest nightly releases. You can find different versions for all Mozilla releases since 1.0 on the download page. If there is trouble, try installing Calendar before any other extensions.
現在インストールされている Mozilla Calendar のバージョンを確認するにはどうすればよいですか？
Calendar を起動し、Help | About Calendar 【訳注：[ヘルプ] ->[Calendarについて]】 メニューを選択してください。 Calendar のバージョンは、2004080310-cal のように、末尾に -cal のついたタイムスタンプで表されます。
How can I tell what version of Mozilla Calendar is installed?
Start Calendar, then click menu Help | About Calendar. The Calendar version is a date and hour stamp that ends with -cal, such as 2004080310-cal.
Mozilla Calendar をインストールするにはどうすればよいですか？

もし、Firefox 0.9.2 以降または Thunderbird 0.7.2 以降で、20040803以降のCalendar xpi を使用しているのなら、 Tools 【訳注：ツール】メニューの Extensions Manager 【訳注：拡張機能】 を使用してCalendar xpiをまずアンインストールしてください。

20040803 より前の Calendar は、ユーザーインターフェースを使用したアンインストールができないので、以下のファイルを直接削除します。： 【訳注： &Zt;appdir>：アプリケーションのインストールされているフォルダ】
<appdir>/chrome/calendar.jar
<appdir>/chrome/chrome.rdf
<appdir>/components/xpical.dll (windows の場合)
<appdir>/components/libxpical.so (linux の場合)
<appdir>/components/calendar.xpt
<appdir>/components/calendarService.js

How do I uninstall Mozilla Calendar?

If you are using Calendar xpi's 20040803 or later on Firefox 0.9.2+ or Thunderbird 0.7.2+, uninstall from the Extensions Manager in the Tools menu.

Otherwise, uninstalling through the user-interface is not yet possible. But you can delete certain files to accomplish that task:
Delete <appdir>/chrome/calendar.jar
Delete <appdir>/chrome/chrome.rdf
Delete <appdir>/components/xpical.dll (on windows)
Delete <appdir>/components/libxpical.so (on linux)
Delete <appdir>/components/calendar.xpt
Delete <appdir>/components/calendarService.js

「８月の第一月曜」のように、繰り返しの予定を入れるにはどうすればよいですか？
12ヶ月ごとの予定を入力するには： new event 【訳注：新しい予定？】 ダイアログで、「2000年8月7日」のように、 start date 【訳注：開始日】を８月の第１月曜日に設定します。 recurrence 【訳注：繰り返し？】タブを選択し、 "Repeat every" 【訳注：？？？？】 にチェックを入れて、間隔を 12 、単位を "Months" 【訳注：月】に設定して、 "First Monday of the month" 【訳注：月の第一月曜】をクリックします。
How can I make a recurring event on, say, the first Monday every August?
Set the event to repeat every 12 months: In the new event dialog, set the start date to the first Monday of August of the first year, say 2000-08-07. Click the recurrence tab, check "Repeat every", set interval to 12, set unit to "Months", then click "First Monday of the month".
リモートサーバーで自分の予定を公開するにはどうすればよいですか？
Calendar のイベントは、FTP サーバか、 webDAV に対応したサーバで公開できます。 逆に、サーバに公開されているイベントをあなたの Calendar に取り込む事もできます。
How can I publish my events on a remote server?
You can publish events from the calendar to an FTP server or a webDAV enabled webserver. You can also use the calendar to subscribe to these events as well.
How can I serve my shared calendars?

カレンダーを共有するには、webDAV サーバを使います。 自分で web サーバを持っているのでしたら、 Apache 用のフリーのモジュール、 mod_dav があるので これをインストールして下さい。これで、 web サーバは webDAV サーバにもなります。（その方法は上記のページに書いてあります。）
webDAVサーバが使えるようになったら、あなたのカレンダーのイベントをそのサーバに公開すれば、そのイベントをすべての Mozilla Calendar に登録できます。 すると、あなたの更新に合わせて他の人のカレンダーも自動的に更新され、安上がりのカレンダーサーバは完成です。

追加情報として、ここや、この文章も参考にしてください。

How can I serve my shared calendars?

To share your calendars, you need access to a webDAV server. If you run your own web server, you can install mod_dav, a free Apache module that will turn your web server into a webDAV server. Instructions on how to set it up are on their website. Once you set up your webDAV server, you can publish your calendar to the site, then subscribe to it from any other Mozilla Calendar. Automatically updating the calendar will give you a poor man's calendar server.

Additional info available here.

Import, Export

Mozilla Calendar の使用しているファイル形式は何ですか？
iCal 標準フォーマット を使用しています。 このデータファイルは、Mozilla のプロファイルが保存されているフォルダの、Calendar という名前のフォルダの中にあります。
<
What is the file format for the calendar?
The calendar uses the iCal standard for storing its calendar data. The file is located in a Calendar/ subdirectory of your Mozilla profile.
Outlookとカレンダーを共有できますか？
Outlook はデータを iCal 標準フォーマットで扱っていないので、直接共有はできません。 でも、 Outlook は、イベントを iCal 標準フォーマット( ics 形式)でエクスポートできるので、 それを Calendar にインポートして下さい。 Linux で使用するなら、 このページが参考になるかもしれません。 (Outlook との共存には、 bug 167102 も参考にして下さい。)
Will it work with Outlook?
Outlook does not store its calendar data in an open standard format, so Mozilla calendar currently does not support Outlook directly. However, you can export your Outlook events as an .ics file, and import them into the calendar. Linux users may find this page of some use. (Follow bug 167102 for more details on Outlook integration)
Netscape のカレンダーサーバを使用できますか？
いまのところ、 Calendar は Netscape のカレンダーサーバと連携する事はできません。 カレンダーを共有するには、 Tools->Subscribe 【訳注：[ツール] -> [別のカレンダーを購読] 】メニューで 公開されているカレンダーを取り込んだり、 Tools->Publish 【訳注：[ツール] -> [選択イベントを公開] 】メニューで、 webDAV に対応したサーバにあなたの予定を公開してください。
Will it work with Netscape's calendar server?
The calendar does not currently work with Netscape's calendar server. To share calendars, subscribe to remote calendars using Tools->Subscribe and publish your calendars back to a webDAV enabled server by using Tools->Publish.
アップルの iCal とカレンダーを共有できますか？
iCal も Mozilla Calendar も同じフォーマットを使用しているので、お互いのイベントを問題なく表示できるはずです。 なので、アップルのサイトにあるイベントリスト 【訳注：日本語はこちら】も登録できます。
Does it work with Apple's iCal?
iCal and the Mozilla Calendar use the same file format, so events from one should show up without problem in the other. You can also subscribe to the list of events on Apple's website if you like.
Palm Pilot と同期できますか？
今の所、同期することはできません。
Does it synch with my Palm Pilot?
The calendar does not yet synch with your Palm Pilot.
カレンダーファイルをインポートしても、何も表示されないのですが。
(すべての 【訳注：アイテム？】)をインポートしているみたいなのですが、表示されません。
デフォルトに設定されているカレンダーが、(Calendars タブで） not visible【訳注：非表示】にされている気がします。 新しい予定を設定するときは表示され、その後すぐに非表示になってしまい、クリックするとクラッシュするという バグ が報告されています。
簡単に言えば、デフォルトに設定されているカレンダーが非表示になっていないか確認してください。
Why doesn't it show any events when importing a calendar file?
(It seems to be importing all the items but doesn't show any events.)
Sounds like your default calendar might be set to not visible (in the Calendars tab). There is a bug that sets a non-visible calendar to visible for a short time while adding a new event to that calendar. The events are drawn to view during that time, but when you click one of the events the calendar is not available, and a crash results.
In short, check if your default calendar is visible..

The Long date format and Short date format are defined by the local operating system so date formats are consistent across the screen.

You can choose between the Long date format or Short date format in the calendar preferences to control how dates appear in tables and exported text.

How can I access Local Calendar(s) from different Mozilla products?
Suppose Firefox calendar is the base application. Switch to the "Calendars" tab. Right click and select "New Calendar". Enter a location and filename to be your base calendar file. To test add an event and make sure the event has been added to that file by viewing it. Now launch Thunderbird calendar and do the same with the same filename. Once you get the "Overwrite?" dialog choose "Yes". Refresh your view by navigating between months. You should see the event you added in firefox calendar. Proceed with sunbird just the same.

Mozilla/Firefox/Thunderbird を起動しないでカレンダーを開くにはどうすればよいですか？
Mozilla の実行ファイルに "-calendar" パラメータをつけたショートカットを、デスクトップやクイックランチに置いてください。 これで Calendar を直接起動できます。
  • Windows では、コントロールを押しながら Mozilla、 Thunderbird、 または Firefox のアイコンをドラッグしてコピーしてください。 そしたら複製されたアイコンを右クリックしてプロパティを選択します。まず名前をCalendarに変更し、[ショートカット]タブの参照先を以下の用に変更します： クオーテーションマークの次に、-calendar コマンドオプションを追加します。 例： 
    "C:\Program Files\Mozilla Thunderbird\thunderbird.exe" -calendar
  • Mac OSの場合、(firefoxを使用していて、[アプリケーション]ディレクトリにインストールされているものとします。）
    /アプリケーション/AppleScript にある、スクリプトエディタを起動します。
    ウインドウの上の入力枠に、以下の様にタイプしてください。 
    do shell script "nohup /Applications/Firefox.app/Contents/MacOS/firefox -calendar > /dev/null 2>&1 &"
    ファイルメニューから、保存をクリックします。
    後で分かるようにファイル名をつけ、フォーマットを アプリケーション にして保存をクリックします。
    Viola! 【訳注：やったね???】 これでカレンダーを起動するアップルスクリプトができました。
  • どんな OS でも、
    firefox -calendar
    のように、Mozilla/Firefox/Thunderbird のコマンドの後にコマンドオプションを付ければ、Calendar が起動します。
How can I open the calendar without opening Mozilla/Firefox/Thunderbird first ?
Create a desktop/quick launch icon as a link to Mozilla executable with the "-calendar" parameter. This should start Calendar directly.
  • On Windows, control-drag the Mozilla, Thunderbird, or Firefox shortcut icon to copy it, then right-click and choose Properties. Rename the copy to Calendar. In the Target input box of the Shortcut tab, put the -calendar command option after the quote, for example 
    "C:\Program Files\Mozilla Thunderbird\thunderbird.exe" -calendar
  • On Mac OS:(assuming the application is firefox and that it is installed in the /Applications directory)
    Open Script Editor (located in /Applications/AppleScript)
    In the top pane of the script editor window, type: 
    do shell script "nohup /Applications/Firefox.app/Contents/MacOS/firefox -calendar > /dev/null 2>&1 &"
    Select "Save As..." from the "File" menu
    Enter a meaningful name in the "Save As:" box, select "Application" as the "File Format:" and click save.
    Viola! an applescript has been created that starts the calendar application.
  • On any OS command line, type the command option after the Mozilla/Firefox/Thunderbird command, for example, 
    firefox -calendar

Help Develop Calendar/Sunbird

ユーザーインターフェースを変更したり、パッチを付け加えるにはどうすればいいですか？
GUI は XUL、JavaScript、CSS で書かれています。また、ローカライズテキスト用には、DTD とプロパティファイルが使われています。 【訳注：？？】 これらのファイルは、すべて、user extensions extensions ディレクトリ (Firefox, Thunderbird の場合) またはアプリケーションディレクトリ (Mozilla Suite の場合) の、 chrome/calendar.jar に保存されています。
  • Linux の場合、 profile extensions のパスは、だいたいこんな感じです。 ~/.mozilla/firefox/default.xxx/extensions/{8e117890-a33f-424b-a2ea-deb272731365}/chrome/calendar.jar
  • アプリケーションのパスは、 多分 /usr/local/mozilla/chrome/calendar.jar でしょう。
  • Windows の場合、 profile extensions のパスは、だいたいこんな感じです。 C:\Documents and Settings\(user)\Application Data\Mozilla\Firefox\Profiles\default.xxx\extensions\{8e117890-a33f-424b-a2ea-deb272731365}\chrome\calendar.jar
  • On Windows the application directory path is probably something like c:\Program Files\Mozilla\chrome\calendar.jar
calendar.jarを解凍してください。中には３つのディレクトリが入っています： content, locale, skin.
  • content: XUL と JS が、コンポーネント 【訳注：?】とイベントハンドラを定義しています。 XBL ファイルは、 XUL と JS の両方を含んでいます。
  • locale: DTD とプロパティが、地域ごとのテキストを定義しています。
  • skin: CSS とイメージが、デフォルドテーマの見た目を定義しています。
変更を加えるには：
  1. calendar.jar を解凍し、
  2. ファイルを編集して、
  3. calendar.jar にパッケージし直して、
  4. Mozilla を再起動してテストします。
以上です。

javascript.options.strict を true にしてテストしてください (ブラウザで、以下のURI about:config を入力して環境設定を表示させます)。 よくある問題に JavaScript コンソールが警告を出してくれます。

もし、 CVS のファイルが最新のインストール XPI よりも新しいなら、 CVS よりファイルをダウンロードして jar にパッケージした方がバグ取りの為には有効です。 （ディレクトリ構造には注意してください）

How can I tinker with or patch the user interface?
The GUI is written in XUL, JavaScript, and CSS, with DTDs and properties files for localized text. They are all contained in the chrome/calendar.jar in either the user profile extensions directory (Firefox, Thunderbird), or the application directory (Mozilla Suite).
  • On Linux the profile extensions path is probably something like ~/.mozilla/firefox/default.xxx/extensions/{8e117890-a33f-424b-a2ea-deb272731365}/chrome/calendar.jar
  • On Linux the application directory path is probably something like /usr/local/mozilla/chrome/calendar.jar
  • On Windows the profile extensions path is probably something like C:\Documents and Settings\(user)\Application Data\Mozilla\Firefox\Profiles\default.xxx\extensions\{8e117890-a33f-424b-a2ea-deb272731365}\chrome\calendar.jar
  • On Windows the application directory path is probably something like c:\Program Files\Mozilla\chrome\calendar.jar
Unzip the calendar.jar. It contains three directories: content, locale, skin.
  • content: XUL and JS, defines the components and event handlers. XBL files contain both XUL and JS.
  • locale: DTD and properties, defines the text for each locale.
  • skin: CSS and images, defines the look for each default theme.
To make a change:
  1. Unzip the calendar.jar,
  2. Edit files,
  3. Pack it back up into calendar.jar,
  4. Restart Mozilla and test.
That's it.

Test with javascript.options.strict set to true to get JavaScript console warnings for common problems (open URI about:config in browser to set preference).

For fixing bugs, if files in cvs are more recent than the latest install xpi, it is better to get files from cvs, edit, and pack them into the jar (note directory structure).

Mozilla Calendar の最新のソースはどこにありますか？
Mozilla's CVS の、 mozilla/calendar にあります。詳しくは、 Mozilla 説明書 のソースコードの取得方法に従ってください。

また、ソースをブラウズ したり、 検索 する事も出来ます。そして、直接 /mozilla/calendar. をみて、何が 変更されたのか、 確認することもできます。

Where do I get the latest source for Mozilla Calendar?
The source for the calendar is in Mozilla's CVS under mozilla/calendar. Follow Mozilla's instructions on how to get the source code.

You can also browse or search the source, or check what just changed in directory /mozilla/calendar.
