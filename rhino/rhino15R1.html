Rhino 1.5 Release 1 新着情報

ECMA 262 Edition 3 (第 3 版／ 3rd Edition) に適合

Rhino 1.5 は JavaScript 1.5 の実装で、ECMA 262 Edition 3 (しばしば "ECMAScript" と称される) に適合しています。 Edition 3 では、JavaScript 1.4 に存在していた以下のものを含む JavaScript の機能が標準化されました …
  • 正規表現 (regular expression)
  • switch ステートメント
  • do...while ループ
  • ステートメントのラベルと、break と continue のラベル指定
  • オブジェクト・リテラル (オブジェクト直定数)
  • ネストされた(入れ子の) 関数
  • 例外ハンドリング (例外処理)
  • instanceof 演算子
  • in 演算子
さらに、Edition 3 と JavaScript 1.5 に、これらを含む新機能が追加されました …
  • Perl 5 の正規表現。これには、くり返し優先の数量子 (greedy quantifiers) のような演算子を含んでいます。
  • 例外の形のエラー
  • 数字の整形 (number formatting) (Number.prototype.toFixed, Number.prototype.toExponential, Number.prototype.toGeneral)

Rhino 1.4 Release 3 からの変更点

オープンソースへの初期リリース (1.4 Release 3) からの、Rhino に関する他の重要な変更は、以下の通りです。 ここでは、バグ・フィックスについては言及せず、API の変更や重要な機能の変更について扱っています。

コンパイル・モード

Rhino では 2 つの実行モードが利用できます。 インタープリター・モードは、Java により実現される、インタープリター・ループです。 コンパイル・モードは、JavaScript コードを クラスファイルの Java バイトコードへとコンパイルします。 このコンパイル動作は、インタープリターにある 同様の API を使用して、スクリプトの評価の一環として行うこともでき、コンパイル時ステップを分離できます。 そのコードは、インタープリターでは、org.mozilla.javascript.optimizer パッケージ内にあります。

JavaScript Compiler

配布物は現在、コマンドラインから呼び出せる、特別なクラスを含んでいます。これは jsc (=JavaScript Compiler) です。 このツールは、JavaScript を元に Java クラスを作成するのに使用できます。 任意のインターフェースを実装 (implement) したり、任意のベースクラスを拡張 (extend) して、Java クラスを作成可能な、オプションがあります。 このオプションで、JavaScript のスクリプトが、Applet や Servlet のような主要な規約を満たせるようにできます。 http://www.mozilla-japan.org/rhino/jsc.html を参照してください。

LiveConnect 3

Rhino は現在、LiveConnect 3 仕様 (あるいは LC3) をサポートしています。最も注目すべき変更点は、多重定義／オーバーロード・メソッド解決 (overloaded method resolution) のサポートです。 LiveConnect Release 3 目標／機能 を参照してください。

Java のプロパティーを反映したような JavaBeans プロパティー

getFoo/setFoo のメソッドを持つ Java クラスは、JavaScript への反映では "foo" プロパティーになるはずです。 ブーリアンのメソッドも反映されます。

ダイナミック・スコープのサポート

Rhino 1.5 では、ダイナミック・スコープのサポートを実装しています。それは特に、サーバー組み込みのようなマルチスレッド環境に役立ちます。

ScriptableObject.defineClass のための新しいセマンティクス

Java クラスを使用して JavaScript オブジェクトを定義するための古い規定はバロック式でした。 【訳注: 過度に装飾的？】 それらの規定は今もなおサポートされていますが、よりきれいな定義が現在サポートされています。 詳しくは、javadoc を参照してください。

Java 2 の -jar オプションをサポート

現在は、Java 2 で新しく用意された -jar オプションを使用して、シェルを開始することが可能です。

シェルの変更

2 点の変更 ⇒ トップレベル変数 "environment" と "history" が追加されました。

Java クラスをスクリプトに見えるように

JavaOne にて、ある出席者から、多くの組み込みでは、スクリプトがすべての Java クラスにアクセスできる必要はないだろう、との指摘がありました。 これは素晴らしい指摘であり、SecuritySupport インターフェースの追加を実施しました。 これは、組み込み開発者が、どのクラスをスクリプトに公開するのかを選択できるものです。 【訳注: Rhino 1.5R4 以降は, 新しいセキュリティー・インターフェースに置き換えられた】

SecuritySupport と JavaAdapter

Andrew Wason 氏が指摘した、新しい JavaAdapter 機能 (クラスファイルを生成することで、任意の Java インターフェースを実装した JavaScript オブジェクトを可能にするもの) の問題があります。 それは、SecuritySupport インターフェースをサポートしていませんでした。 Rhino へ、組み込みルーチンに バイト配列からクラス生成を委譲できる ようなインターフェースです。 この能力は、セキュリティーの観点から重要で、なぜなら、クラス生成は特権的な動作と考えられるからです。 【訳注: Rhino 1.5R4 以降は, 新しいセキュリティー・インターフェースに置き換えられた】

この問題の修正についての変更を (CVS リポジトリーに) チェックインしました。 Context を作成するときに SecuritySupport を指定するのなら、JavaAdapter の利用は、その SecuritySupport クラスに対しての 委譲クラスを生成するでしょう。

Context.exit()

Context.exit() は、インスタンス・メソッドから静的 (static) メソッドへと変更されました。 static であっても、それが Context.enter() メソッドとマッチするようにできるからです。 その操作についての詳細は、javadoc を参照してください。

Context.enter(Context)

新しく、Context.enter の多重定義 (オーバーロード) が追加されました。 このメソッドの追加がなければ、既存のコンテキストをスレッドに接続(attach) するのは不可能でした。 その操作についての詳細は、javadoc を参照してください。

Context のリスナー

Context は現在、そのいくつかのプロパティーのための、property change listener をサポートしています。
