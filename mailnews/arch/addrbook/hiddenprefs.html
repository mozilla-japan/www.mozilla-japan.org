"Get Map" ボタンの設定 ("mail.addr_book.mapit_url.format" )
この設定の書式は以下の通りです。

@A1 == アドレス 1 行目【訳注:日本では番地に相当】
@A2 == アドレス 2 行目
@CI == 都市名【訳注:日本では市町村名に相当】
@ST == 州名【訳注:日本では都道府県名に相当】
@ZI == ZIP コード【訳注:日本では郵便番号】
@CO == 国名

この設定が空文字列の場合、"Get Map" ボタンはアドレス帳のカード表示ペインに表示されません。

(mailnews.js で定義された)初期設定は以下の通りです。

pref("mail.addr_book.mapit_url.format", "http://www.mapquest.com/maps/map.adp?country=@CO&address=@A1%20@A2&city=@CI&state=@ST&zipcode=@ZI");

アドレス帳クイック・サーチ・クエリー設定 ("mail.addr_book.quicksearchquery.format" )
この設定の書式は以下の通りです。

@V == アドレス帳のクイック・サーチ・バーに打ち込まれた単語

設定値の中の @V は全て置き換えられます。

(注意) これは「名前や電子メールアドレスに含まれる単語」【訳注：Mozilla 1.7.3 日本語版】というユーザーインターフェイスを備えています。この実体 (SearchNameOrEmail.label) は messenger.dtd で定義されています。クイック・サーチ・クエリー設定で、電話番号での検索に替えたなら、このユーザーインターフェイスの表示とは一致しなくなります。

(mailnews.js で定義された)初期設定は以下の通りです。

pref("mail.addr_book.quicksearchquery.format","?(or(PrimaryEmail,c,@V)(DisplayName,c,@V)(FirstName,c,@V)(LastName,c,@V))");

"and" や "or" 、 "not" が使えます。
有効な語句のリストを得るには http://lxr.mozilla.org/mozilla/source/mailnews/addrbook/public/nsIAddrDatabase.idl を参照して下さい。
"c" は含まれる単語です。他の識別子は nsAbQueryStringToExpression::CreateBooleanConditionString() を参照して下さい。【訳注: 該当箇所のコード】
