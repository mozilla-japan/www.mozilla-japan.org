拡張機能のカスタムインストール

拡張機能のインストール と 拡張機能マニフェスト で説明したように、拡張機能はそれぞれにインストールマニフェストを用意する必要があります。拡張機能がこのマニフェストだけしか用意していなければ、拡張機能マネージャシステムはマニフェストが提供するデータを利用してその拡張機能をインストールします。この「シンプルモード」では XPI アーカイブ内の ファイル構造 が重要になります。

拡張機能の作者は、上級ユーザ向けに、今まで通りインストールスクリプト (install.js) を使ってインストール方法をより細かく設定できます。この場合でも、シンプルモードの拡張機能と同様のマニフェストを用意する必要がありますが、install.js が XPI ファイルのトップレベルにあれば、実際のインストールにはそちらが利用されます。

以下は高度なインストールスクリプトの一例です。

    // extensions/<myExtension>/uninstall.log にログエントリーを生成
    var err = initInstall("My Extension 1.0 for Firefox 1.0", "@bengoodger.com/software/magpie", "4.0");
    // これによって [bin またはプロファイル]/extensions/[拡張機能の ID]/ 以下に新規フォルダが作成されます
    ...

    if (verifyDiskSpace(getFolder("Profile Extension Chrome"), spaceRequired)) {
      addFile("My Extension Chrome",             // 登録名
              "chrome/myExtension.jar",          // (XPI 内の) ソースファイル
              getFolder(getFolder("Profile Extension Chrome")), // ターゲットフォルダ
              "");                               // ターゲットサブディレクトリ
      addFile("My Extension Component",          // 登録名
              "components/myExtension.dll",      // (XPI 内の) ソースファイル
              getFolder(getFolder("Profile Extension Components")), // ターゲットフォルダ
              "");                               // ターゲットサブディレクトリ
      addFile("My Extension Default Prefs",      // 登録名
              "chrome/myExtension.js",           // (JAR 内の) ソースファイル
              getFolder(getFolder("Profile Extension Default Preferences")), // ターゲットフォルダ
              "");                               // ターゲットサブディレクトリ

      var file = getFolder("Profile Extension Chrome", "myExtension.jar");
      registerChrome(PACKAGE | DELAYED_CHROME, file, "content/myExtension");
      registerChrome(LOCALE | DELAYED_CHROME, file, "locale/en-US/myExtension");
      registerChrome(SKIN | DELAYED_CHROME, file, "skin/classic/myExtension");

      // 後処理
      performInstall();
    }
    ...

"Profile Extension Chrome"、"Application Extension Components" などは、プロファイルまたはアプリケーションの拡張機能フォルダ内で適切なフォルダを決める、getFolder 関数の新しいパラメータです。

initInstall 関数の 2 番目のパラメータは拡張機能固有の ID で、3 番目はその拡張機能のバージョンです。この情報は、拡張機能マニフェスト内の chrome:id アークや chrome:version アークと一致していなければなりません。

これらのエントリーはアンインストールログの生成に利用されます。
