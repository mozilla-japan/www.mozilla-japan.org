Mozilla 1.3.1 リリースノート

このリリースノートでは、Mozilla 1.3.1 に関する、システム必要条件、インストール手順、既知の問題について記述しています。このリリースノートはフィードバックを受けて更新されますので、後日またチェックして新しい情報がないか確認してください。（訳注 : 最新の情報は 原文（英語） をご覧ください） ソースコードや、あなたが使っているプラットフォーム向けの最新版の Mozilla をダウンロードするには リリース ページをご覧ください。Mozilla の使い方や設定に関する詳しい情報は エンドユーザ資料 をご覧ください。（訳注 : もじら組のサイトにある はじめての Mozilla 1.0 も大変参考になる資料です）

訳注 : 下に書いてある通り、Mozilla 1.3 からの変更点は Mac OS X 版 XPI 機能の修復のみです。リリースノートもそれ以外特に変更はありません。「新機能」「追加事項」以降の全文については Mozilla 1.3 リリースノート [差分ドキュメント] を参照してください。

Mozilla 1.3.1 の新機能

Mozilla 1.3.1 は、Mac OS X ユーザのために XPI 機能を修復するためにリリースされました。XPI 機能を必要とする Mac ユーザで、Mozilla 1.4 ベータ版にアップグレードできない、あるいは Mozilla 1.4 のファイナルリリースを待っているという人は、ぜひ今回のリリースにアップグレードすることをお勧めします。

  • Mozilla メール に迷惑メール分類機能が追加されました。最初に少し「トレーニング」をしておくと、クライアントがスパムメッセージと一般のメッセージを識別し、それを隔離することができます。Mozilla の迷惑メール分類機能に関する詳細は、Mozilla スパムフィルタリング のページをご覧ください。
  • ニュースグループのフィルタが実装されました。
  • リッチテキスト編集管理機能である Mozilla Midas が、1.3.1 で初めて Mozilla に実装されました。詳しい情報は Midas のページ をご覧ください。
  • 画像自動リサイズは、画像を、実際の大きさ／ブラウザウィンドウに合わせた大きさを切り換えて表示できる機能です。この機能を試すには、大きな画像をブラウザウィンドウに読み込むか、ウィンドウを小さくします。それから、画像の上でクリックすると、自動サイズと実際のサイズを切り換えられます。この機能は、設定の［表示］パネルから無効（有効）にできます。
  • ユーザは「動的に」プロファイルを切り換えられるようになりました。この機能を試すには、［ツール（Tools）］メニューから［プロファイルを切り換える（Switch Profile）...］を選んでください。
  • Find As You Type - 以前は Type Ahead Find と呼ばれていました - のための新しい設定パネルが付きました。（［詳細（Advanced）］ - ［キーボードナビゲーション（Keyboard Navigation）］）
  • ChatZilla に専用の設定パネルが付きました。（ChatZilla をインストールした場合）
  • また、Mozilla 1.3.1 には、パフォーマンス、標準準拠、サイト互換性の修正も含まれています。
  • Mozilla に新しいスプラッシュスクリーンが付きました。私たちは既に知っていることなので、これに関して Bugzilla にバグを報告しないでください。
  • Red Hat 8.x 用 RPMS は GCC 3.2 でコンパイルされています。

（訳注 : Mozilla 1.2.1 以降のものも含む）

リリースノートへの追加事項

ここには、前回のマイルストーン以降このリリースノートに追加された事項が載っていますが、バグ自体は以前からあったものである可能性もあります。

  • Mac OS X では、XPInstall が 1.3.1 で無効になっています。これは、Mac OS X 版の Mozilla 1.3.1 に拡張機能やテーマをインストールできないためです。（bug 196959）
  • Mozilla の以前のバージョン向けにリリースされた mozdev のスペルチェッカーは、1.3.1 以上のビルドでは動作せず、クラッシュの原因となります。新しいスペルチェッカー にアップグレードしてください。（bug 185251）
  • MRJ Java プラグインは Mozilla 1.3.1 では利用できません。そのため LiveConnect が機能しません。Mozilla は、アプレットの表示に Apple の Java Applet プラグインを利用します（このプラグインを利用できる場合）。（bug 97613）
  • Mozilla 1.3.1 やトランクビルドでは、XBL ベースのフォームコントロールが壊れており、利用できません（bug 185612）。HTML ページで HTML フォームコントロールが見られず、機能しない場合は、「pref.js」ファイルの「user_pref("nglayout.debug.enable_xbl_forms", true);」を確認してください。注意 : これは、以前の Mozilla ビルドで、「デバッグ」パネルから設定を変更している場合のみ有効になっています。一般のユーザはこれらの設定を変更しない方がいいでしょう。
