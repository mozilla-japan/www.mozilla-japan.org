メモリ破壊の形跡があるクラッシュ (rv:1.8)
重要度
最高
公開日
2006/04/13
報告者
複数

Firefox
Thunderbird

Firefox 1.5

Thunderbird 1.0.8
SeaMonkey 1.0
Mozilla Suite 1.7.13

概要

Firefox 1.5 リリースの一環として、製品の安定性を向上させるため、いくつかのクラッシュバグを修正しました。これらのクラッシュの一部に、任意のコードを実行可能と思われるメモリ破壊の痕跡が見つかりました。この問題は、Firefox 1.0.x と Mozilla Suite 1.7.x にも影響します。

脆弱性ではない再帰誘発クラッシュの修正中に、CSS ボーダーの描画コードが、配列の末端を越えて書き込みを行う可能性のあることが Bernd Mielke 氏によって発見されました。

JavaScript の極端に大きい正規表現によって引き起こされるクラッシュが Alden D'Souza 氏によって報告されました。このクラッシュから、攻撃者が用意したデータを JavaScript バイトコードとしてブラウザが解釈してしまう潜在的な可能性を持った 16 ビット整数オーバーフローが特定されました。

表示スタイルの -moz-grid と -moz-grid-group を動的に変更する際に起きる、潜在的に悪用可能な 2 件のクラッシュが Martijn Wargers 氏によって発見されました。

MFSA 2005-58 で説明しているリグレッションの修正の一環によって Firefox 1.0.7 で紛れ込んでしまった、InstallTrigger.install() メソッドのメモリ破壊を引き起こすクラッシュが Bob Clary 氏によって発見されました。

注意: Thunderbird は Firefox と同じブラウザエンジンを利用しており、JavaScript が有効になっている場合には影響を受けます。デフォルト設定では無効になっており、メールを表示する際は JavaScript を実行させないよう強く推奨します。

回避策

修正版にアップグレードしてください。Thunderbird と Mozilla Suite のメールクライアントでは JavaScript を有効にしないでください。
