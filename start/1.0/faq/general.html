1.1. Mozilla 1.0 が対象にしているユーザーは？

Mozilla 1.0 は、Mozilla テクノロジーで何ができるかに興味を持っている人々と、 Mozilla をベースにした製品とパッケージを作成したい人々のための、完全に機能するテクノロジーのデモです。意図しているターゲットは、開発コミュニティの人々です。Mozilla は フリーなソフトウェア ですので、ライセンス を遵守する限り、どんな個人あるいは会社でも、自由に変更あるいは再頒布できます。

（mozilla.org からリリースされた） Mozilla 1.0 は、一般のユーザーが使用しても快適なように作成されていますが – そして、mozilla.org からリリースされた Mozilla を使用したい、という人を歓迎しますが – mozilla.org には エンドユーザーサポート を提供する機構がありません。しかし、 mozilla.org は新しい テスターとバグ・レポーター を常に求めています。

Mozilla 1.0 をベースにした 製品とパッケージ は、次の数週間のうちに出るものと期待されます。Mozilla テクノロジーの他のアプリケーションも現在開発中です。

1.2. Mozilla の新しいバージョンをインストールするには？

まず最初に、Mozilla 1.0 リリースノート であなたの OS について書かれている記載に従って、Mozilla をアンインストールした後に、新しいバージョンをインストールしてください。新しいバージョンの Mozilla を、古いバージョンに上書きしてはいけません。

もしあなたの plug-ins フォルダが、Mozilla application フォルダの中にあれば、再インストールの後でリストアできるように、事前に他の場所に保存しておけます。また、plug-ins フォルダは、あなたの プロファイル にも保存できます。ヘルプ｜Plug-in について （Help | About Plug-ins） を開いて、Mozilla が現在使用しているプラグインの場所を確認してください。

この方法で Mozilla をアンインストールすれば、あなたのプロファイルとユーザー設定は変更されずに残るでしょう。いくつかのアドインにはアクセスできず、再インストールの必要があるかもしれません。テーマとスペルチェッカーは （もしインストールされていれば）新しいバージョンのインストール時に検出される筈です。

Mozilla の新しいバージョンをインストールする時は、残された古いバージョンのプロファイルを使おうとせず、新規のプロファイルを作成するべきです。Mozilla の異なったバージョンでプロファイルを共有すると、プロファイルの破壊 を招く事があります。

もし、Mozilla の新規のインストールで問題が生じたら、トラブルシューティング を見てください。

1.5. 「隠し設定」があると聞いたのですが、どのように設定したら良いのでしょう？

編集｜設定 (MacOS X では： Mozilla | Preferences) にある設定と同様に、上級ユーザーは多数の Mozilla のパラメータを精緻にコントロールできます。最初に、 Customizing Mozilla、 Hidden Mail Compose Prefs そして Hidden Address Book Prefs を必ず読んでください。また、アドレスバーに、  about:config を入力すると、Mozilla で設定可能な変数のリストが表示されます。

隠し設定を設定するには、Mozilla を（高速起動も含めて）終了し、あなたの プロファイル・フォルダ に行き、問題が生じたときのために prefs.js ファイルをバックアップして、user.js ファイルを作成（または編集）します。次の例のように、行を追加してください。

user_pref("extensions.irc.nickname", "MyName")

Mozilla を再起動すれば変更が反映されているでしょう。

user.js は、ユーザーが作成するファイルで、あなたがカスタマイズした設定を、Mozilla が変更しないファイル (prefs.js では変更されます) に保存できます。これにより、あなたは自分が行った変更を確認できます。

隠し設定を操作するときは、充分注意してください。Mozilla が使えなくなるようなミスを犯したときは、user.js を削除して、バックアップから prefs.js をリストアしてください。（prefs.js がない状態で Mozilla を起動すると、全ての設定がデフォルトでリストアされ、あなたのメールとニュースのアカウント設定が失われます。POP メールのファイルは残存するので回復の役に立つでしょう）

Windows と MacOS 8/9 のための、設定を tweak する実験的アプリケーションが TweakMoz で入手できます。

1.6. Mozilla に、クラッシュ回復機構はありますか？

クラッシュ回復プロジェクトは、 http://recall.mozdev.org/ で開発中でしたが、この数か月間進行していません。しかし、それまでの成果は入手可能です。

1.7. Mozilla で、Netscape のように、AOL インスタント・メッセンジャーを直接使用できますか？

AIM は、AOL の独自の製品で、Netscape だけに統合されています。 スタンドアローン 版の AIM や、（もし Java プラグインをインストールしていれば） Java 版 は使用できます。

Mozilla のための、Jabber インスタント・メッセージ・クライアントも、 Jabberzilla で開発中です。

1.8. Mozilla のニュースや Mozilla を支持するサイトにはどのようなものがありますか？

Mozilla のニュースとリソースを含むサイトは：

  • MozillaZine – ナイトリー テストビルド の コメント もあります。（訳注：mozillaZine 日本語版）
  • MozillaNews
  • BlogZilla – ニュース と tips
  • Grayrest’s Mozilla News
  • MozillaZine developer weblogs

あなたのウェッブサイトに置くための、 Mozilla プロモーション・バナー のページもあります。

インクジェット・プリンターで、shirt2.gif または shirt3.gif を、T-シャツ転記紙に印刷して、自分の T-シャツを作成することもできます（結局、これはオープンソースプロジェクトです）。また、 MozillaZine シャツ を買うこともできます。
