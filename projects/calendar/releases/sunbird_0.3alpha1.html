Sunbird 0.3 alpha1 リリースノート

2005 年 11 月 4 日、Sunbird/Calendar プロジェクトは Sunbird 0.3 alpha1 をリリースしました。 長期にわたるバックエンドコードの書き換え作業が始まって以来、最初の正式リリースです。 Sunbird 0.3 alpha1 は Sunbird 0.3 に向けてのロードマップにおいて最初のマイルストーンになります。 このリリースノートでは、このバージョンにおいてユーザが期待すべきこと、期待すべきでないことを明らかにしていきます。

0.3 alpha1 リリースの目標

Sunbird 0.3 alpha1 は製品品質のリリースというものを意図したものではありません。 （"制限事項および既知のバグ" を参照。） その代わりに、新しいバックエンドコードの公開やテストのためのリリースという位置付けです。 これは以下のようなものです。

  • 繰り返し起きるイベントの計算の改善。
  • SQLite を組み込むことによる、ローカルカレンダーの保存機能の改善。 (http://www.sqlite.org/)
  • WebDAV によるリモートカレンダー操作機能の強化。 (http://www.webdav.org/)
  • インポート・エクスポート機能の改善。

Sunbird 0.3 alpha1 のリリースに際して、このバックエンドコードに潜むバグや問題をさらけ出すことができるテスタを広く募集しています。 より綿密なテストをしたいと思っている人は、ガイドとして Calendar:Tests page を参考にしてください。 他の目標は以下のようなものです。

  • 新しい UI 要素に関するユーザからのフィードバックの取得
    • 'New Calendar' ウィザード
    • インターネット接続・プロキシの設定
    • カテゴリの色分けに関する設定
  • 拡張機能・テーマを完全にサポートした Sunbird のリリース

制限事項および既知のバグ

Sunbird 0.3 alpha1 を使用する前に、このリリースに存在する以下の制限事項に目を通しておくことを強くお勧めします。 新しいバックエンドコードを使用しながらも Sunbird 0.2 と同様の機能を備えているものを提供することが目標でした。

  • Sunbird 0.3 alpha1 は Sunbird 0.2 で使用していた古いカレンダーファイルを自動的には移行しないので、プロファイルフォルダ から手動でインポートするか、リモートカレンダーファイルを再購読する必要がある
  • Windows 版の Sunbird 0.3 alpha1 では 2038 年以降および 1970 年より前を扱おうとするとクラッシュする
  • ドラッグアンドドロップ機能が 0.2 での機能に比べ大幅に制限されている
  • イベントダイアログでタスクからイベントへの切り替えに失敗する
  • イベントにファイルを添付できない
  • 印刷機能が非常に制限されている
  • 'Tasks in view' オプションが機能しない
  • アラームが 5 分しかスヌーズできない
  • Sunbird が閉じられている間に起動し損なったアラームがその後起動しない
  • Sunbird ではメールアラームが機能しない
  • 公開時のエラーの通知が貧弱である
  • タイムゾーンの設定が機能しない
  • カレンダーのチェックを外すと JavaScript エラーが生じる

WebDAV によるカレンダーバックアップシステム

繰り返すと、0.3 alpha1 にはデータ消失を引き起こす可能性があります。 この危険性をできる限り軽減するために、このリリースには WebDAV カレンダーによるデータバックアップシステムも組み込まれています。 新しい変更点を webdav カレンダーに反映させる前に、Sunbird は次のような名前のファイルを生成し現在の状態のバックアップコピーを行います。

calBackupData_|何らかの数字|_edit1.ics

その前のバックアップは

calBackupData_|何らかの数字|_edit2.ics

のようになります。 他に 3 つのバックアップコピーが 24 時間ごとに更新され、そのファイル名の最後には "day1.ics" などの文字列が付きます。 これらすべてのバックアップファイルはプロファイルフォルダ（http://kb.mozillazine.org/Profile#Sunbird を参照）内の "Backup Data" フォルダの中に作られます。 これらのファイルはカレンダーを直前の状態に戻すために直接アップロードすることができるはずです。 もしこのような必要に迫られた時には、標準のバグ報告手順に従ってその問題を開発者に知らせてください。

次のステップ

Sunbird/Calendar の開発プロセスにおける次のステップは、Lightning 向けに作られた新しい UI 要素の多くを実装し、バックエンドをさらに改良することです。 これは以下のようなものです。

  • インライン編集や完全なドラッグアンドドロップのサポート、視認性の向上がなされた新しいカレンダービュー
  • 新しい、よりユーザに優しい 'Create Event' ダイアログ
  • 公開・インポートダイアログなどの改善
  • CalDAV (http://www.caldav.org/) プロトコルのサポートの改善

これらの機能の実装具合を追いかけたいユーザはぜひナイトリービルドをダウンロードしてください。 いつもどおり、テストや役に立つバグ報告を大いに歓迎します。
