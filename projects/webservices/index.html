Web サービスとは何ですか？

Web サービスとは XML ベースのテクノロジで、アプリケーションが環境に依存することなくお互いにやり取りできるようにします。これは、Web インターフェース（XML）を通じて標準化されたフォーマット（SOAP および WSDL API）でメッセージをやり取りすることで成立します。

Mozilla は Web サービスをサポートしていますか？

「はい」であり「いいえ」です！　Mozilla は Web サービスをホストはしませんが、他のサービスとやり取りするために低次と高次の API （それぞれ SOAP と WSDL を指します）を提供しています。一般に利用できるサービスが多くあり、それらは XMethods Web サイトにリストアップされています。その中には Mozilla 実装をテストや事例として用いています。Mozilla SOAP API の使用についてより詳しく知るためには、SOAP Scripts in Mozilla ドキュメントを参照してください。SOAP の詳細を隠してサービスの呼び出しを単純化する、プロキシモデルを理解するためには、Accessing Web Services In Mozilla Using WSDL Proxying ドキュメントを参照してください。

Mozilla はどのようにして Web サービスを使いやすくしているのですか？

リソースがロードされたドメイン内のリソースのみにアクセスするようにスクリプトを制限することで、あるドメインから別のドメインへとロードされたあらゆるスクリプトは、ロードされたドメイン内のリソースにアクセスできなくなります。Mozilla は、クロスドメイン問題を克服する独特の方法を提供しています。それは、特殊な設定のためにセッティング／ハッキングで苦労することなく、スクリプトが分散されたサービスにアクセスできるようにしています。この新しいセキュリティモデルによって、信頼できないスクリプトや認証から、外部リソースへの正当なアクセスを要求する可能性のあるスクリプトに至るまで、内部リソースを区別し保護します。より深く Mozilla の Web サービスセキュリティを理解するためには、New Security Model ドキュメントを参照してください。
