M16 リリースノート(詳細)

システムの必要条件

Windows:
Windows 95/98、Windows NT、Windows 2000、Pentium 133 MHz、32 MB の RAM
Mac OS:
Mac OS 8.5/8.6/9 と PowerPC 200 MHz、PowerPC 604 または G3、 32 MB の RAM と仮想メモリ (または 48 MB の実メモリ)
QuickTimeLib ファイルをマシンにインストールしておかなければなりません。
Linux:
Red Hat Linux 6.1、Pentium II 233 MHz、 32 MB の RAM
もし SuSe Linux 6.2 で Netscape 6 を動かしているのであれば、ここから libjpeg.so.62(jpegsrc.v6b.tar.gz) をインストールしなければなりません。

新着情報

M16の開発サイクルは速く猛烈でした。 この 6週間で多くの新しい機能がチェックインされました。 この章は M15リリース以来の新しいことのうちのいくつかの簡潔な記述です。

Chatzillaは Mac, Linuxと Win32で起動し動きます。 あなたは、タスクメニュー上の IRC Chat アイテムの選択によりそれを使う事ができます。

スキン切り替えはこの開発サイクルで実装されました。それが何かを確かめるには http://www.mozillazine.org/chromezone/articles/article12.html を訪れてください。

セッションヒストリは最近入れられました。 URL バーの Forward/Back ボタンからヒストリーリストが出てきます。

Mozillaはオートコンプリートを持ちます。試してみるには URL バーへタイピングしてください

オンライン/オフライン スイッチ はステータスバーの左下の角に見る事ができます。

Mozilla Mail はメッセージフィルタを書くための UIを持ってます。 Mailの Edit メニューから見つける事ができます。(Message Filtersという名前です)

オートコンプリートは更にメールやニュースのアドレス入力でも実装されました。

News は購読設定ダイアログを持ってます。試してみるには Newsアカウントを設定した後に Fileメニューの Subscribeアイテムを。

Composer は HTMLソースをインラインプレビューで表示できます。

HTTP 1.1は完全に機能的で、デフォルトで動作します。同様にディスクキャッシュもデフォルトで動作します。

これらは直前のマイルストーン以降に Mozillaへ実装された新しい機能の全てではありません。 新しい機能は熱狂的なペースで取り込まれたので、安定したものではないかもしれません。（訳を後で見直す）

インストールする前に読んでください

AOL 4.0 かそれ以前のバージョン と Mozilla を同時に動かすことは問題になるかも知れません。 AOL 5.0 と Mozilla を動かす事による問題は報告されていません。

もし起動時にクラッシュしたならば、あなたの Mozilla のレジストリを削除してください。 Mozillaのレジストリーは M9から全てのプラットフォームで変わっています。

Mac OS:
システムフォルダー内の Preferencesフォルダーを開いて、 "Mozilla Registry" ファイルを削除してください。
Windows:
Windowsディレクトリーを開いて、"mozregistry.dat" を削除してください。
Linux:
~/.mozillaディレクトリーを開いて、"registry"ファイルを削除してください。

さらに詳しく知りたい方はここをご覧ください。 Win32にインストールしている場合には インストール手順にある起動時のクラッシュに関する追加情報を読んでください。

空のディレクトリーにインストールするか、あるいはインストールする以前に、前のビルドを削除してください。以前のマイルストーンや製品に上書きでインストールすると問題を引き起こすかもしれません。 ソフトウェアが開発中である間はマイルストーン間での不整合が生じるかもしれません。

ディレクトリーが書き込み可能であることを確かめてください。 Mozillaは、Mozillaを実行する人がインストールしたディレクトリーに対して書き込みパーミッションを行っていることを必要とします。

glibc 2.0を用いた (RedHat 5.2 のような) Linux システムでは、thread safeではない dl関数 (dlopen(), dlsym()) の問題があります。 これは M16ビルドの起動時にランダムにクラッシュすると思われます。 もしこの問題が発生したならば、アプリケーションを何度か起動し直してください。 ftp.mozilla.orgにポストされたものは RedHat 6.0(それはglibc 2.1を使用して、この問題を持っていません)でビルドされます。より詳しい情報は Bug 8849 を参照してください。

M16 リリース・ビルドをインストールする

インストールの手順 -- Unix

  1. "mozilla" という名のディレクトリーを作成し、その mozilla ディレクトリーに tar.gz ファイルを移動して下さい。

    mkdir mozilla
    mv mozilla*.tar.gz mozilla

  2. mozilla ディレクトリーに移動して、アーカイブファイルを展開(解凍)して下さい。すると "package"というディレクトリーが作成されます。

    3. cd mozilla
    gzip -dc mozilla-i686-pc-linux-gnu-M16.tar.gz | tar -xvf -

  3. "package" ディレクトリに移動して下さい。

    mv mozilla*.tar.gz ../
    cd package

  4. mozilla を起動スクリプトで起動して下さい。

    ./mozilla

重要: Seamonkey を Solarisで実行するには、Mozilla on Solaris FAQ を参照してください。 ここにはビルドと実行に必要な Solaris 2.6パッチのドキュメントがあります。

Unixでメールを使用する手順

既に使用している 4.xのプロファイルから移行することもできます。また、あたらしいプロファイルを作ってアカウントの設定（Account Setup）を使ってサーバー情報を設定することも出来ます。

インストールの手順 -- Win32

mozillaをインストールで推薦する方法は、インストーラー・プログラムを使用することです。

  1. mozilla-win32-installer-M16.exeを一回クリックしてください。そうすればファイルがデフォルトのフォルダーにダウンロードされます。
  2. mozilla-win32-isntaller-M16.exeをダブルクリックして、表示される操作指示にしたがいます。
  3. mozillaを実行するには スタート - プログラム - Mozilla Seamonkey - Mozilla をクリックしてください。
注釈:
移行していない 4.xのプロファイルも 5.0のプロファイルも既にあり、そして mozilla コマンドを実行しても Seamonkeyが起動しない場合は、mozilla -installerコマンドを実行してプロファイル・マネージャーを立ち上げ、プロファイルを選択してください。

Win32 でメールを使用する手順

既に使用している 4.xのプロファイルから移行することもできます。また、新しいプロファイルを作ってアカウントの設定（Account Setup）を使ってサーバー情報を設定することも出来ます。

インストールする手順 -- Mac OS

もし StuffIt Expander ™ がインストールされており、使用しているブラウザーがそれを使うように設定されている場合:

  1. mozilla-mac.sea.binのリンクを一回クリックしてください。ファイルはデフォルトのフォルダー（通常はデスクトップ）にダウンロードされ、StuffIt ™ が起動されて mozilla-mac.sea に変換されます。 それから StuffIt は mozilla-mac.sea を開いて mozilla-macフォルダーに変換してくれます。
     
  2. mozilla-mac フォルダーを開いて "mozilla." ファイルをダブルクリックしてください。 (注意: この段階で .bin と .sea ファイルは不要になります。）

注意: もし手順の一つ目でブラウザーから何をするか問い合わせられた場合は、StuffIt ™ の設定がされていないのが原因です。 その場合には保存(Save)をクリックして下記の 手順 3番 に進んでください。

もしも StuffIt Expander ™ がインストールされていないか、あるいはブラウザーが適切に設定されていない場合:

  1. StuffIt ™ がハードディスクのどこかにあることを確認してください。
     
  2. mozilla-mac.sea.bin のリンクをクリックしてください。
     
  3. "Save this link as...(リンクを名前をつけて保存)" を選択し...

    ファイルダイアログが現われて、ファイルをどこに保存するか選択することができます。(例えばデスクトップ)

    注意: Macのインストーラーのアクセスパスにはスラッシュ (/) を使わないでください。 スペースは使用しても構いませんが、その他の特別な文字は避けてください

  4. Save(保存)ボタンをクリックしてください。
  5. 新しく作られたアイコン "mozilla-mac.sea.bin" をダブルクリックしてください。
     
  6. StuffIt ™ が起動し、同時に "mozilla-mac.sea" ファイルと "mozilla-mac." という名前のフォルダが作成されます。
     
  7. mozilla-mac フォルダーを開いて "mozilla." ファイルをダブルクリックしてください。 (注意: この段階で .bin と .sea ファイルは不要になります。）

もしもインストーラを使用して "error 340" となったならば、再び始めからやりなおしてください。

Macintosh でメールを使用する手順

既に使用している 4.xのプロファイルから移行することもできます。また、あたらしいプロファイルを作ってアカウントの設定（Account Setup）を使ってサーバー情報を設定することも出来ます。

M15 リリース・ビルドをアンインストールする

Linux

  • Seamonkeyをインストールした場所の package/ディレクトリを削除します。
     
  • ~/.mozilla (それはプロファイル情報を含む) を削除します。

Windows

  • タスクバーのスタートメニューから 設定 | コントロールパネル | アプリケーションの追加と削除 と操作して Mozilla Seamonkey を削除してください。

Mac OS

  • ドキュメント(Documents)フォルダーの中の Mozillaフォルダーを削除します。
     
  • システム・フォルダーか、あるいは設定(Preferences)フォルダー内の "Mozilla Registry" ファイルを削除します。
     
  • Seamonkeyをインストールしたフォルダー(恐らく mozilla-mac-M16)を削除します。

一般(General Issues)

インターフェースのいくつかの項目(コントロール用の HTML と XP Toolkit widget を含む)は、マウスでコンテキストメニューを呼び出すようにクリック(右手のボタンを右クリック)されたとき、完全には反応しない。 (bug 30497)

(Windows only) モーダルウィンドウ(いくつかのプラットフォームでは、ユーザの入力をブロックする)は、いくつかの環境下では他のウィンドウに隠される。 もしトップモーストウィンドウ(訳注: 一番手前に表示されるウィンドウのこと)がキーボードかマウスに反応しない場合、その下に、モーダルウィンドウが隠れているかも知れない。モーダルウィンドウを手前に持ってきて、閉じて下さい。 (bug 28459)

(Linux only) ある状態では、スクロールバーはブラウザの内容をスクロールしない。解決するには: 新しいブラウザウィンドウを開く。新しいウィンドウのスクロールバーは機能するでしょう。    (bug 34033)

(Linux only) タスクメニュー内のウィンドウ名をクリックしても、コンポーネントアイコンをクリックしても、タスクバー内の "Go To Window" アイコンをクリックしても、ウィンドウを手前に持ってくることはできません。  (bugs 8002 and  26082.)

(Windows only) Ctrl-Tab を押しても、現在開いているウィンドウ内を循環することはできません。解決するには: タスクメニューを開き、望みのウィンドウを選んでください。 (bug 30864)

キーボードショートカット(Keyboard Shortcuts)

キーボードショートカットは完全には実装されていません。編集のためのいくつかのショートカット(例えば、テキストの切り取りやコピー)は、あるダイアログボックス中では動作しません。ナビゲーションのためのショートカットは機能しません。 メニューアイテムのための複数のキーボードショートカットは機能しません。  (bugs 31233,  22529, and 20298.)

(Linux only) キーボードニモニック (メニューアクセスキー) は一時的に無効にされています。 (bugs 22515 and 24413)

設定(Preference)

設定(Preference) パネルを巡回するために上下矢印キーを押しっぱなしにすると、クラッシュする知れません。 押しっぱなしにする代わりに、よりゆっくりクリックしてください。 (bug 29882)

設定(Preference)ウィンドウから抜けた後、mozilla はアクティブなアプリケーションでは無くなっています。 mozilla ウィンドウをクリックして、mozilla を手前に持ってきて下さい。  (bug  6058)

フォント設定ダイアログボックス: 12 pt より小さいサイズのフォントは選択できません。 (bug 30813).

アプリケーション設定ダイアログはまだ存在しません。 (bug 10958)

スタイルシートを無効にする方法はありません。 (bug 32372)

メニュー(Menus)

全て選択(Select All)はコンテキストメニュー内では機能しません。代わりに Edit メニュー内の全て選択(Select All) を使って下さい。 (bug 28508)

(Linux and Windows only) コンテキストメニューからコピー(Copy) を選択した場合、メニューが画面上に残ります。このメニューを取り除くには、その外側をクリックしてください。 (bug 28773)

マウスホイールスクロール(Mousewheel scrolling)

Unix: X サーバが適切に設定されている必要があります。XFree86 では、X86Config ファイルの Pointer セクションに、"ZAxisMapping 4 5" というような行を追加してください。 また、Protocol 行がお使いのマウス用に正しく設定されているか確認してください。

Windows: ロジテックマウスをお使いの場合、ロジテックの em_exec プログラムが動作していると Mozilla 内ではホイールでのスクロールは機能しません。 この問題の解決法(それはマウスドライバのアップグレードを伴うでしょう)は、バグ 20618 に文書化されています。

Mac OS: マウスホイールスクロール(USB マウス用) は Mac OS にはまだ実装されていません。

ブラウザ(Browser Issues)

(Mac OS): SSL を使用したウェブサイトにはアクセスできません。

一回にダウンロードできるファイルは一つだけです。現在複数の並行ダウンロードはサポートされていません。 (bug 33163)

(Mac OS only) アピアランスコントロールパネル内にある、Apple が提供する "なめらかな文字で表示する" オプションの代わりに SmoothType コントロールパネルを使う場合、Mozilla には SmoothType version 2.1.1 が必要です。Mozilla は SmoothType の古いバージョンと互換性がなく、いくつかのページは誤って描画されます。 最新情報を得るには、http://www.kaleidoscope.net/greg/smoothtype.html に行ってください。

(Linux only) ファイル選択ダイアログボックスでは、ファイル名を入力してそれからディレクトリを変更すると、そのファイル名は記憶されず、再入力しなければいけません。 (bug 3025)

"/" で終わる URL(つまり、その URL は、ファイル名部分を持たない) のウェブページを保存しようとすると、ファイル保存ダイアログはファイル名を含まず、手動でそれを入力しなければいけません。 (bug 24817)

Mozilla 動作中に色数を切り替えると、結果として画像が汚れるかも知れません。(bug 6061)

起こり得るクラッシュ(Possible Crashes)

ムービー再生中の Mozilla の終了中にクラッシュが起こるかも知れません。 (bug 33105)

(Mac OS) Quicktime ムービーはブラウザをクラッシュさせるかも知れません。 (bug 24718 )

My Sidebar

My Sidebar の Search タブはブラウザ内でのみ動作します。メールやエディタでそれを使おうとしても、検索は行われません。 (bug 32034.)

追加ダイアログを手前に持ってくるには、しばしば 2回のクリックが必要です。 (bug 41260)

ソースを表示(View Source)

ソースを表示(View Source) は機能しません。 (bug 41090)

M16 ではページソース表示ウィンドウにメニューが欠けていて、キーボードショートカットが機能しなくなっています。  (bug 26953)

フレームのソースを表示(View Frame Source) は機能しません。 (bug 32360)

ページ読込みの問題(Problems Loading Pages)

(Linux only) 256色モードでは、test10.html 文書を Mozilla に読込むとクラッシュします。 (bug 11090)

読込み問題(可能性があるのはクラッシュ又はフリーズ)は、しばしば Java アプレットを含むページで発生します。

非常に長いテキストがあるページのレイアウトは遅いです。 
(bug 29641) 

(Mac OS) Quicktime ムービーは中央に隠れてしまいます。 (bug 36046)

(Mac OS) mozilla を使って Shockwave をインストールする方法は存在しません。解決するには、Communicator 4.7 でインストールしてください。 (bug 35915)

ページを開く(Open Web Location) ダイアログは、新しい Navigator ウィンドウで URL を開くことを許しません。 (bug 42058)

ブックマークと履歴(Bookmarks and History)

ブックマークの正しくない振る舞いは、プロファイルの問題に関連しています。 プロファイルの問題 を参照してください。

ブックマークフォルダをそれ自身にコピーペースとしないで下さい。クラッシュします。 回復するには、bookmarks.html ファイルをハードディスクから取り除いて下さい。 (bug 29212)

履歴ツールから URL を開くことはできません。 (bug 37243)

フォーム(Forms)

戻る(Back)、進む(Forward)、もしくは再読込み(Reload) ボタンをクリックした際、フォームデータは Post し直されません。 ("Post" は、ユーザがページ内のフォームに入力した情報を、その情報が処理されるように、サーバに送信するのに使われる方法です。)

HTML

(このセクションで解説される HTML 要素や属性を含んだページは正常に表示されないかも知れません。)

ISINDEX 要素は機能しません。この要素は検索機能に使われます。 (bug 3326)

"vlink" 属性は機能していません。ページ上の行ったことのあるリンクは、ページ作成者が割り当てた色では表示されないかもしれません。 (bug 12493)

"char" 属性は機能していません。この属性は、いくつかの文字(通常 10進数の点)で指定される、要素配置に使われます。 (bug 2212)

COLGROUP 要素の "valign" 属性は機能していません。 (bug 29055)

Profiles

IMPORTANT: All the information regarding profiles is stored in the registry file. The registry file gets created even if you have used earlier versions of Mozilla (Apprunner). When you check out a new build, make sure you delete or rename the following items before you install, build, and run the new executable:
  • Windows mozregistry.dat ( c:\winnt or c:\windows or equivalent),
  • Mac OS Mozilla Registry (System Folder:Preferences),
  • Linux registry (~/.mozilla/)
Choosing a folder to store a profile does not work on any platform -- the default  folder is always used. Linux and Mac OS: you can select a custom folder but the  profile is still stored in the default folder   (bug 32808).  Windows: when you press the custom folder button, nothing happens and you  must save the profile in the default directory    (bug 34071).

Default profile locations for profile directories are <3 levels up from the exe location>\Users50 on Windows, :Documents:Mozilla:Users50: on Mac OS, and ~/.mozilla on Linux.

 

To create a new profile name with a space in it, enclose the name in quotation marks (for instance, "My Profile").

You open Profile Manager with the command line argument : -ProfileManager. The Profile Manager user interface appears, supporting creating a new profile, and deleting and renaming existing profiles. When you finish any of the tasks you can click Exit to quit the Profile Manager. You can click a Start Communicator button to start Mozilla with a selected profile. Also, users can migrate 4.5 Communicator profiles.

Various command line options to create and manage profiles are:

# mozilla -P <Profilename>
Runs Mozilla with the <Profilename>.
# mozilla -nosplash -CreateProfile "<Profilename> <ProfileDir>"
Creates a profile (i.e., <Profilename>) and creates a directory at the specified location i.e., <Profiledir> and quits the application.
# mozilla -nosplash -CreateProfile <Profilename>
Creates a profile and creates a profile directory under default profile location.
# mozilla -ProfileWizard
Launches the Create Profile Wizard to create a new profile. The Create Profile Wizard lets users create a new profile at the default location or the location of their choice. At the end of the process a profile is created and marked as the current profile, and Mozilla is launched with that profile. The wizard is currently just two screens. Mail and News related preferences/information will be collected by a different application.
# mozilla -ProfileManager
Brings up Profile Manager. Users can select a profile and click on the Start Communicator button to launch Mozilla with the chosen profile. The current profile (usually the last used profile) appears preselected.
# mozilla -SelectProfile
Brings up Profile Selector. Users can select a profile from those displayed or click "manage profiles" to reach the Profile Manager and proceed as above.
# mozilla
Several scenarios:
 number of profiles = 0 --> Brings up Create Profile Wizard
 number of profiles = 1 --> Launches Mozilla directly with that profile
 number of profiles > 1 --> Brings up Profile Manager

To migrate 4.5 profiles, run Mozilla with the command line option -installer. Users can select a profile to be migrated and click on the Start Mozilla button. If the migration is done successfully, the profile is no longer marked with an icon in the display, indicating that profile as a 5.0 profile. Currently, only the prefs.js file and mail and news folders are copied in the migration process. Users cannot rename or start Communicator with a profile that must be migrated. 

Rename profile fails. (bug 39423)

(Mac OS 9.0) If you delete the folder 'Documents:Mozilla', which contains the user profiles, and restart mozilla, then user preferences, history, and other profile information will continue to be updated while this folder is in the Trash (bug 40180). If you choose to throw this folder in the Trash, then you should empty the Trash before restarting mozilla.

The terms "template" and "stationery" are used interchangeably in menus and dialog boxes.

To save as message as a draft or a template, your Copies and Folders preference must point to folders that exist. To set your Copies and Folder preference:

  1. Open the Edit menu and choose Mail/News Account Settings.
  2. Click Copies and Folders for the account you wish to set.
  3. Click "Drafts on [mail account/server]" or "Templates on [mail account/server]" and choose a folder. (The options contain your own mail account and server name.)
(bug 25146)

Passwords

If the Password Manager is turned off, passwords are not remembered. To turn Password Manager on:
  1. Open the Edit menu and choose Preferences.
  2. Click the Advanced category to open it.
  3. Click Forms and Passwords.
  4. Put a checkmark next to Enable Password Manager.
(bug 27994)

The "Save Password" check box in the Account Setup won't save a password. To save a password in the Master Password database, check the "Save password" box in the server password dialog that you see when you access a server. You will then see either a dialog box asking you to enter your master password, or a dialog box that lets you create a new Master password.

To remove a mail server from the Master password list, do one of the following: 

  • Choose Preferences from the Edit menu, then click the Advanced: Password Manager preference. Click the View Stored Passwords and then select the server and click Remove. 

    •  
  • From the Tasks menu choose Autofill: Display Signons, click View Stored Passwords, then select the server and click Remove. 
Sometimes the password box hides behind other windows. When this happens no action is possible until you enter your password and dismiss the password box. Move windows until you see the password box. (bug 28459)

Fonts Issues

If the fonts are too small in your environment, it may be possible to alleviate  this by setting a preference in Mozilla. Under the Edit menu select the Preferences option. In the Preference dialog selects the Fonts item and change the dpi entry to 120.

There is some info about setting the X server's DPI value at http://www.mozilla.org/unix/dpi.html.

If your fonts are still too small (or too large) after setting the DPI correctly, you can adjust the font size as follows:
setenv GECKO_FONT_SIZE_FACTOR 1.5

Note that the latter only affects fonts. The screen_resolution preference affects anything that specifies absolute size (e.g. CSS "pt" points).
