JavaScript 2.0
互換性

03/14/2003 (Fri)

JavaScript 2.0 では JavaScript 1.5 以前のスクリプトとのほぼ完全な上位互換を考えている。互換性に関する話題は現在のところ以下の通りである:

  • カンマが各括弧中で特別な意味を持つようになった。よって expr[expr, expr] は expr[(expr, expr)] と書き直さなければならない。
  • for-in 中の変数の初期化が認められなくなった。つまり、文 for (var a=b in c) ... の =b は認められない。
  • as 、is 、namespace 、及び use は予約語になったので、識別子として使う場合は \_ でエスケープする必要がある。
  • ラッパクラス Boolean 、Number 、及び String は JavaScript 2.0 で除外された。Boolean 、Number 、及び String はプリミティブな論理値、数値、文字列をインスタンスとして保持するクラスを指すようになった。これらの新しいクラスのメソッドに相当するのは JavaScript 1.5 のラッパクラスのメソッドである。Boolean 、Number 、及び String に対する new の呼び出しは実装依存であり、実装は JavaScript 1.5 との互換性のためにラッパクラスを保持してもよい。が、そうしなければならないわけではない。
  • JavaScript 2.0 は実装が配列について4294967296以上の添字をとることを認めている (そうしなければならないというわけではない)。4294967296以上の添字は配列の添字としては正しくないと信じ込んでいるコードは動かないかもしれない。
  • Object や String といった JavaScript 1.5 標準オブジェクトを変更しているコードは動かないかもしれない。
  • JavaScript 2.0 ではグロバール定数の定義が追加されるため、プログラムの最上位識別子と衝突する可能性がある。
  • 正しくない JavaScript 1.5 コードが JavaScript 2.0 のコードとしては正しいと解析される可能性がある。そのような形式的に正しくないコードに対して例外が出ることを期待している JavaScript 1.5 プログラムは動かないかもしれない。

ウェブブラウザなどのように 100% の互換性を要するアプリケーションのために、HTML 、XML 属性で、或いは use ecmascript(4) か use javascript(2) 文により明示的に JavaScript 2.0 であると示されていない限り、スクリプトは JavaScript 1.5 であるとみなされる。
