拡張機能のアンインストール

フロントエンドからの削除

アンインストールは、ユーザが管理 UI から拡張機能の削除を選択し、アプリケーション (Firefox) を再起動することで行われます。

  1. ユーザが管理 UI から拡張機能の削除を選択します。
  2. 拡張機能マネージャのデータソースで、その拡張機能に対して chrome:toBeUninstalled プロパティが設定されます。
  3. プロファイルディレクトリに、プロファイルのコンポーネントレジストリと並んで .autoreg ファイルが作成されます。その拡張機能コンポーネントへの相対パスがこのファイルに書き込まれます。

アプリケーションが再起動して拡張機能マネージャが初期化されると、以下の処理が行われます。

  1. XPCOM が初期化され、.autoreg ファイルがあることでコンポーネントの再登録が行われます (拡張機能はそれほど頻繁にインストール・アンインストールされるものではないので、ここでこの処理を行っても特に問題はないでしょう)。このとき .autoreg ファイルに書かれたパスは特別にスキップされます。
  2. GetSources メソッドを使って toBeUninstalled プロパティがセットされているリソースを特定します。(ここで最適化を行う可能性があります - 起動時に毎回複雑な RDF 操作が行われるのを防ぐため、このようにデータソースを操作する必要があるときは、必ずプロファイルディレクトリにファイルを作成します)
  3. その拡張機能をアンインストールするために、拡張機能マネージャは chrome:file アークを使って、その拡張機能のファイルに関連付けられたパッケージに対するクロームの登録を取り消します。
  4. uninstall.log ファイルがその拡張機能のルート階層に置かれて解析されます。これによってファイルがアンインストールされます。
  5. uninstall.log が解析されると、そのファイル自体が削除され、それが含まれていたディレクトリも削除されます。
  6. アプリケーションの起動が続行されます。

拡張機能の無効化

拡張機能の無効化は、その拡張機能を使えなくしてしまうので、アンインストールと似ていますが、ファイルは削除されず、拡張機能マネージャのデータソースにも残り、クロームパッケージは登録されたままになります。

フロントエンドで拡張機能の無効化が選択されると、以下の処理が行われます。

  1. 拡張機能マネージャのデータソースで、その拡張機能に対して chrome:toBeDisabled プロパティが設定されます。
  2. プロファイルディレクトリに、プロファイルのコンポーネントレジストリと並んで .autoreg ファイルが作成されます。その拡張機能コンポーネントへの相対パスがこのファイルに書き込まれます。

アプリケーションが再起動して拡張機能マネージャが初期化されると、以下の処理が行われます。

  1. XPCOM が初期化され、.autoreg ファイルがあることでコンポーネントの再登録が行われます (拡張機能はそれほど頻繁にインストール・アンインストールされるものではないので、ここでこの処理を行っても特に問題はないでしょう)。このとき .autoreg ファイルに書かれたパスは特別にスキップされます。
  2. GetSources メソッドを使って toBeDisabled プロパティがセットされているリソースを特定します。(ここで最適化を行う可能性があります - 起動時に毎回複雑な RDF 操作が行われるのを防ぐため、このようにデータソースを操作する必要があるときは、必ずプロファイルディレクトリにファイルを作成します)
  3. その拡張機能を無効化するために、拡張機能マネージャは chrome:file アークを使ってクロームレジストリの setAllowOverlaysForPackage メソッドを呼び出し、ダイナミックオーバーレイを無効化します。
  4. デフォルト設定は読み込まれません。
  5. アプリケーションの起動が続行されます。

何か問題がありますか?

新しいバージョンのアプリケーションがインストールされると、自動アップデート の間、拡張機能は無効化されます。コンポーネントの登録取り消しを実行するために、アプリケーションを再起動する必要があります...
