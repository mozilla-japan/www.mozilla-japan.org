Bezilla: ビルドの方法

お知らせ
私たちはあなたの援助を必要としています。時間を割いていただけるのであれば宜しくお願いします。 まずは mozilla.org のサイトを読むことから始めてください。 "Get Involved" や "Hacking" は必読です。
 
必要なもの
開発版の Bezilla をビルドするために必要なハードウェアとソフトウェアがあります。 実際にソフトウェアを実行する必要条件はもっと少ないです。 しかし、利用可能な安定版のリリースが現在は存在しないので、 私たちが開発している環境をお知らせします。

ハードウェア

  • BeOS が動作するコンピュータやハードウェア
    （訳注：現在は上記リンク先に情報はありません。日本語の情報はこちらで入手可能です。）
  • BeOS R5 Personal Edition （開発ツールがインストールされていること） または 同 Pro Edition
    [ いずれも x86版 のみ。 PPC 版はボランティアを募集中です。 ]
    （訳注：現在は上記リンク先に情報はありません。 Personal Edition および開発ツールはこちらより入手可能です。 Pro Edition は販売終了して流通在庫のみとなっています。）
  • 大量の RAM が必要です！最低 128MB の仮想メモリおよび最低 128MB 以上の物理メモリも！
  • 1.5GB または 2.0GB の空きディスクスペースが全ソースのコンパイルとデバッグに必要です。 ソースコードだけで 380MB もあります！
  • ソースコードを同期させるためのBeOSから接続できる良好なネット環境。 ケーブル接続で CVS を経由してのフルダウンロードにはおよそ 60 分かかります。

ソフトウェア

  • CVS

      cvs [ 548 kB ] をダウンロードして、ディスクの任意の場所に展開してください。 home/config/bin ディレクトリに CVS のバイナリのリンクを作成します。 インストールが成功したかをテストします。 "Terminal" を起動して、home ディレクトリから cvs --version とタイプしてください。 成功していれば以下のような結果になるでしょう。

        Concurrent Versions System (CVS) 1.11 (client/server)

Copyright (c) 1989-2000 Brian Berliner, david d `zoo' zuhn,
                        Jeff Polk, and other authors

CVS may be copied only under the terms of the GNU General Public License,
a copy of which can be found with the CVS distribution kit.

Specify the --help option for further information about CVS
  • Perl

      perl [ 2.27 MB ] をダウンロードして、ディスクの任意の場所に展開してください。 "Terminal" を起動して、展開したディレクトリへ cd して、 install-perl を実行します。 インストールが完了したら、 home/config/bin ディレクトリへ Perl のバイナリのリンクを作成します。 インストールが成功したかをテストします。 "Terminal" を起動して、 home ディレクトリから perl --version とタイプしてください。 成功していれば以下のような結果になるでしょう。

        This is perl, version 5.005_03 built for BePC-beos

Copyright 1987-1999, Larry Wall
BeOS port Copyright Tom Spindler, 1997-1998

Perl may be copied only under the terms of either the Artistic License or the
GNU General Public License, which may be found in the Perl 5.0 source kit.

Complete documentation for Perl, including FAQ lists, should be found on
this system using `man perl' or `perldoc perl'.  If you have access to the
Internet, point your browser at http://www.perl.com/, the Perl Home Page.
  • glib & libIDL

      glib & libIDL [ 837.8 kB ] をダウンロードして、ディスクの任意の場所に展開してください。 "Terminal" を起動して、展開したディレクトリの glib-libIDL-beos へ cd して、 以下のようにタイプします。 

        buildscript

      ライブラリ群の作成が始まり、成功したら以下のような結果になります。

        **********************************************
The support libraries have been built and installed, please set these environment
vabiables before running the main Mozilla 'configure' script:
export GLIB_CONFIG=/boot/home/glib-libIDL-beos/glib/bin/glib-config
export LIBIDL_CONFIG=/boot/home/glib-libIDL-beos/libIDL/bin/libIDL-config
**********************************************

      "/boot/home/glib-libIDL-beos/glib/bin/" の部分は、あなたのマシンでは違う表示になることがあります。

  • ~/.mozconfig

      Bezilla はさまざまなプラットホームでコンパイルされるため、 "configure" というツールを使います。 "configure" はプラットホームごとに異なる環境に応じてコードをコンパイルできるようにするためのもので、テンプレートに基づいて makefile を作成し、コンパイルするプラットホームに適合する大量の DEFINE 文を作ります。

      Bezilla のビルドシステムを適切に設定するために、ホームディレクトリ内に .mozconfig という新しいファイルを作成する必要があります。このファイルの内容は次の内容であるべきです

        # sh
# Build configuration script
#
# See http://www.mozilla.org/build/unix.html for build instructions.
#
# Options for 'configure' (same as command-line options).
ac_add_options --enable-cpp-rtti
# Remove the # in the next line for building a Mozilla without debug info.
#ac_add_options --disable-debug

      これは、ビルドシステムに RTTI ( Be のライブラリの1つによって要求される ) 機能を有効にするものです。 この作業を行わなければ mozilla/gfx/beos セクションをコンパイルすることができません。

  • ~/setupbezilla

      ビルドプロセスを単純化するために、あなたのホームディレクトリ内に setupbezilla と呼ばれるファイルを作成する必要があります。このファイルを実行することで、ビルドシステムが要求する glib や libIDL のようなライブラリの場所や、 CVS へ Bezilla のソース・コードを保持しているサーバーを見つけるためにどこを探せばよいかを伝えるための環境変数が設定されます。 テキストエディターでこのファイルを作成してこのテキストをコピー＆ペーストしてください

        export GLIB_CONFIG=/boot/home/glib-libIDL-beos/glib/bin/glib-config
export LIBIDL_CONFIG=/boot/home/glib-libIDL-beos/libIDL/bin/libIDL-config
export CVSROOT=:pserver:anonymous@cvs-mirror.mozilla.org:/cvsroot

      最初の 2 行を glib & libIDL のインストーラが出力したものと同じになる必要があります。 (実際、 /boot/home/glib-libIDL-beos の部分は、あなたの環境では異なっている場合が多いです。） ホームディレクトリ以下に setupbezilla というファイル名で保存してください。

 
Bezillaのビルド
Bezilla をビルドするときは毎回、 "Terminal" を起動し、home ディレクトリから "setupbezilla" スクリプトを実行します。 ソフトウェアの要求に適切に従った場合は以下のようにします。 （ "〜" は /boot/home と等価です。あなたが setupbezilla ファイルを他のどこかに保存したならば、それを実際のパスにする必要があります。最初のドットはシェルのバックグラウンドと環境変数を交換するために重要です。） 
    . ~/setupbezilla

今や環境変数は glib や libIDL ライブラリ群や Bezilla のソースコードを持っているサーバがどこにあるかを知っています。ソース・コードを得るために、私たちはサーバーにログインする必要があります。以下のようにタイプします。 

    cvs login

パスワードを要求されたら、 anonymous と入力します。 適切に行えば、あなたは Terminal のプロンプトに戻るでしょう。 その後、以下のコマンドを入力することでコード・サーバーから主要なメイクファイルを取り出せます。 

    cvs -z3 checkout mozilla/client.mk

"mozilla" ディレクトリを作ります。（ただし同名のファイルやディレクトリが既にある場合や読み出せない場合はエラーになります。） "mozilla" ディレクトリ内にはソースコードと "client.mk" というメイクファイルがあります。 メイクファイルはソースコードのダウンロード方法の指示を含み、 ビルドシステムの設定し、コードのコンパイルを行います。 あなたはそれらの作業を行って欲しいときにただ "client.mk" を呼び出せばよいのです。 最初にビルドする場合、全てのソースを取り出して、ビルドシステムを設定してコンパイルします。 あなたにとって幸運なことに、それはまさに client.mk のデフォルト動作です。 引数を渡さずに実行すると、ソースをダウンロードしてビルドシステムを設定して Bezilla をコンパイルします。それではやってみましょう。 

    cd mozilla
make -f client.mk

CVS 経由でハードディスクにダウンロードされたソースツリーを見てください。 configure スクリプトが終了したら、最後に自動的にコンパイルが始まります。 実行される場合、編集は始まるでしょう。 256MB の RAM を持つ Pentium 500 MHz のマシンで 2 時間近くかかるので、リラックスして忍耐強く待ちましょう。

 
Bezillaの実行
実行するためには mozilla/dist/bin へ cd して、以下のようにタイプします。 
    ./run-mozilla.sh mozilla
間違いなく “Terminal” から始めるべきです、多くのデバッグ情報がウィンドウ内に表示さわれるでしょう。さあ仕事を始める時間です。デバッグするか支援を求めてください。 私たちに連絡する方法についてのより多くの情報を得るためには contact us セクションを見てください。
