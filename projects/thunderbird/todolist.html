To Do リスト

メインの Thunderbird プロジェクトページで述べたとおり、私たちの最初の目標は Mozilla 1.4 release に基づいた0.1リリースに向かって突き進むことです。我々の短期的な目標は、長期的目標を現実のものとすべく体制を整えることです。To Do リストには、私たちが最初のテスト用ビルドを公開した際に皆さんが特定した、Thunderbird におけるバグや諸問題も含まれています。 これらのタスクがまだ完了していないことを確認するために、最近の変更リストも確認してください。

短期的 To Do リスト

以下は必ずしも重要度順に並んでいるとは限りません。

  • メール独自のコンテキストメニューを再実装します。特に、メッセージ表示部の、ブラウザ向けのコンテキストメニューアイテムが山ほど出てきたり隠れたりする、contentAreaContextOverlay.xul と nsContextMenu.js を使うコンテキストメニューをです。この二つのファイルを削除して、メール用にカスタマイズされたバージョンに置き換えます。
  • アドレス帳の列とアドレス帳の選択用の列のためのデフォルト値を localstore.rdf に、追加します。
  • mail/extensions の中に Thunderbird 拡張をビルドするためのビルド設定オプションが必要です。例えば "--enable-mail-extensions=offline smime"などです。
  • Phoenix がデフォルトのメールソフトとして Thunderbird を起動することを確実にするために、ウィンドウズのレジストリキー、 HKEY_CLASSES_ROOT\mailto に書き込む必要があります。(Bug #202497)
  • 私たちは nsIExternalProtocolService::loadUrl を Linux 用に実装するために、Linux の達人を必要としています。この助けが得られないと、私たちが望むようにディフォルトブラウザで http url（や他の非メール url）をロードすることができません。このルーチンを Linux 上で実装するための OS integration hooks が何か、私は知りません。もしかしたら使っているデスクトップ環境に依存するのでしょうか？(ひょっとしたらこの件について、Bryner が gnome に関することで何か知っているかもしれません。)
  • mail chrome から、contextHelp.js への参照を全て削除する。Thunderbird でこのファイルを見つけられないというエラーがあるため、（ontextHelp.js への参照を全て削除することで）パフォーマンスが劇的に改善する可能性があります。このファイルをロードする chrome を全て削除することで、私たちの作業を加速することができるかもしれません。
  • Linux 上での、添付ファイルやメッセージを保存する機能に関する問題のレポート。
  • テキスト版のファイルをメモ帳(あるいはデフォルトのテキストビューワ)で開く View Source を作る。
  • Thunderbird について ダイアログを Help メニューに追加します。(Jason)
  • Unix 版の Thunderbird を作成してくれるボランティアを探します。
  • mach-o 版の Thunderbird を作成してくれるボランティアを探します。
  • threadPane.xul や folderPane.xul のような、chrome 内の主要な XUL ファイルを検査し、fakeAccount.css のような、不要な CSS ファイルを除去します。
  • classic テーマを置き換える新しいテーマを調査します。0.1リリースにおいては、これはただ単に、何か新しいツールバーアイコンを入手し、ブランチのclassic テーマにダンプするということを意味するだけかもしれません。(Arvind がこの作業をする予定です。)
  • Help Viewer を持たないであろうことから、Help ボタンを取り除くか、隠すかします。
  • Junk Mail Control についての説明をする情報ダイアログを追加し、ユーザーが初めてその機能を使おうとしたときに表示されるようにします。これは、Phoenix における popup blocking dialog のように動作します。
  • ブラウザchrome ではなくメールを示す、新しい example テーマ アイコンが必要です。
  • ウィンドウズ用のデスクトップアイコンファイル (.ico)が必要です。現在は phoenix/mozilla の .ico ファイルを使っています。(Jason Kersey が作業中です。)
  • 単体のメールクライアントに必要なくて、ブラウザが必要とするだろうモジュール(gopherやftpなど)をビルドするのを止める作業を続けます。
  • プラグインと、JAVA のコードをビルドから取り除きます。(mail/config コードを通してなされました。)
  • ウィンドウズ用のデスクトップアイコンファイル (.ico)が必要です。現在は phoenix/mozilla の .ico ファイルを使っています。(Jason Kersey が作業中です。)
  • Mozilla プロファイルがすでにマシン上にインストールされていない状態での、Thunderbird の動作不良に関するレポート。これは私たちが Thunderbird プロファイルに移行したら問題ではなくなるかもしれません。(mscott が現在これを修正中です。) (Bug #201891)
  • 作業が全く進行していない際にプログレスバーが隠されていません。
  • Preferences を Options に改名して、Tools 以下にメニューを移動します。
  • カスタマイズ可能なツールバー。(Mozilla/xpfe toolkit への逆移植。)
  • classic をデフォルトのテーマにする。
  • Tools / Import が mail import dialog を表示しません。
  • 現在の設定 UI を検査し、Advanced や System Panel などのスタンドアローンのメールソフトでは意味をなさないアイテムを削除します。
  • フォルダ表示部の列のヘッダが、実際はソートできないのに、ソートできそうに思わせる表示になっています。このツリーにソートインジケータを表示しないようにする方法が何かあるはずです。
  • Mach-o と Unix 用のビルドを発表する。
  • Mozilla 1.4 trunk における修正の中で、Thunderbird の0.1ブランチに逆移植したいものはありますか？おそらくジャンクメールに関する改良がそれにあたるでしょう。
  • メール スタートページと throbberのurl がThunderbird プロジェクトページを指すようにする。

長期的 To Do リスト
  • より良いパフォーマンスおよびフットプリントを目指してメールニュースを静的アトムに変換します。これは Thunderbird や mozilla にも利益をもたらします。Bug #201561を見てください。
  • 必要とされる Thunderbird ライブラリ群と chrome を一まとめにするための mail/config ディレクトリを Thunderbird に追加します。
  • フットプリントとパフォーマンスの向上のため、全てのメール用DLLを一つのDLLにまとめます。Bug #179533を見てください。.
  • 私たちのブランチから離れ、 trunk に作業の場を移します。これは、trunk 上での衝撃を最小化するために、Thunderbird の作業の場を mozilla/mail に移すことを意味します。ビルド設定の変更を Mozilla の trunk に移植します。それに、私たちが手を加えた XUL/JS ファイルを枝分かれさせます。 目標は、trunk に移る際の衝撃を最小化し、新しい Mozilla/mail ディレクトリに可能な限りのものを持たせることです。
  • trunk との同期が済んだら、私たちは作業の場を 新しいツールキット に移行し、Phoenixの UI の流儀に従います。
  • 新しいテーマをどのようにしてダウンロードすることになるのか？ ブラウザがなければ、私たちはテーマのリストを表示させるブラウザの chrome を一切持っていないことになります。どのようにユーザーを Thunderbird のテーマページに連れて行き、彼らに Thunderbird 用の新しいテーマをダウンロードさせるのですか？
  • パフォーマンス向上のために、オーバーレイをなくします。
  • mozilla/editor と mozilla/xpfe の chrome にまだ依存しているものを取り除きます。
  • Phoenixで使われている、拡張の概念を開発します。smime、オフライン機能および受信確認のような機能をダウンロード可能な拡張機能に移行します。
  • Thunderbird ビルドから、不必要な chrome やモジュールを削除し続けます。
  • GRE を活用したインストーラを組み立てます。
  • blake によってデザインされた新しいメール作成 UI に移行します(Bug #146259)。
  • Mozilla GRE 問題を解決します。そうすれば私たちはより快適に利用できるし、GREを活用することもできます。これは、私たちがインストーラを必要としていることを意味するのですか？Phoenix の皆さん と一緒に作業してください。そうすれば双方が GRE を利用でき、双方のアプリケーションのダウンロードサイズを小さくすることができます。
  • Phoenix の開発者と、Send Page 機能を Phoenix に追加するために作業します。
