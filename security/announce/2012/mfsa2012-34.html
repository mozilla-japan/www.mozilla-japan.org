Mozilla Foundation セキュリティアドバイザリ 2012-34

タイトル: 様々なメモリ安全性の問題 (rv:13.0/ rv:10.0.5)
重要度: 最高
公開日: 2012/06/05
報告者:
影響を受ける製品: Firefox、Thunderbird、SeaMonkey

修正済みのバージョン: Firefox 13.0
  Firefox ESR 10.0.5
  Thunderbird 13.0
  Thunderbird ESR 10.0.5
  SeaMonkey 2.10

概要

Firefox やその他の Mozilla ベースの製品に搭載されているブラウザエンジンに含まれていたメモリ安全性に関するいくつかのバグを、Mozilla の開発者が特定、修正しました。これらの問題の一部は、特定状況下でメモリ破壊の形跡が見られ、条件が整えば任意のコード実行に悪用可能と思われるものでした。

Thunderbird と SeaMonkey では JavaScript が初期設定で無効になっているため、一般的にこれらの脆弱性はメールを通じた悪用が不可能です。ただしこれらの製品でも、ブラウザもしくはブラウザに類似した機能を利用した場合、潜在的なリスクが生じます。

参考資料

Firefox 12 に影響するメモリ安全性の問題とクラッシュが、Jesse Ruderman、Igor Bukanov 氏、Bill McCloskey 氏、Christian Holler、Andrew McCreight、Brian Bondy 氏によって報告されました。

  • Firefox 13 で修正されたメモリ安全性の問題
  • CVE-2012-1938

Firefox ESR と Firefox 12 に影響するメモリ安全性の問題とクラッシュが Christian Holler によって報告されました。

  • Bug 748613 – Assertion failure: [infer failure] Missing type pushed 0: float, at jsinfer.cpp:348
  • CVE-2012-1939

Firefox ESR と Firefox 12 に影響するメモリ安全性の問題とクラッシュが、Igor Bukanov 氏、Olli Pettay、Boris Zbarsky、Jesse Ruderman によって報告されました。

  • Firefox 13 と Firefox ESR 10.0.5 で修正されたメモリ安全性の問題
  • CVE-2012-1937

Chromium WebGL の実装において回避せざるを得なかった NVIDIA グラフィックスドライバのバグが、Google の Ken Russell 氏によって報告されました。Mozilla も Firefox 13 と Firefox ESR 10.0.5 で同様の措置を取りました。

  • Bug 744888 – Work around NVIDIA driver bug in glBufferData
  • CVE-2011-3101
