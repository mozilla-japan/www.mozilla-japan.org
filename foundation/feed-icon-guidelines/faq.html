フィードアイコン使用ガイドラインに関する FAQ (よくある質問)

この FAQ は、フィードアイコン使用ガイドライン に関して想定される質問に対する回答をご提供するものです。

フィードアイコンの基礎

フィードアイコンとは何ですか?

フィードアイコンはそもそも、Mozilla Firefox ブラウザの ライブブックマーク 機能とともに使用するために作成されました。このような状況で、表示された Web ページにこのアイコンがあると、そのページに含まれる情報が RSS または Atom Web シンジケーションフォーマットを使用した Web フィードの形式でも使用できることを示しています。

Mozilla Foundation はそれ以降、フィード対応のアプリケーション、またはオンラインサービス、ブログ、ニュースサイトに関連して、またオープン Web シンジケーションフォーマットを介して情報を提供する一般的な Web サイトでこのアイコンを広く使用することを提案してきました。

フィードアイコンのバージョンはどこで入手できますか?

「正規」のアイコンは、14 x 14 ピクセルバージョン と 28 x 28 ピクセルバージョン で使用できます。feedicons.com の Web サイト から高解像度のバージョンやさまざまなバージョンも入手できます。

提案したフィードアイコン使用ガイドラインに従わなければなりませんか?

いいえ。提案した使用ガイドラインには法的拘束力は一切ありません。しかしながら、アイコンの使用者がガイドラインの意図する精神を尊重し遵守してくれることを願っています。

この件すべてにおいてより広範なコミュニティの役割は何ですか?

上記のとおり、フィードアイコンに関して提案した使用ガイドラインには法的拘束力はなく、アイコンの使用は商標法や関連する仕組みを使用して規定すべきだとは考えていません。むしろ、コミュニティがガイドラインの使用を監視し、法的ではない手段を使用してガイドラインを実施することに任せたいと考えています。

コミュニティのメンバーには、特定の状況におけるガイドラインの適切な解釈を決定し、今のところ想定されていない将来の使用に対応するためにガイドラインとアイコン自体を発展させる上でも支援していただきます。

理由および論理的根拠

使用ガイドラインを発行したのはなぜですか?

以前にも検討した ように、Mozilla Foundation は、ユーザ、特に Web シンジケーションの全体的な概念を知って日が浅く、Web フィードを見つけるための比較的信頼性の高いガイドであり Web フィードを操作する方法としてアイコンを見なす可能性が高いエンドユーザにとってアイコンが「何らかの意味を持つ」ことが重要だと感じていました。全く自由放任な方法を取ることは、独自に開発したがほとんどのフィード対応アプリケーションでは処理することができない Web シンジケーションフォーマットの情報に頻繁に遭遇するなどによって、これらのユーザの期待を完全に踏みにじることになる可能性があります。

提案した使用ガイドラインで、アイコンの使用がオープンフォーマットに制限されているのはなぜですか? そのことによって、Web シンジケーションフォーマットの将来のアプリケーションにおける革新が妨げられることにはなりませんか?

私たちはそうは思っていません。私たちがご提案する方法は革新の推進と相反しないと考えます。その理由を以下に挙げます。

  • ご提案した使用ガイドラインには、アイコンを使用できるフォーマットとプロトコルに対する制限的な定義はありません。特に、ガイドラインはアイコンの使用を現在の RSS または Atom フォーマットに制限していません。さらに、ガイドラインは Web フィードを提供するために HTTP 以外のプロトコルの使用を禁止していません。

  • ご提案した使用ガイドラインは、オープン Web シンジケーションフォーマットを使用して発行された Web フィード内のコンテンツ項目に独自のメディアタイプを使用するなどの技法を禁止していません。確かに、やりすぎるとこのような行為はオープンフォーマットの使用の有効性を阻害する場合がありますが、それはガイドラインの解釈および発展のためにより広範なコミュニティが検討すべき問題です。

  • 最後になりましたが、ガイドラインには法的拘束力はありません。革新の目標を達成するために、独自に開発した Web シンジケーションフォーマットを定義して一部または全部使用する必要があるとあなたが考える場合には、定義することも使用することも、そしてこれらのフォーマットに関連してアイコンを使用することも自由です。ただし、ガイドラインへの準拠をこれまでに一般に向けて確約している場合、確約と異なる行動をとることが必要であると思った理由の正当性をより広範なコミュニティに説明することはあなたの判断に任されています。

提案したガイドラインで、アイコンに関連して使用されるフォーマットをフリー・オープンソースソフトウェア開発者が実装可能にすることを求めているのはなぜですか?

アイコンは元々フリー・オープンソースソフトウェア (FOSS) で使用するために開発され、また、新しく興味深いアプリケーションで Web フィードの使用を促進するコミュニティの大部分を構成しているのは FOSS の開発者であるからです。アイコンが使用されているのに実装できない (たとえば特許またはその他の制限のために) Web シンジケーションフォーマットがあることを私たちは望んでおらず、他の FOSS 開発者も同じ意見であると思っています。

提案したガイドラインで、オープンプロセスを使用して、アイコンに関連して使用されるフォーマットを開発・メンテナンスすることを求めているのはなぜですか?

結局のところ、アイコンに関連した価値を生み出すのは Web シンジケーションフォーマットを実装および使用する広範なコミュニティであり、コミュニティが影響を持たないフォーマットに関連してアイコンを使用することはそのコミュニティにとって公正なことであると思わないからです。

ガイドラインの解釈

私はオレンジが好きではありません。別の色のフィードアイコンを使用できますか?

ガイドラインで述べたように、ガイドラインの全体的な精神に準拠しているのであれば、あなたが適切であると考える場合には (あなたの Web サイトで全体的なカラースキームに合わせるためなど)、アイコンに異なる色を使用することを含み、アイコンの代替の使用や表現を妨げることを意図していません。

しかしながら、アイコンの色は重要な視覚的役割があり、自己裁量で色を変更することはその役割を妨げユーザを混乱させる可能性があると考えています (たとえば、道路標識に使用される標準的な色を変更することはドライバーを混乱させることになるのとまったく同じです)。このため、フィードリーダーと関連製品およびサービスで使用される場合にはアイコンの色の変更を推奨しません。

購読専用・有料の Web フィードに関連してアイコンを使用できますか?

はい、できます。提案された要件では、オープンフォーマットに関連してアイコンを使用することのみを定めており、このフォーマットによって送信されるコンテンツが一般に使用可能でなければならないとは定めていません。

それ自身が独自のフォーマットである、またフォーマットの仕様に必ずしも準拠していると思われないコンテンツを含む Web フィードと関連してアイコンを使用できますか? (ポッドキャストまたはフォトキャストなど)

原則としてできますが、このような行為の特定の事例は個々に検討する価値があります。オープン Web シンジケーションフォーマットは、任意のコンテンツを使用できるように設計できます。たとえば、Atom の atom:content 要素はテキストや (X)HTML 以外のメディアタイプに使用できます。このようなオープンフォーマットは、さまざまな方法で拡張するように設計することも可能で、たとえば Atom の Simple Extension および Structured Extension 要素などによって拡張されます。

追加のメディアタイプと拡張要素をオープンフォーマットで定義できる場合とできない場合があります。またフィードを処理するアプリケーションによって追加項目を解釈する必要がある場合と必要がない (たとえば、オーディオまたはビデオのメディアタイプをサードパーティのメディアプレーヤーに渡すだけ) 場合があります。

最終的には、特定の状況で独自のフォーマットの使用に関わる「エッジケース」をカバーするためにアイコンの使用をどの程度拡大するのかを決定するのはコミュニティに任されています。たとえば、独自のフォーマットのコンテンツ項目を含む Web フィードに関連してアイコンを使用することは、提案した使用ガイドラインの精神に違反することにはならないでしょう。その一方で、Atom の定義された拡張メカニズムによって認められている方法以外の独自の方法で Atom を拡張したフィードフォーマットに関連してアイコンを使用することは、ガイドラインの精神に違反することになるでしょう。その他の事例は、さらに曖昧となるでしょう。

アイコンを使用する組織がアイコンで指すもののフォーマットを必ずしも管理していない場合はどうなりますか? (たとえば、Web ポータルがアイコンを使用してサードパーティによって使用可能となった Web フィードを識別する場合)

一般的に言って、個人および組織は自身の行動に責任を負うべきですが、他者の行動に対しては必ずしも責任を負うことはできません。あるサイトでそのサイトの Web フィードがオープン Web シンジケーションフォーマットであることを明言しているまたは暗示しているが実際的にそうではない場合には、元のサイトのフィードに関連してフィードアイコンを誤ってまたは不注意から使用した人ではなく、フィードを提供するサイトに責任があります。

提案した使用ガイドラインの違反をどのように報告したらいいですか?

それはあくまでもあなた次第です。誰かがアイコンを誤用していると思う場合には、それに関するブログを立ち上げる、適切なオンラインフォーラムに投稿する (公開されている mozilla.legal ニュースグループ、またはそれに相当する メーリングリスト など)、またはこの件に興味を持っていると思われる他の人に通知するという方法をとることができます。

提案したガイドラインへの違反はどのような処罰を受けますか?

前にも述べたように、提案したフィードアイコン使用ガイドラインには法的拘束力がありません。したがって、ガイドラインに違反した人に対して法的措置を取ることはできません。しかしながら、違反者 (企業のシニアマネジャーなど) に直接クレームを述べる、その他のコミュニティのメンバー、マスコミ、一般社会に違反を公表する、違反者の製品やサービスをボイコットするなど、コミュニティのメンバーは自由に措置を取ることができます。
