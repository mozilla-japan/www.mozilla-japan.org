Mozilla Thunderbird 0.2

リリースノート

Mozilla Thunderbird メールクライアントとは 統合型 Mozilla アプリ - スイートのメールコンポーネントを 再デザインしたものです。私たちの目的は、Mozilla メールコンポーネントに今ある機能の多くを活用し、シンプルで拡張性のあるスタンドアロンメールアプリケーションを作成することです。Mozilla Thunderbird プロジェクトについてのこれ以上の情報はこちらを参照してください。

この二番目のマイルストーンリリースでは、安定性とパフォーマンスを改善し、有望な 0.1 マイルストーンリリースの外観を磨き上げることに焦点 を当てています。さらにまだ多くの作 業が必要ではありますが、開発者たちはこの最近の進展にわくわくし、コミュニティのみなさんとこの最新の作業成果を分かち合いたいと切望しています。では 楽しんで！

Mozilla アプリ - スイートの基礎的なコードが成熟しているおかげで Mozilla Thunderbird はかなり使いものになります；しかしながら、状況的にはこのリリースはまだアルファ版の前段階であり、従って欠陥があることも十分考えられます。こうした 欠陥を見つけるのに役立てるため、開発者たちは喜んでみなさんのフィードバックを受け付けています（ MozillaZine が親切にもホストしてくれている Mozilla Thunderbird forums へのコメントか Bugzilla にバグを報告してください）。どんなバグでも Bugzilla にバグを報告する前にはこのリリースノートを十分に読んでください。

新機能

進展した主要な点の 以下のリストは、より詳細な変更履歴の要約です。ここでは主要なものの内いくつか を紹介します：
  • フットプリントとパフォーマンス

    0.1マイルストーンリリース以降、コードフットプリントの縮小とアプリケーションのパフォーマンスの改善に関してすばらしい進展が見られ まし た。ウインドウズ用のダウンロードサイズは 1.5MB 削減され、実質上 0.1 よりも早くなりました。

  • 改善されたデフォルトテーマ

    Arvid Axelsson が、彼のすばらしい Mozilla Firebird 用テーマである Qute を元にした、生き生きして斬新で、そして魅力的なテーマを改良し続けています。今回のアップデートにはオプションダイアログ用の新しいアイコン、Mail とメール作成ウインドウツールバー両方のアイコンの改良が含まれます。また、ウインドウズ用には新しいタスクバーアイコン、Mac OS X 用には Dock 用のアイコンが追加されました。

  • 改善されたスペルチェッカー

    スペルチェッカーが、返信（HTML やプレインテキスト）作成時の引用されたテキストを無視するようになりました。またメッセージ本文中であればどの URL も無視します。

  • 再デザインされたオプションダイアログ

    'Options' ダイアログのテーマがアップデート（上記参照）され、'Advanced' パネルが能率的になり、添付書類を開くときの動作を管理する 'Attachments' パネルが新しく追加されました。

  • 改善されたカスタマイズ

    メール作成ウインドウツールバーが 'Quote' ボタンをサポートするようになり、Mail ウインドウツールバーでは 'Previous' ボタンをサポートするようになりました。

  • 改善された安定性

    このリリースは Mozilla アプリ - スイート 1.5 ベータ ブランチを基礎にしています。

ダウンロード、インストールとアンインストール

古い Mozilla Thunderbird ディレクトリ（profile ディレクトリではない）は、そこにあるファイルをただ上書きするのではなく、必ず削除してください。 そうしないと結局問題を抱えるはめになります。Mozilla Thunderbird に関するバグ報告を行なう場合、まず最初にクリーンインストールを行なってからにしてください。Thunderbird 0.2 では Thunderbird 0.1 と同じ profile を使用します。このリリースにアップグレードしてもメール設定は失われません。

ボランティアの方達が Windows、Linux (GTK2)、そして Mac OS X 用の Mozilla Thunderbird バイナリーを提供してくれました。私たちはなるべく早くこれらと GTK2 以外の Linux バージョンを mozilla.org から直接提供したいと考えています。

Windows: http://core.ring.gr.jp/archives/net/www/mozilla/thunderbird/releases/0.2/thunderbird-0.2-win32.zip

Linux（要 GTK2 と XFT）: http://core.ring.gr.jp/archives/net/www/mozilla/thunderbird/releases/0.2/thunderbird-0.2-i686-pc-linux-gtk2-gnu.tar.bz2

Mac OS X: http://core.ring.gr.jp/archives/net/www/mozilla/thunderbird/releases/0.2/thunderbird-0.2-macosx.dmg.gz

訳注 : これらのリンクは、日本国内のミラーサーバ (Ring Server) からダウンロードできるようになっています。もし反映されていない場合は ftp.mozilla.org から直接ダウンロードしてください。

このバイナリーをダウンロードしたら、Winzip か gzip を使って Mozilla Thunderbird を空のディレクトリに展開 し、thunderbird.exe を起動してください。Thunderbird は専用のプロファイルを作成し、Mozilla アプリ - スイート（Seamonkey）のプロファイルとは干渉しません。

Mozilla Thunderbird をアンインストールするにはただ単にその Mozilla Thunderbird のディレクトリを削除してください。こうしておくと、将来 Mozilla Thunderbird をインストールして使用する際に利用できるプロファイルを残しておくことができます。もし Mozilla Thunderbird のプロファイル （profile）（日本語版） も取 り除きたいのであれば、Windows の場合は Application Data ディレクトリから Thunderbird フォルダーを削除し、Linux の場合はホームディレクトリから .thunderbird ディレクトリを取り除いてください。

既知の問題

以下のリストは、Mozilla Thunderbird 0.2 での既知の問題をいくつか取り上げています。新しいバグを報告する前に、このリストを読んで、定期的に確認してください。（このリリースでの新しいバグが 見つか り 次第このリストを更新していく予定です）これはまだ二番目のマイルストーンリリースなので、目前に課題が山積になっていることはよく分かっています！同 時に、 Thunderbird にわくわくしていますし、進化の過程が今のレベルまで来たことを皆さんと分かち合いたいと思います。

  • デフォルトのテーマがまだ完全には実装されていません。まだ Mozilla クラシックのテーマアイコンをたくさん使っています。Arvid のデザイン作業は進行中であり、これから先も改良がなされていきます。
  • Unix のユーザの方へ：オペレーティングシステムとの統合がまだ済んでいません。メールメッセージ内の http url をクリックしてもデフォルトの Linux ブラウザを立ち上げることができません。この問題は次のリリースで解決したいと考えています。この問題が Linux ユーザーにとってどれだけ重要な問題かは理解していますので、あらかじめお詫びしておきます。
  • 英語以外の辞書を追加するには、イン ストールの方法を参照してください。
  • OSX ユーザの方へ：Thunderbird の開いているウインドウのすべてを閉じると、たいていは何の動作もしない Seamonkey に似たメニュー項目を持つ変なメニューが表示されることがあります。この '隠れたウインドウ' をどうにかする必要があります。できれば三番目のマイルス トーンビルドまでには何とかするつもりです。
  • OSX ユーザの方へ：開いているアプリケーションウインドウを切り替えることのできる 'ウインドウ' メニュー項目がメニューバーにまだありません。これも近い将 来に追加していと考えています。
  • 学習機能つきジャンクメールフィルタを有効にするには、アカウントごとの設定（Tools / Junk Mail Controls / Adaptive Filters）で行ってください。ただし、UI だけを見ると全体設定のように見えてしまうので注意してください。複数のアカウントでフィルターを有効にするには、Settings タブをクリックして、アカウントメニューリストから設定したいアカウントを選択して、Adaptive Filter パネルに戻り、自動ジャンクメール検知機能の有効／無効を切り替えてください。
  • 作業中は、メールの３ペイン内でのプログレスバーは表示されません。常に隠れている状態のままです。
  • 現時点では Mozilla プロファイルを Thunderbird に移行するようにはなっていません。Thunderbird を試してみたい場合は、新しいプロファイルを使用してメールアカウントの情報を入力してください。Mozilla プロファイルを手動で（日本語版）移 行することもできますが、私たちとしては新しいクリーンなプロファイルの使用をすすめます。
  • POP アカウント設定であるサイズ以上のメッセージ本体をダウンロードしないように設定すると、後から残りのメッセージをダウンロードしようとしても失敗するこ とがあります。
  • Windows で Mozilla Thunderbird をデフォルトのメールクライアントに設定する場合、OS からではなく、Tools / Options General から設定することをおすすめします。Windows のインターネットオプションからでは、この OS にキーを登録できません。
  • Linux では Mark All Read のキーボードショートカットが動きません。

ユーザサポート

初めて Mozilla Thunderbird を使用する方は、このすばらしい Mozilla Thunderbird 入門 を参照するとすぐに使えるようになるでしょう。MozillaZine のホストする Mozilla Thunderbird forums では、全般的な問題を話し合ったり、フィードバックを提供したり、仲間のユーザのサポートを得ることが出来ます。FAQ、ヒントやコツ、その他のカスタマ イズに関するヘルプは Mozilla Thunderbird Help （日本語版）を参照 してください。（訳者注：MozillaZine 日本語版にも Mozilla Thunderbird フォーラムがあります）
