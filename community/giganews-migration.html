Giganews 移行に関してよくある質問

このページは現在も編集中です （ページ最下部にある最終更新日を確認してください）。

なぜ新しいニュースサーバへ移行するのですか？
これまでのサーバは、かつて Netscape をサポートしていた古い機材を利用して、AOL によってホストされてきました。そのため、サーバの管理権限がないのみならず、運営継続そのものに関する保障も一切ありません。しかし移行することで、再びニュースグループリストの管理権限を手に入れることができ、必要に応じてグループを追加していくことができるようになります。この私たちの新しいニュースグループサーバは Giganews の Newsgroups によって運営されることになります。
ニュースグループへのアクセス方法は？
現時点では NNTP を経由して news.mozilla.org に接続できます。（secnews.netscape.com へのエイリアスは新サーバでは対応しません。）
ではドメイン名は引き続き news.mozilla.org ですか？
はい、そのとおりです。現時点では、Giganews サーバへのドメイン名の移行を、太平洋標準時 2006年 1月 9日月曜日の午前中に予定しています。
「snews」（NNTP over SSL） を使わなければなりませんか？　それとも普通の 「news」 アクセス （NNTP） でも大丈夫ですか？
普通の news:// アクセスにはこれまでどおり対応します。secnews.netscape.com 上の snews:// でしか利用できないサポートグループもあります。こうしたグループのユーザの皆さんも新しい階層へと移行してくださるようお願いします。
非 SSL アクセスが標準の接続方法である場合、SSL によるアクセスはできますか？
現時点では、SSL によるアクセスは計画されていません。というのも、誰もサポートしなければならないという説得力のある理由を提示していないからです。個人的には、SSL 接続が 唯一の アクセス方法であれば、例えばスパム対策には有効かもしれないと思います。ただ、今回の場合、そうしたケースにはあたりません。
購読にあたっては GigaNews の有料サービスを利用しなければならないのですか？
もちろんその必要はありません :-) グループは引き続き無料で利用できます。
新しいニュースグループは Usenet で広く公開されるのですか？　新しいニュースグループは Google Groups からアクセスできますか？
この 2 件の質問はお互い関連していますね。広く公開することにはメリットとデメリットがあります。デメリットを挙げると、広く公開されると、簡単には投稿を削除できなくなります。そのためスパムが蔓延り、開発者の参加意欲を削ぐことにもつながりかねません。そこで、初めのうちは、公開場所を Giganews の news.mozilla.org と Google Groups に限定してみようということになりました。どちらのサービスもそれ以上メッセージを拡散させないようにするという方針で合意しています。コミュニティからのフィードバックによっては、こうした設定は変更されるかもしれませんが、一旦広く公開すると決めてしまうと後戻りはできませんので、まずは 「安全策」 から始めてみようということです。:)
過去のニュースグループコンテンツへのアクセスはできますか？　もしアクセスできるとすると、どの程度の期間利用できるのでしょう？
アクセスできます。AOL がニュースグループを維持する限り、古いサーバ （secnews.netscape.com または oldnews.mozilla.org） を経由してアクセスできます。また、古いサーバのコンテンツをミラーしているあらゆるサーバを経由してアクセスすることも可能です （それらサーバがコンテンツを保持している限り）。他には Google Groups から参照することも可能です。こちらは、ほぼ恒久的に可能だと思われます。過去のコンテンツは、Giganews への移行が完了するまで は（もしかすると永続的に）、Giganews 上の news.mozilla.org からもアクセスできます。しかし、Giganews には最近 3 年分の投稿しかなく、メッセージポインタも異なります。つまり、1月 9日にドメイン名が変更されたら、古いニュースグループの購読を解除して、新しいニュースグループを購読する手続きをして、「最後に読んだ」 箇所まで同期させなければなりません。
ニュースグループへの投稿を読んだり、投稿したりする方法として、これまで通りメーリングリストを使えますか？　できるとしたら、どうやって？
これまでと同じように、利用できます。リストにもあるように 【訳注：mozilla.org へ飛びます】、明確な移行パスがあるメーリングリストは自動的に、古いグループから新しいグループへと移されます。
スパムフィルターについてはどうですか？　GigaNews や MF はどのような対策を講じているのでしょう？
GigaNews は、スパムを発信するサーバに対して UDP を発動する強力な権限 （投稿全体に対して 10% 強） を持っているため、スパムは問題にならないと語っています。とりあえず、彼らの言葉を信じて、状況を見極めようと考えています。
ニュースグループやメーリングリストで問題が発生した場合の連絡先は？
mozilla.org の product から Server Operations を選んで bugzilla.mozilla.org に bug として報告してください。
