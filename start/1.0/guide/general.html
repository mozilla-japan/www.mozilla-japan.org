Mozilla アプリケーション スイートとは何ですか？

Mozilla アプリケーション スイートは、インターネット クライアント ソフトウェアの洗練されたセットで、次のものを統合しています。

  • Web ブラウザ
  • E-mail プログラムとニュースリーダー
  • チャット クライアント

このスイートは多くのオペレーティングシステムを越えて矛盾なく動くように設計されていて、これは異種混合のソフトウェア・ハードウェア環境と対話しているだれとっても重大な利益です。サポートしているプラットフォームには次のものがあります : Win32 （Windows 9x、ME、NT、2000、XP）、MacOS （8.5、8.6、9.x、OS X）、Linux、BSD/OS、FreeBSD、HPUX、Caldera OpenUNIX8 and Unixware 7.1.1、OpenVMS、OS/2、Solaris、Tru64 Unix

そしてコードベースはオープンソースであり、私有ソフトウェアを利用するときには入手できない莫大な権利と選択をユーザに与えます。

 

誰が Mozilla アプリケーション スイートを利用できるのですか？

このスイートはディストリビューターに使われることを意図していますが、MPL / NPL （Mozilla and Netscape Public Licenses） のもとで、だれでも自分たちの製品に Mozilla アプリケーション スイートを利用することができます。

Mozilla 製品はそのままでも、また変更を加えた形でも、再配布は自由で、サードパーティーは自分たちが適当と考えるようにこれを再パッケージ・再商標化することが期待されます。これらのサードパーティーは、自分たちの観客のニーズに合うように、追加文書と（あるいは）サポートの提供を選択するかもしれません。

さらに、インターネットブラウジングやメール／ニュースクライアント、チャットがほしいと思っているエンドユーザは mozilla.org から直接アプリケーション スイートを ダウンロード するかもしれません。私たちの最近のマイルストーン リリースは毎回 30 〜 40 万回ダウンロードされています。それだけの数の人々が mozilla.org からリリースされている Mozilla ソフトウェアの利用を選択しているのです。

しかし、mozilla.org からのリリースはディストリビューターや開発者をターゲットにしています。mozilla.org は一般大衆向けに販売を行うつもりはありませんし、一般大衆にエンドユーザ サポートの提供もできません。サポートはいくつかのエンドユーザ セルフサポート ニュースグループ に限られています。

私たちに希望は、多くのユーザーが、Netscape、Red Hat、IBM、Beonex といった、有料・無料のサードパーティーの再配布製品の一部として Mozilla を手に入れることです。Mozilla テクノロジーの他のアプリケーションもまた、開発中です。

 

Mozilla ツールキットとは何ですか？

Mozilla ツールキットは、（Gecko レイアウトエンジン、JavaScript エンジン、ネットワーク・ライブラリ、ポータビリティ・レイヤーといった）コンポーネントのセットと、XML、CSS、ECMAScript （JavaScript） のような技術を使ってプログラムすることを考えている、便利な プログラミングツール （JavaScript デバッガ、DOM Inspector など）です。

この単一のフレームワークの中で、（Web ページのような）取るに足らないものから、Mozilla ブラウジング・メール・チャット アプリケーションそのもののような洗練されたものまで及ぶアプリケーション制作が可能です。

 

Mozilla コードベースから他にどんな製品を制作することができますか？

Mozilla 製品は別として、OEM （original equipment manufacturers） は、幅広い製品の制作に、モジュラー式の Mozilla コードベースから一部を組み合わせたり調和させたりしています。それらの製品のいくつかは、Galeon や Chimera （Mac OS X 向けの Gecko ベースブラウザ）といった その他のブラウザ です。

他のアプリケーションに Mozilla のレンダリングエンジン （「Gecko」として知られています） を内蔵した一部の製品は、それらのアプリケーションに HTML を理解・表示させています。Intel の Dot.Station や、Worldgate からまもなく発売されるセットトップ型の製品といった例もあります。

その他の製品は ECMAScript （JavaScript） や Mozilla の暗号化能力のようなコンポーネントを、ブラウザと少し関係のある製品に使っています。たとえば、ActiveState は Komodo IDE 開発者ツールの制作に Mozilla のテクノロジーを利用していますし、一方 OEone は、家庭をベースにしたコンピュータ ユーザーのための オペレーティング環境 「hometop」の制作に Mozilla のテクノロジーを利用しています。

Mozilla テクノロジーを利用した 70 以上のプロジェクト が www.mozdev.org でご覧いただけます。

Mozilla コードベースはオープンソースソフトで、これは無償で利用、修正、再配布ができると言うことを意味しています。他の方々が、私有コードと Mozilla のコードを結合したり、有償の商用製品を制作することができるように、このコードのライセンス条件は注意深く開発されました。もちろん、これはその他の無私有ソフトウェアの制作に利用することもできます。

 

Mozilla 1.0 とは何ですか？

Mozilla 1.0 は、以下の製品の、最初のメジャーバージョン リリースです :

  • Mozilla アプリケーション スイート
  • クロスプラットフォーム、インターネットベースの GUI アプリケーション 用 Mozilla ツールキット

Mozilla 1.0 はまた、API 安定・完成のレベルをマークしています。Mozilla 1.0 到達までの主要な努力は、Mozilla を付加的なアプリケーションと機能性を制作するために利用する人にとって、安定したベースを提供する API （アプリケーション・プログラミング・インターフェース）のセットの一体化と決着でした。

API のより抜きのグループが「＠フローズン」をマークしました。mozilla.org は Mozilla 2.0 まで、このセットの API 互換性を維持するつもりです。Mozilla 1.0 現在でフリーズされた API のリストも 入手可能 です。

Mozilla 1.0 また、mozilla.org が安定用と開発用のコードベースのブランチのどちらも維持するという、最初のポイントをマークしました。

かなりの新しい成果が主要部にみられ、ブランチがその安定性とパフォーマンス特性を維持しておくようにします。これによって、Mozilla のコードを使った製品を制作したいと望んでいる企業やプロジェクトは、長持ちする安定したブランチにアクセスすることができ、最新の開発サイクルにおける毎日・毎週の変更に頭を悩ませる必要はありません。

 

オープンソースソフトの利益とは何ですか？

オープンソースソフト のソースコードは、無償で、用途を問わず、利用することができます。これは多くの利益を提供します :

  • ソースコードは常に利用可能です
  • ソフトウェアを利用したり、修正したり、配布したりするコストがかかりません
  • 受け取り側 - 利用者や OEM - が、自分たちのニーズに適合させたいと思うように、ソフトウェアを修正できます
  • 受け取り側が、彼らにとって重大なバグを修正することができます
  • 「使用期限終了」の決定が、受け取り側に強制させられることはありません
  • 開発者の技術的、あるいは商業的な決定が、受け取り側に強制させられることはありません
  • 専門的技術やリーダーシップの広範囲にわたる蓄積が開発できます

オープンソースソフトの技術と開発の利益は The Cathedral and the Bazaar の中で詳しく論じられています。オープンソースソフトは受け取り側にも多くの利益を提供するのです。

たとえば、売り主が特に重要だと受け止めていないけれどもユーザーや OEM にとっては致命的なバグが、ソフトウェアにはしばしばあります。オープンソースソフトなら、バグを修正するために、受け取り側は売り主に圧力をかけたり、料金を払うことを制約されません。受け取り側は自分自身で 自由にバグを修正できる のです。

もし受け取り側が専門的技術を持っていなかったり、特殊なオープンソースソフトで仕事をすることを選ばないのなら、バグを修正するために、他のところ - 最初の売り主とは全く違ったところ - とのサポートや契約に加入することができます。

売り主が製品のエンドユーザー・サポートをしたいと思うとき、ユーザーや OEM はしばしば否定的に受け取ることから、使用期限のことも重要で、それは彼らの意志に反して、製品をアップグレードや切り替えを受け取り側に強制することがあります。

オープンソースソフトなら、だれも製品の使用期限を強制される ことはありません。人々はプロジェクトの作業をやめるかもしれませんが、他の人がコードの維持・開発を決めることができます。このように、ユーザーや OEM には私有ソフトウェアでは入手できない選択の自由があり、彼らは最高のソリューションだと思われる他の製品に乗り換えることを選べるのです。しかし彼らはまた、問題のソフトを使い続けるよう選択することもできますし、そうではなく他の製品への加入を選択することもできるのです。

製品の開発がユーザーや OEM のニーズと相容れない方向に進んだようなときにも、同じことが当てはまります。受け取り側は他の製品に乗り換えることを選択できますが、そうではなくそのソフトを使い続ける（または他の製品に加入する）ことも選択できます。

このようなソフトウェアの維持・開発の方法の 1 つの結果は、広範囲にわたる専門的技術やリーダーシップの蓄積の開発です。私有ソフトでは、専門的技術は通常、売り主の従業員に集中、限定されていますが、オープンソースのプロジェクトでは、技術と興味がある人ならだれでもプロジェクトに参加し、専門的技術を開発することができます。

自分たちの製品に Mozilla のコードを利用する企業は、多くのレベルで組織内の専門的技術を開発することができます。彼らは、自分たちの製品にとって重要な、独自の機能性の特殊化に専念したい、あるいは Mozilla の主要なコンポーネントについての専門的技術を開発したいと思うでしょう。専門的技術と責任がリーダーシップとなり、リーダーシップは将来のソフト開発にさらに大きな刺激となります。
