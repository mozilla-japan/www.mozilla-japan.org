Warpzilla とは何？

Mozilla プロジェクトでは、誉高い・オープンソースな・Firefox ウェブと関連製品およびテクノロジーを開発することで、インターネットでの選択肢を提供し 革新を続けています。 Warpzilla はこれら製品の OS/2 移植版で、OS/2® や eComStation 上で実行できるようデザインされています。

現在のインストールガイドとシステム要件

  • 現在このページ上部に記載されている全製品には C runtime DLL (libc051 か libc061 のどちらか) が必要ですが、ダウンロードした製品内には含まれていません。 後者は 2006 年 3 月 22 日のナイトリー 1.8 ブランチ及びトランクビルドから必要となりました。 前者は、それ以前のさまざまな古いリリースビルドに必要です。 希望のビルドに合う正しいバージョンを決定するためには、上記のそれぞれの README やニュース項目を参照してください。 物事を簡単にするには、それらをすべて LIBPATH 内のどこか一ヶ所に置いてください。 他の方法としては、製品の exe ファイルと同じディレクトリにそれらを置いてください。 インストーラは C runtime DLL を導入しないはずですが、実行のためには必要です。
  • Mozilla 全製品の最小必要条件 …
    • Pentium クラスの CPU
    • 64 MiB RAM
    • 64 MiB の空きスワップ・スペース
    • 35 MiB のインストール用 空きハードディスク空間と、 ディスクキャッシュとメールのための記憶空間
    • OS/2 Warp 4 (Merlin カーネル)
    • Fixpack 15 かそれ以降
    • MPTS バージョン 5.3 かそれ以降
    • TCP/IP バージョン 4.1 かそれ以降
    • INETVER: SOCKETS.SYS=5.3007, AFOS2.SYS=5.3001, AFINET.SYS=5.3006 かそれ以降

      注記: これら INETVER レベル以前の MPTS ＆ TCP/IP は使用しないでください。 古いプロトコルスタックでも Mozilla の起動・実行は正常に動くように思えますが、Mozilla が必要とするいくつかの機能は、古い MPTS バージョンでは実装されていないので、 クラッシュやデータ損失を招く恐れがあります。

    • 長いファイル名をネイティブにサポートしているファイルシステム、例えば HPFS や JFS
  • Mozilla 全製品の推奨動作条件 …
    • 500 MHz CPU (もちろん、より速いに超したことはありません)
    • 256 MiB RAM

      注記: より多くの物理 RAM が利用可能であれば、パフォーマンス＆安定性が向上します。 特に、長いブラウジングや IRC セッションには、メモリー 512 MiB がお勧めです。

    • Fixpack 16
    • MPTS バージョン 5.5 または 6.0
    • TCP/IP バージョン 4.3
    • INETVER: SOCKETS.SYS=6.2004, AFOS2.SYS=6.2000, AFINET.SYS=6.2013

      あるいは

    • OS/2 Warp 4.52 Convenience Pack 2 (サブスクリプション: 加入)

      あるいは

    • eCom Station 1.0 かそれ以降

      注記: MPTS 6 と FP 16 は Software Choice 加入者のみ入手できます。 その他のアップグレードは今のところ無償で入手できます。
