Mozilla アクセシビリティ

このページの管理: Aaron Leventhal - aaronleventhal@m o o n s e t .net

このページには、Mozilla や Netscape 社内で進行中の Web ブラウザアクセシビリティ作業に関するリソースやドキュメントが載っています。

ユーザ向け

  1. キーボードに関するドキュメント
      このリストは、Mozilla のキーボードに関する情報を 1 つの長いドキュメントにまとめたものです。
      また、並び替え可能なキーボードコマンド のページもあり、ここには (キープレス、プログラム領域、コマンドごとに) 好きなように並び替えができるキーボードショートカットのリストが載っています。
  2. Find As You Type
      Find As You Type (以前は Type Ahead Find と呼ばれていました) は、ブラウザ内で文字を続けてタイプしたときに、すばやい Web ページのナビゲーションを可能にする機能です。 Find As You Type のコードは Netscape Corporation によって Mozilla に提供されたものです。

開発者向け

  1. アクセシブルな XUL 記述ガイドライン
      XUL は、これらのガイドラインに従って記述すれば、アクセシブルなユーザインターフェイスを作成できます。コーダー、レビュー担当者、設計者、品質保証技術者は、これらのガイドラインをよく理解しておくべきでしょう。
  2. XUL アクセスキー FAQ
      XUL にアクセスキーを追加する前に、誰もが目を通すべき資料です。
  3. nsIAccessible を組み込んだ JavaScript 使用例
      このページは、JavaScript で nsIAccessible をどのように使えるかを示したものです。ここにあるサンプルアプリケーションは、HTML に対する Mozilla のアクセシビリティ API サポートをテストするために利用できます。このアプリケーションを改良してくれるボランティアを募集中です。
  4. Mozilla Developer Day Accessibility Talk
      アクセシビリティのトピックを紹介します。いくらか Mozilla 特有の問題にも触れています。スライドショー形式での画像版です。テキスト版もあります。 これはテキスト版です。完全版が必要であれば aaronleventhal@ m o o n s e t  . net. に メールしてください。
  5. Netscape、Mozilla、Gecko におけるアクセシビリティサポートの発表
      代替テクノロジー開発者向け: Netscape は、Netscape 7.01、Mozilla 1.2、Gecko 1.0 時点でのアクセシビリティサポートを発表しています。なお、私たちはこれからも、スクリーンリーダー、スクリーン拡大ソフト、音声入力ソフトのベンダーによるサポートを求めていきます。
  6. Windows アクセシビリティベンダー向け Gecko 情報
      Mozilla は、HTML や (XUL テクノロジーをベースとした) ユーザインターフェイス内で MSAA (Microsoft Active Accessibility) をサポートしました。これによって、スクリーンリーダー、スクリーン拡大ソフト、音声入力ソフトといった Windows 向けアクセシビリティソフトのベンダーによる Mozilla のサポート組み込みが可能になります。さらに、私たちは、読み取り専用 DOM (Document Object Model) を公開し、いくつかの簡素なインターフェイスを提供します。
  7. Mozilla のアクセシビリティアーキテクチャ
      これは、クロスプラットフォームのアクセシビリティレイヤーを Mozilla に実装する方法を解説したアーキテクチャドキュメントです。クラス階層やアクセシビリティキャッシュに関する説明が載っています。その他の資料: Sun の北京駐在 Mozilla アクセシビリティチームによって提供された イベントフロー図 もご覧ください。
  8. MSAA サーバ実装のための解説書
      デスクトップアプリケーションに MSAA サポートが必要な開発者向けのドキュメントです。サードパーティの支援技術によって、アプリケーションをアクセシブルにするための、役に立つ情報や Mozilla の実装に関する詳細が載っています。
  9. Mozilla のキーボードユーザインターフェイスプラン
      これは人々が集まって譲歩した成果です。ほとんどの部分については、だれもが満足する仕様に決まりました。あなたの意見を Mozilla アクセシビリティメーリングリストにお寄せください。

      付録 1: Microsoft による優れたリソース - キーボードショートカット実装ガイドライン [日本語訳]

      付録 2: Mac OS X の Aqua ユーザインターフェイスにおけるキーボードガイドライン

  10. 組み込みアクセシビリティ API
      組み込み製品におけるアクセシビリティは不可欠です。組み込み開発者が各自の製品でアクセシビリティをサポートするために可能なことを説明します。
  11. アクセシビリティ向上のためのプラン
      このドキュメントは、ユーザインターフェイス設計チームが、十分に利用可能なアクセシビリティを実現するために、さらにユーザインターフェイスに追加しなければならないことを理解し、計画するために役立つことを意図したものです。
  12. Mozilla の Section 508 準拠
      Section 508 (米国リハビリテーション法 508 条) は、アメリカ合衆国の連邦政府機関が使用できるソフトウェアアプリケーションを決定する際に役立つ、一連のアクセシビリティ要求事項です。
  13. Mozilla の W3C ユーザエージェントガイドライン評価レポート
      W3C の WAI は、UAAG と呼ばれる、もう 1 つの重要なアクセシビリティ標準を推進しています。これらのガイドラインは Section 508 よりさらに進んだもので、Mozilla は大変よくこれらの項目を満たしています。
  14. アクセシビリティ API クロスリファレンス
      マルチプラットフォームな製品にアクセシビリティ API サポートを実装しようと考えているすべての人にとって役に立つ資料です。アクセシビリティの役割、現状、イベントなどに関する情報が載っています。
  15. アクセシビリティツールキット・チェックリスト
      新しいツールキットを開発する際に必要な、アクセシビリティに関する作業を、ウィジェットごとにまとめたリストです。
  16. 自動アクセシビリティテスト
      この自動テストは、Mozilla の nsIAccessible API が正常に機能しているどうかを判断するのに役立ちます。説明に注意深く従ってください。テストを実行する際には、あなたの pref.js ファイルに設定を 1 行追加する必要があります。
  17. アクセシブルな Web ページ作成 [日本語訳]

      Web ページ制作者が、時間を掛けずに簡単な方法でアクセシビリティを学ぶための、簡潔で実用的な入門書を IBM がまとめました。ご覧になることを強くお勧めする、優れた資料です。

      Dive Into Accessibility は、アクセシブルな Web サイト制作に関する、もう 1 つの優れた分かりやすい資料です。

  18. ソフトウェアアクセシビリティの現状
      コンピュータソフトウェアのアクセシビリティは、過去 20 年間で飛躍的な向上を見せました。このドキュメントは、その経過と、この間に開発された技術を再検討したものです。
  19. 役に立つ資料
      Web アクセシビリティやオープンソースアクセシビリティについてさらに詳しく学びたい人のためのリンク集です。
  20. Unix アクセシビリティプロジェクト
      これは Sun のアクセシビリティプロジェクトに関する説明です。

ディスカッションリスト

    Mozilla アクセシビリティニュースグループ を追って、私たちの Mozilla アクセシビリティ開発の取り組みについて話し合いましょう。キーボードで Mozilla を使う方法や、スクリーンリーダー、スクリーン拡大ソフト、音声入力ソフトをどのように機能させるべきかを議論しましょう。メーリングリスト形式がよければ mozilla-accessibility-request@mozilla.org を購読してください。あるいは aaronleventhal@ m o o n s e t  . net にメールを送ってくださっても構いません。
