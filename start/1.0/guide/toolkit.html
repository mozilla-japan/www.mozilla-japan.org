Gecko とは何ですか？

埋め込み可能な、クロスプラットフォームの Gecko レンダリングエンジンは Mozilla の心臓です。 これにはユーザインターフェースがありません。ただ Web コンテンツを理解し、表示するだけです。 この能力は広範なアプリケーションのセットにとって価値のあるもので、その一部は「ブラウザ」と少し関係があります。 たとえば、Gecko は （XUL と協力して（下をご覧ください）） Mozilla アプリケーションのユーザインターフェースの表示に使われています。

Gecko は 1997 年に開発が始まり、公開で移植性・拡張性があり、カスタマイズ可能なアーキテクチャを特徴とした、やせた、高速で強壮なレイアウトエンジンに進化しました。 それは、HTML-PNG コンバータから市販用 IDE まで、様々なサードパーティーのソフトウェア・ハードウェア製品に組み込まれています。 Gecko は最新の Compuserve 7.0 サービスのためにレンダリングエンジンとして役立ち、また近頃リリースされた Netscape 7.0 Preview Release や AOL サービスにおいて、ベータテストのエンジンにもなっています。

いくつかの純粋なブラウザのプロジェクトを含むその他の例 :

  • Galeon （XUL 無視、Unix/Linux 向けの GNOME デスクトップの特徴、過去数年で普及）
  • Chimera （XUL 無視、MacOS X 専用）
  • K-Meleon （XUL 無視、Windows 専用）
  • m/b （XUL 使用、クロスプラットフォーム）

組み込み Gecko に関する詳しい技術情報は mozilla.org と codeguru.com の両方のページで入手できます。

 

XUL とは何ですか？

XUL （eXtensible User-interface Language） は Mozilla の XML ベースのクロスプラットフォーム・ユーザインターフェース技術です。

この技術は、開発者が XML、CSS、ECMAScript (JavaScript) を混ぜて使って、クロスプラットフォームのグラフィカル ユーザ インターフェースを定義できるようにします。 ユーザは、スタンダードな Web ページ構文を使用しているファイルを変更することによって、簡単に（Mozilla 製品そのもののような）XUL ベースの Mozilla アプリケーションのユーザインターフェースの外観をいくらでも変更できます。

XUL は定義されたユーザインターフェースを実装するために Gecko を使っています。 Gecko/XUL の組み合わせは、Mozilla アプリケーションを、Web ページのようにシンプルなものから Mozilla 1.0 製品のように洗練されたものまで拡張させる基礎となっています。 これは、メニュー、入力コントロール、ダイアログ、（たとえば設定ダイアログに見られるような）ツリーコントロール、キーボードショートカットを含め、現代のグラフィカル ユーザ インターフェースに見られる基礎的な要素をすべてサポートしています。

XUL のチュートリアルは XUL Planet でご覧いただけます。 また、XUL と JavaScript によるアプリケーション制作 に関するスライドもあり、 さらに詳細な技術情報（1、2）も付いています。 詳しい情報は mozilla.org でも入手できます。

 

XPCOM とは何ですか？

Mozilla コンポーネントは XPCOM コンポーネントです。 XPCOM （Cross-Platform Component Object Model） は、最初 Mozilla プロジェクトの一部として制作され、現在いくつかの他のプロジェクトで利用されています。 XPCOM は、よく知られた Component Object Model （COM） の軽量なクロスプラットフォームの変形です。 IBM の Web サイトにある 論説記事 がそれについてさらに論じています。

すべての XPCOM オブジェクトは自動的に ECMAScript （JavaScript） の送り込みを増やすので、大部分の Mozilla 開発者は XPCOM そのものについて頭を悩ます必要なしに、クロスプラットフォームや XPCOM が供給するその他の開発関係の恩恵を享受するでしょう。

 

Necko とは何ですか？

Necko （「Gecko」と韻（いん）を踏んでいます）は Mozilla のネットワークライブラリです。 これは、トランスポート層からプレゼンテーション層まで、ネットワーク値域の各層のためにプラットフォーム独自の APIを提供します。 ライブラリ内部のコンポーネントは、プロトコルハンドラ（HTTP、FTP など）のスタンダードベースの実装や、DNS、ユーザ識別、プロキシ接続、キャッシュを含めた、いくつかの主要なネットワーク機能を含んでいます。

API は Mozilla 製品の中で使用され、その他のネットワーククライアントの書き込みにも利用できます。 Necko ネットワークライブラリに関する詳しい情報は http://mozilla.org/projects/netlib/ で入手できます。

 

Imagelib とは何ですか？

イメージ レンダリング ライブラリ （Imagelib） は多くの画像フォーマットをデコードし、Mozilla で表示できるようにします。 これは完全なモジュラー式で、スタンダードに準拠し、JPEG、PNG、GIF、PPM （Portable Pixel Map） といった一般的な画像フォーマットのサポートを含んでいます。

 

なぜ JavaScript (ECMAScript) が Mozilla ツールキットの大変重要な部分なのですか？

JavaScript は当初 Netscape で開発された、広く使われているスクリプト言語です。 現在 ECMA-262 委員会 によって「ECMAScript」として標準化されています。 Web ページ上だけではなく、サーバアプリケーションなど、至る所にあります。

Mozilla は 2 つの JavaScript の実装を備えています。 オリジナルでもっとも広がっている実装は C 言語で書かれ、Mozilla ブラウザに組み込まれており、それに対して 2 つ目の実装は Java で記述されています。 JavaScript の実装、規格、テストスイート、チュートリアル、ニュースグループについての情報、は http://mozilla.org/js/ でご覧いただけます。

Mozilla ツールキットによって、JavaScript デバッガ Venkman は、Mozilla ベースのブラウザ（K-Meleon や Galeon といった Gecko のみのブラウザを除く）向けに、強力な JavaScript デバッグ環境を提供します。

今日まで開発されてきた結果、Venkman は開発者が次のようなことをできるようにしました。

  • ブレークポイントのセット／クリア
  • 単独のステップ・イントゥー、ステップ・オーバー、ステップ・アウト
  • スタック・フレームのリスト
  • 例外が投げられたときの中断／追跡
  • エラーが発生したときの中断／追跡 （捕まえられていない例外）
  • あらゆるスタック・フレームでの局所変数の検査
  • 数えられないプロパティを含む、オブジェクトのプロパティの列挙
  • あらゆるスタック・フレームでの任意の JavaScript の評価
  • 時計のセット
  • JavaScript の描画 （呼び出し時間での収集と報告）
  • JavaScript の「プリティ・プリント」 （余白と復帰改行の「修正」）
  • 生成子の場所の特定、任意のオブジェクトのための位置の生成

ウォークスルー を含む Venkman に関する追加の情報は http://mozilla.org/projects/venkman/ でご覧いただけます。

 

Mozilla ツールキットを使うことの主要な利益とは何ですか？

Mozilla アプリケーションは、サポートされているどんな Mozilla プラットフォームでも、変わらず、十分機能的に実行します。

Mozilla は完全なオープンソース／フリーソフトで、これは必然的に多くの開発者の利益を提供します。 たとえば、広く使われているオープンソースコードは、私有ソフトウェアよりも より強壮で安全 という傾向があります。

そしてバグがあるときは、開発者はコードを調べて問題を修正する権限を与えられます（そしてむしろすべての利益を得るために好んで変更をチェックするでしょう。「コードに与えれば、それは自分に帰ってくる」）。

Mozilla はデスクトップ アプリケーションと Web ページの区別をほとんど取り除きます。これはいくつかのさらなる利益へ帰着します。 普及している、コンポーネント、API、オブジェクト、イベント、メソッド、プロパティ、洗練されたグラフィカル ユーザ インターフェースのような要素に信頼をおいて、現在デスクトップアプリケーションのプログラミングをしている開発者は、快適なプログラミング環境を見つけるでしょう。 しかしながら、より多くのテクノロジーは合意によるスタンダードに基づくようになるでしょうし、Web デザインの最大限のパワーはユーザ体験を豊かにすることにまもなく役に立つようになるでしょうし、アプリケーションはクロスプラットフォームになるでしょう。

逆に、多くの Web デザイナーは、自分がよく知っている分野の外へ飛び出す必要なしに、成熟したアプリケーションを開発する技術を発達させることができる、ということに気付くでしょう。

 

Mozilla にはどんな開発者ツールが組み込まれていますか？

Mozilla 1.0 には、DOM Inspector （この チュートリアル も入手できます）や JavaScript デバッガ、Java/JavaScript コンソール、構文ハイライトソース表示が含まれています。

 

ツールキットはどんなセキュリティを使っていますか？

Mozilla のセキュリティは、Netscape の技術者によって書かれた NSS （Network Security Services ライブラリ）から始まっています。 このライブラリは、1998 年の「Free the Lizard」キャンペーンを見越したもので、次のような利益を受けています :

  • 公開検査 : セキュリティのエキスパートはたいてい 優れたセキュリティ はコードを公開しておくことで成し遂げられる、ということに同意します。
  • 経験 : Netscape は Secure Sockets Layer （SSL） プロトコルを開発しました。それは現在 Web 中のソケットベースの暗号化と認証のスタンダードで、最初に広まった商業利用の X.509 v3 公開鍵証明書が実装されています。

NSS の一番上位の層を成しているのは Personal Security Manager （PSM） です。 PSM は Mozilla のようなクライアントに代わって暗号化演算を行い、ECMAScript （JavaScript） を利用しているコンポーネントのように近づきやすいのです。

 

Mozilla コードベースを利用したいと思っている OEM は、追加の情報を得るには誰にコンタクトしたらいいですか？

Mozilla コードベースの利用に興味のある OEM は、mozilla.org の Chief Lizard Wrangler である Mitchell Baker にコンタクトするといいでしょう。
