セキュリティ モジュール データベース ツールを利用する

ニュースグループ： netscape.public.mozilla.crypto
著者： Sean Cotter
管理者： Bob Lord

セキュリティ モジュール データベース ツールは、ハードウェアトークン内、または secmod.db ファイル内にある PKCS #11 モジュール情報を管理するコマンドライン ユーティリティです。 このツールを利用して PKCS #11 モジュールの追加／削除、パスワードの変更、 ディフォルトの設定、モジュールのコンテンツの一覧表示、 スロットの使用可／不可の決定、 FIPS-140-1 準拠 （FIPS-140-1 compliance） の開始／停止の決定、および暗号化操作のためのディフォルト プロバイダの決定に利用することができます。 このツールはまた、 key3.db、 cert7.db、および secmod.db の各セキュリティ データ ファイルを作成することもできます。

セキュリティ モジュール データベース管理に関連する作業には、 多くの場合、暗号鍵データベース （key3.db ファイル） や証明書データベース （cert7.db ファイル） の管理プロセスが含まれます。 暗号鍵、証明書、および PKCS #11 モジュール管理プロセスは、一般的に、 証明書や証明書データベースを作成し管理するのに必要な、 暗号鍵や暗号鍵データベースの作成から始まります。

この文書では、セキュリティ モジュール データベースの管理について解説します。 証明書データベースや暗号鍵データベースの管理についての情報は、 証明書データベースツールの利用 をご覧ください。

利用環境

このツールは、 Solaris 2.5.1 （SunOS 5.5.1） および Windows NT 4.0 上でビルドされています。

構文

セキュリティ モジュール データベース ツールを起動するには、 次のコマンドを入力してください：

modutil option [arguments]

options および arguments となっているところは、 次のセクションで一覧表示したオプションと引数の組み合わせです。 それぞれのコマンドには一つのオプションが付きます。 それぞれのオプションには、引数が全く付かない場合もありますし、 複数の引数がつく場合もあります。利用法を示す文字列を見たい場合には、 オプションをつけずにコマンドを入力してください。

オプションと引数

オプションは処理 （action） を特定します。オプション引数はその処理を調整します。 modutil に対してのオプションおよび引数は、 次のように定義されています：

オプション

-create

新しい secmod.db、 key3.db、および cert7.db ファイルを作ります。引数 -dbdir directory を利用してディレクトリを指定します。 もし指定したディレクトリに何らかのデータベースがすでに存在する場合には、 セキュリティ モジュール データベース ツールは、エラーメッセージを表示します。

-list [modulename]

secmod.db ファイルの内容について基本的な情報を表示します。 modulename を利用して、特定のモジュール、 そのスロット、そしてトークンについての情報を表示します。

-add modulename

名前付き PKCS #11 モジュールをデータベースに追加します。 このオプションは、引数 -libfile、 -ciphers、 および -mechanisms と共に使用します。

-jar JAR-file

名前付き JAR ファイルを使って、新しい PKCS #11 モジュールをデータベースに追加します。 引数 -installdir および -tempdir と共にこのオプションを利用します。 JAR ファイルは、 Netscape Server PKCS #11 JAR フォーマットを使用し、インストールされる全てのファイル、モジュール名、 メカニズム フラッグ、および暗号フラッグを特定します。 JAR ファイルはまた、 PKCS #11 モジュール ライブラリー ファイルやドキュメンテーションのような他のファイルを含めた、 目的のコンピュータにインストールされるあらゆるファイルを備えていなくてはなりません。 サーバを通したり、セキュリティ モジュール データベース ツールを利用しての （つまり、 JavaScript サポートのない環境での） インストールを行うのに必要な特別なスクリプトについての情報は、 セクション JAR インストレーションファイル をご覧ください。一般的なインストールガイドや、 JavaScript によるサポートが得られる環境 （Netscape Communicator でのような） でのモジュールのインストールについては、 PKCS #11 暗号モジュールのインストールに JAR インストレーションマネジャーを利用する をご覧ください。

-delete modulename

名前付きモジュールを削除します。 Netscape Communicator のインターナル PKCS #11 モジュールは削除できないことにご注意ください。

-changepw tokenname

名前付きトークンのパスワードを変更します。トークンがイニシャライズされていない場合、 このオプションはパスワードをイニシャライズします。引数 -pwfile および -newpwfile と共にこのオプションを利用します。ここで 「パスワード」 とは、 個人識別番号 （PIN） に相当するものです。

-default modulename

名前付きモジュールがディフォルトプロバイダとなるセキュリティ機構を指定します。 セキュリティ機構は、引数 -mechanisms mechanism-list で指定されます。

-undefault modulename

名前付きモジュールがディフォルトプロバイダとならないセキュリティ機構を指定します。 セキュリティ機構は、引数 -mechanisms mechanism-list で指定されます。

-enable modulename

名前付きモジュール上の全てのスロットを利用可能にします。 引数 [-slot slotname] を使って特定のスロットを利用可能にします。

-disable modulename

名前付きモジュール上の全てのスロットを利用不能にします。 引数 [-slot slotname] を使って特定のスロットを利用不能にします。

-fips [true | false]

Netscape Communicator のインターナルモジュールに対しての FIPS-140-1 準拠を開始 （true） または停止 （false） にします。

-force

セキュリティ モジュール データベース ツールのインターアクティブ プロンプトを、 スクリプトで走らせることができるように停止します。 警告に対してそれぞれの予定の操作を手動でテストし、また、 プロンプトをバイパスすることによってセキュリティに欠陥が起きたり、 データベースの保全性が失われないことが確認できた後にのみ、 このオプションを利用してください。

引数

-dbdir directory

セキュリティ モジュール データベース ファイルにアクセスしたり、 作成したりするためのデータベースディレクトリを指定します。 UNIX でのセキュリティ モジュール データベース ツールは、 ユーザーのネットスケープ ディレクトリをディフォルトで指定します。 Windows NT にはディフォルト ディレクトリがないため、 ディレクトリを指定するため -dbdir を使わなければなりません。

-dbprefix prefix

cert7.db および key3.db ファイルで使われるプレフィックスを指定します （例えば、 my_cert7.db や my_key3.db）。このオプションは、特別な場合のために提供されています。 証明書および暗号鍵データベースの名前を変更することはお勧めできません。

-libfile library-file

DLL または他の、データベースに追加されている PKCS #11 インターフェイス モジュールの実装を含むライブラリ ファイルへのパスを指定します。

-ciphers cipher-enable-list

データベースに追加されているモジュール内の特定の暗号を利用可能にします。 cipher-enable-list は、コロンにより区切られた暗号名の一覧です。 スペースが含まれる場合、引用符でこの一覧を囲みます。 現在利用可能な暗号は次のとおりです：FORTEZZA

-mechanisms mechanism-list

特定のモジュールがディフォルトプロバイダとしてフラッグを立てられたセキュリティ機構を指定します。 mechanism-list は、コロンにより区切られた機構名の一覧です。 スペースが含まれる場合、引用符でこの一覧を囲みます。 モジュールは一覧に挙げられた機構のディフォルトプロバイダとなり、 その機構が利用可能になります。複数のモジュールが特定の機構へのディフォルト プロバイダとなっている場合、その機構へのディフォルトプロバイダは未定義のままになります。 次の機構が現在利用可能です： RSA, DSA、 RC2、 RC4、 RC5、 DES、 DH、 FORTEZZA、 SHA1、 MD5、 MD2、 RANDOM （乱数の生成用）、および FRIENDLY （meaning certificate は一般的に読むことができます）。

-installdir root-installation-directory

オプション -jar JAR-file でインストールされるファイルに対して相対的なルート インストール ディレクトリを指定します。 このディレクトリは、ダイナミック ライブラリ ファイルの保存に適切なディレクトリ （例えば、サーバのルート ディレクトリ、または Netscape Communicator のルート ディレクトリ） に対し、一段下のディレクトリでなければなりません。

-tempdir temporary-directory

テンポラリ ディレクトリとは、インストールの過程で、 オプション -jar JAR-file によって作られるテンポラリ ファイルが作られる場所のことです。テンポラリ ディレクトリが指定されていない場合、 カレント ディレクトリが使用されます。

-pwfile old-password-file

オプション -changepw tokenname を使用してパスワードを変更する際、 自動的にパスワードが入力されるように、 トークンの既存のパスワードを含むテキストファイルを指定します。

-newpwfile new-password-file

オプション -changepw tokenname を使用してパスワードが自動的に入力されるように、 新しい、または入れ替えられたパスワードを含むテキストファイルを指定します。

-slot slotname

オプション -enable modulename または -disable modulename で、使用可能／不能にする特定のスロットを指定します。

-nocertdb

証明書データベースまたは暗号鍵データベースを開けないようにして下さい。 開くことによっていくつかの影響があります：

  • コマンド -create を利用すると、 secmod.db ファイルのみが作成されます。 cert7.db および key3.db は作成されません。

  • コマンド -jar を利用すると、 JAR ファイル上の署名はチェックされません。

  • コマンド -changepw を利用すると、 Netscape インターナル モジュールのパスワードが設定または変更できません。 パスワードが key3.db に保存されているからです。

利用例

上で示したようなオプションと引数を使用したセキュリティ モジュール データベース ツールの機能は、次のように区分けすることができます： 四角括弧で囲んだオプションと引数はオプショナルであり、 そうでないものは必ず必要なものです。

  • セキュリティ マネージメント データベース ファイルの一式を作成します （key3.db、 cert7.db、および secmod.db）：
    • -create

  • モジュールの基本的な情報または、そのモジュールの内容についての詳細情報を表示します：
    • -list [modulename]

  • PKCS #11 モジュールを追加します。このモジュールには、 サポート ライブラリ ファイルの設定が含まれており、暗号を利用可能にし、 そして様々なセキュリティ機構に対してのディフォルト プロバイダの状態を設定します：
    • -add modulename -libfile library-file [-ciphers cipher-enable-list]
    [-mechanisms     mechanism-list]

  • 既存の JAR ファイルから PKCS #11 モジュールを追加します：
    • -jar JAR-file -installdir root-installation-directory
    [-tempdir     temporary-directory]

  • セキュリティ モジュール データベースから特定の PKCS #11 モジュールを削除します：
    • -delete modulename

  • トークンのパスワードを初期化または変更します：
    • -changepw tokenname [-pwfile old-password-file]
    [-newpwfile     new-password-file]

  • 既存の PKCS #11 モジュール内の様々なセキュリティ機構に対するディフォルト プロバイダの状態を設定します：
    • -default modulename -mechanisms mechanism-list

  • 既存の PKCS #11 モジュール内の様々なセキュリティ機構に対するディフォルト プロバイダの状態をクリアします：
    • -undefault modulename -mechanisms mechanism-list

  • モジュール内の特定のスロット、または全てのスロットを利用可能にします：
    • -enable modulename [-slot slotname]

  • モジュール内の特定のスロット、または全てのスロットを利用不能にします：
    • -disable modulename [-slot slotname]

  • Netscape Communicator のインターナル モジュール内の FIPS-140-1 準拠を開始／停止します：
    • -fips [true | false]

  • スクリプト操作をサポートするため、 セキュリティ モジュール データベース ツールに対してインターアクティブ プロンプトを停止します：
    • -force

JAR インストレーション ファイル

JAR ファイルがサーバによって、セキュリティ モジュール データベース ツールによって、 または JavaScript スクリプトを理解しないプログラムによって動かされた場合、 特別な情報ファイル （special information file） が次に示された形式の中に含まれなければなりません。

この情報ファイルは、特別なスクリプトを含んでおり、 JAR アーカイブの目録ファイル （manifest file） で宣言されなければなりません。 スクリプトにはどのような名前を付けても構いません。 これに対するメタインフォ タグ （metainfo tag） は、 Pkcs11_install_script です。目録ファイルのメタ情報を宣言するには、 Netscape Signing Tool に通じたファイル内に目録ファイルをおきます。

サンプル スクリプト

例えば、 PKCS #11 インストール スクリプトは、 pk11install 内にあるとします。この場合、 Netscape Signing Tool のメタインフォ ファイルは、 次のようなラインを含んでいるでしょう： 

+ Pkcs11_install_script: pk11install

サンプル スクリプト ファイルには次のものを含むことができます： 


ForwardCompatible { IRIX:6.2:mips SUNOS:5.5.1:sparc }
Platforms {
   WINNT::x86 {
      ModuleName { "Fortezza Module" }
      ModuleFile { win32/fort32.dll }
      DefaultMechanismFlags{0x0001}
      DefaultCipherFlags{0x0001}
      Files {
         win32/setup.exe {
            Executable
            RelativePath { %temp%/setup.exe }
         }
         win32/setup.hlp {
            RelativePath { %temp%/setup.hlp }
         }
         win32/setup.cab {
            RelativePath { %temp%/setup.cab }
         }
      }
   }
   WIN95::x86 {
      EquivalentPlatform {WINNT::x86}
   }
   SUNOS:5.5.1:sparc {
      ModuleName { "Fortezza UNIX Module" }
      ModuleFile { unix/fort.so }
      DefaultMechanismFlags{0x0001}
      CipherEnableFlags{0x0001}
      Files {
         unix/fort.so {
            RelativePath{%root%/lib/fort.so}
            AbsolutePath{/usr/local/netscape/lib/fort.so}
            FilePermissions{555}
         }
         xplat/instr.html {
            RelativePath{%root%/docs/inst.html}
            AbsolutePath{/usr/local/netscape/docs/inst.html}
            FilePermissions{555}
         }
      }
   }
   IRIX:6.2:mips {
      EquivalentPlatform { SUNOS:5.5.1:sparc }
   }
}

スクリプトの文法

スクリプト ファイルの文法は次のようになります： 

--> valuelist
valuelist --> 
value valuelist

               <null>
value ---> 
key_value_pair
            string
key_value_pair --> key { valuelist }
key --> string
string --> simple_string
            "complex_string"
simple_string --> [^ \t\n\""{""}"]+ 
（空白、引用符、中括弧は入れないでください）
complex_string --> ([^\"\\\r\n]|(\\\")|(\\\\))+ 
（引用やバックスラッシュはバックスラッシュを使ってエスケープしなければなりません。
複合文字列では、新しい行を加えたり、改行をしてはいけません。）

複合文字列以外では、全ての空白スペース （例えば、スペース、タブ、および改行） は、 同じものとしてみなされ、区切りトークン （delimit tokens） に対してのみ使用されます。

キー

キーでは、大文字小文字の区別はありません。 このセクションでは、次のキーについて解説します：

グローバル キー （Global Keys）
パー プラットフォーム キー （Per-Platform Keys）
パー ファイル キー （Per-File Keys）

グローバル キー （Global Keys）

ForwardCompatible

前方互換 （forward compatible） を持つプラットフォームの一覧を提供します。 使用中のプラットフォームがサポートされるプラットフォームの一覧の中にない場合、 ForwardCompatible の一覧の中から、同じ OS やアーキテクチャの過去のバージョンを探し出そうとします。見つかった場合は、 その属性が使用中のプラットフォームに利用されます。

Platforms （必須）

プラットフォームの一覧を提供します。一覧のそれぞれのエントリーは、 それ自体がキー値ペア （key-value pair） です： キーはプラットフォームの名前およびプラットフォームの様々な属性を含む値一覧です。 属性 EquivalentPlatform が指定されていない場合、 ModuleName、 ModuleFile、および Files 属性が指定されていなければなりません。プラットフォーム文字列は次の形式です： system name:OS release:architecture。 インストーラは、これらの値を NSPR から取得します。非 UNIX 系 OS では、 OS release の欄は空白です。 NSPR では、 次のシステム名およびプラットフォームが定義されています：

  • AIX (rs6000)

  • BSDI (x86)

  • FREEBSD (x86)

  • HPUX (hppa1.1)

  • IRIX (mips)

  • LINUX (ppc, alpha, x86)

  • MacOS (PowerPC)

  • NCR (x86)

  • NEC (mips)

  • OS2 (x86)

  • OSF (alpha)

  • ReliantUNIX (mips)

  • SCO (x86)

  • SOLARIS (sparc)

  • SONY (mips)

  • SUNOS (sparc)

  • UnixWare (x86)

  • WIN16 (x86)

  • WIN95 (x86)

  • WINNT (x86)

妥当なプラットフォーム文字列の例はこのようになります： 


IRIX:6.2:mips
SUNOS:5.5.1:sparc
Linux:2.0.32:x86
WIN95::x86.

パー プラットフォーム キー （Per-Platform Keys）

これらのキーは、 Platforms リスト内のエントリーの値一覧内でのみ意味を持ちます。

ModuleName （必須）

モジュールに対し共通名を与えます。この名前は、 Netscape Communicator、 セキュリティ モジュール データベース ツール （modutil）、サーバー、または Netscape セキュリティ モジュール データベースを利用するあらゆるプログラムを参照する際に利用されます。

ModuleFile （必須）

このプラットフォーム用に PKCS #11 モジュール ファイル （DLL または .so） に名前を付けます。 名前は、 JAR アーカイブ内のファイルの相対パスとして与えられます。

Files （必須）

このモジュールにインストールする必要のあるファイルを一覧表示します。 ファイルリスト内のそれぞれのエントリーは、キー値ペア （key-value pair） です： キーは、 JAR アーカイブ内のファイルのパスで、 値一覧にはそのファイルの属性が含まれます。それぞれのファイルに対し、 少なくとも RelativePath または AbsolutePath が指定されていなければなりません。

DefaultMechanismFlags

このモジュールがディフォルト プロバイダとなる機構を指定します。 このキー値ペアは、 16 進数 （0x） 形式のビット ストリングです。 次に挙げる定数のビット単位の OR として構築されいます。 DefaultMechanismFlags エントリーが省略されている場合の値は、 ディフォルトで 0x0 となります。 


   RSA:                   0x00000001
   DSA:                   0x00000002
   RC2:                   0x00000004
   RC4:                   0x00000008
   DES:                   0x00000010
   DH:                    0x00000020
   FORTEZZA:              0x00000040
   RC5:                   0x00000080
   SHA1:                  0x00000100
   MD5:                   0x00000200
   MD2:                   0x00000400
   RANDOM:                0x08000000
   FRIENDLY:              0x10000000
   OWN_PW_DEFAULTS:       0x20000000
   DISABLE:               0x40000000

CipherEnableFlags

このモジュールが提供し、 Netscape Communicator が提供しない暗号を Communicator が利用できるように、指定します。 このキーは、 16 進数 （0x） 形式のビットストリングです。 次に挙げる定数のビット単位の OR として構築されいます。 CipherEnableFlags エントリーが省略されている場合の値は、 ディフォルトで 0x0 となります。 


   FORTEZZA:               0x0000 0001

EquivalentPlatform

名前付きプラットフォームが現在のプラットフォームで利用されるように指定します。 同じ設定を使用しているプラットフォームが複数ある場合は、タイピング （typing） を保存します。

パー ファイル キー （Per-File Keys）

これらのキーは、 Files リスト内のエントリーの値一覧内でのみ意味を持ちます。 少なくとも RelativePath および AbsolutePath のいずれかが指定されていなければなりません。この両方が指定されている場合、 相対パスが始めに試され、インストーラ プログラムによって相対ルートパスが提供されていない場合に限り、 絶対パスが使用されます。

RelativePath

インストール時に決定される、 ディレクトリに相対的なファイルの宛先ディレクトリを指定します。 二種類の変数が相対パスで利用できます： 「%root%」 および 「%temp%」 です。 「%root%」 は、 ファイルがインストールされるべきディレクトリに相対的なディレクトリに、 実行される際に、入れ替えられます。例えば、サーバーのルートディレクトリや、 Netscape Communicator ルートディレクトリです。 「%temp%」 は、 インストールの開始時に作られるディレクトリで、インストール終了時に削除されます。

「%temp%」 は、実行ファイル （セットアップ プログラムのような） や、そうしたプログラムが使用するファイルを置くために作られます。 例えば、 Windows インストレーションは、 setup.exe インストレーション プログラム、ヘルプファイル、そして圧縮された情報を収めた .cab ファイルが必要になるでしょう。これら全てのファイルが、 テンポラリーディレクトリに置かれることがあります。 テンポラリーディレクトリに置かれるファイルは、必ず、 実行ファイルが実行される前にそこに置かれます。こうしたファイルは、 実行ファイルが終了するまで削除されることはありません。

AbsolutePath

絶対パスで、ファイルの宛先ディレクトリを指定します。RelativePath および AbsolutePath の両方が指定されている場合、 インストーラは相対パスを利用しようとします。そして相対パスを決定できない場合は、 絶対パスを利用します。

Executable

インストール時に実行されるファイルを指定します。 一般的にこのストリングは、自己解凍式の setup.exe のようなモジュールベンダーによって提供されたセットアップ プログラムが使われます。複数のファイルを実行ファイルとして指定することができますが、 この場合、スクリプトファイル内で指定された順に実行されます。

FilePermissions

UNIX の標準的なフォーマットに従い、 8 進数字の文字列として解釈されます。 この文字列は、次に示した定数のビット単位 OR です： 


   user read:                0400
   user write:               0200
   user execute:             0100
   group read:               0040
   group write:              0020
   group execute:            0010
   other read:               0004
   other write:              0002
   other execute:            0001

プラットフォームの中には、こうしたパーミッションを理解しないものもあります。 これらパーミッションは、プラットフォームに意味を持つ範囲でのみ適用されます。 この属性が省略されている場合、 777 がディフォルトとして仮定されます。

   WinNT::x86 {

      ModuleFile { crypto.dll } 
      DefaultMechanismFlags{0x0000} 
      CipherEnableFlags{0x0000} 
      Files { 
         crypto.dll { 
            RelativePath{ %root%/system32/crypto.dll } 
         }

            Executable 
            RelativePath{ %temp%/setup.exe } 
         } 
      } 
   }

   } 
}
