Mozilla Thunderbird 0.4

リリースノート

Mozilla Thunderbird メールクライアントとは、統合された Mozilla アプリ - スイートのメールコンポーネントを 再デザインしたものです。私たちの目的は、Mozilla メールコンポーネントに今ある機能の多くを活用し、シンプルで拡張性のある単体のメールアプリケーションを作成することです。Mozilla Thunderbird プロジェクトのウェブサイトにはさらに多くの情報があります。

Mozilla アプリ - スイートの基礎的なコードが成熟しているおかげで Mozilla Thunderbird はかなり使いものになります；しかしながら、このリリースはまだプレビューリリースであり、従って欠陥があることも十分考えられます。こうした 欠陥を見つけるのに役立てるため、開発者たちは喜んでみなさんのフィードバックを受け付けています（ MozillaZine が親切にもホストしてくれている Mozilla Thunderbird forums へのコメントか Bugzilla にバグを報告してください）。どんなバグでも Bugzilla にバグを報告する前にはこのリリースノートを十分に読んでください。

この四番目のマイルストーンリリースでは新しい機能の開発に焦点が当てられていて、最近リリースされた Mozilla 1.6 ベータアプリケーションスイートから作られています。さらにまだ多くの作 業を必要としていますが、開発者たちはこの最近の進展にわくわくし、コミュニティのみなさんとこの最新の作業成果を分かち合いたいと切望しています。では 楽しんで！

最新情報

以下のリストは主に進展した箇所の要約です。詳細は変更履歴（訳 者 注：日本語版 は現在更新していません）を参照してください。
  • 外観の改善

    Mozilla Thunderbird のデフォルトテーマが完成に大きく近づきました。このテーマの内約９０％が完成し，"classic" なアイコンは２，３残すのみとなりました。

    メッセージリストペインで返信，転送そして未読のステータスが表示されるようになりました。このステータスはテキストを使用する代わりに、 メッ セージエンベロープのアイコン上のアイコンとして直接表示されます。

    メッセージリストペインに、メッセージに添付された書類用のソート可能な独立したコラムができました。

    Windows のアニメアラートのルック・アンド・フィールが新しくなりました。サーバのユーザ名ではなくアカウント名を表示します。

  • OS との相互運用性の改善

    Linux: Thunderbird がブラウザ URL をデフォルトのブラウザに正しく渡せるようになりました。これを行う方法についてはこちらを 御覧下さい。

    Mac OSX: Thunderbird に Windows メニューがつきました。これによって Thunderbird のウインドウを開く操作が簡単になります。開いているすべてのウインドウを閉じたあとのメニューバーのふるまいが正常になりました。Mozilla Firebird と Thunderbird 間で URL を受け渡しする場合の干渉がなくなり，この二つのアプリケーションを同時に使えるようになりました。

    Windows：Thunderbird をデフォルトのメールクライアント/ニュースクライアント（両方もしくは片方）に設定できるようになりました。

  • 新機能

    Windows のユーザは、 Windows のクリップボードから HTML メール作成ウインドウに（スクリーンショットを含む）画像をコピーアンドペーストすることができます。

    プレビューウインドウで開いたメッセージを既読にするまでの時間（秒）を設定することができます。

    ツールバーをお好みにカスタマイズするオプションが増えました。メインのウインドウでの "Mark" ボタンとメール作成ウインドウでの copy/cut/paste アイコンです。

    Netscape 4.xからのプロファイルの移行が大きく改善されました。

    Thunderbird 0.4 用の Palm Sync Address book conduit 機能拡張が利用可能になりました。

    POP アカウントがサーバ上での経時処理をサポートするようになりました。

  • 最近修正されたバグ

    受信したメッセージがジャンクだった場合新規メールの通知アラートを表示しないようにしました。

    新規メールやメッセージ本体をダウンロードするような操作の進行状況を表示する際に，メール３ペインウインドウ内にプログレスバーが表示さ れる ようになりました。

    自動プロキシ URL が作動するようになりました。

    POP アカウント："Click here to download the rest of the message（残りのメッセージをダウンロードするにはここをクリック）"が作動するようになりました。

    POP アカウント：pop3 サーバからメッセージを削除する新しいフィルタ動作が加わりました。これはデフォルト設定でメッセージがサーバに残されている場合のものです。

    ジャンクフォルダの消去が、最初のアカウントだけではなくすべてのアカウントで作動するようになりました。

    サーバがセキュア認証をサポートしていないのにメールサーバでセキュア認証を使用するように Thunderbird で設定した場合， 何も警告せずにセキュアではない認証に戻ってしまうことがなくなりました。エラーダイアログを表示し，そのサーバへの接続を拒否します。

    News でのエラーダイアログがアプリケーションをハングさせることがなくなりました。

    メッセージ内の HTML アンカーが作動するようになりました。

    スレッドが日付けでソートされている場合，そのスレッド内の新しいメッセージからソートします。

ダウンロード、インストールとアンインストール

初 めてインストールする場合：このリリースには正式なインストーラはありません。下記のインストールの説明は、単一の 圧縮されたプログラムファイルをあなたがダウンロードすることを想定して書かれています。このプログラムファイルはハードディスクに手動で展開しなくては なりません。このようなインストール方法は、難しくはありませんが展 開するためのユーティリティを必要とします。（詳しい使い方はそのユーティリティのヘ ルプを参照してください）

下記にリストされている提供されたビルドの中には、インストールのプロセスを簡単にするかもしれない自己解凍型やインストーラとしてパッケージされたもの があるかもしれません。もう一つの方法としては、mozdev.org にある Thunderbird Installer を試してみてもよいでしょう。

アッ プグレードする場合： 前のバージョンのプ ログラムファイルを含むディレクトリに Mozilla Thunderbird をインストールしないでください。前のバージョンに上書きすると問題を引き起こします。以前にインストールしたディレクトリを再利用するには、 そのディレクトリを削除して再度作り直すか、空にするか、移動するか、あるいは違う名前を付けます。この手順を無視した場合には絶対に Bugzilla にバグを登録しないでください。

このプログラム自体のディレクトリにはプロファイル情報は含まれていません。従って新たにインストールしても、今あるアカウント、アカウントの設定、オプ ション、メール、そしてニュースメッセージが損なわれることはありません。このリリースではプロファイルを変更しなくても正しく作動します。重要： Thunderbird の前のバージョンを使用していて、テーマもしくは機能拡張をインストールしている場合は，以下の手順に従ってください。さもないと Thunderbird は正しく動作しません。まず プロファイル の場所（和訳）を 見つけてください。"chrome" と名付けられたサブディレクトリがあるはずです。そして "chrome" 内のすべてを削除します。この操作がメールのデータや設定に影響を与えることはありません。

インストール：
注意：Mozilla Thunderbird は専用のユーザプロファイル情報を作成します。もし無くても Mozilla アプリ - スイートのプロファイルとは干渉しません。

ステップ１：下記にあるリンクから自分のオペレーティングシステムと環境にあったものをクリックし、バイナリをダウンロードします。
ステップ２a：アップグレードの場合はインストールするディレクトリを準備するか、新しいディレクトリを使用するようにしてください（上記の 黄色いアップグ レードの部分を 参照してください）。
ステップ２b：展 開するためのユーティリティを使用してハードディスクの空のディレクトリにファイルを展開します。
ステップ３：展開したディレクトリに移動し、 thunderbird.exe を実行します。
ステップ４：必要であれば、素早くアクセスするためにショートカットやシンボリックリンクを作成します。

アンインストール：
注意：ステップ２を実行しなければプロファイルを残しておくことができます。これ は将来再び Mozilla Thunderbird をインストールして使用する際に利用ですることができます。

ステップ１：Mozilla Thunderbird のファイルを展開したプログラムディレクトリを削除します。
ステップ２：Mozilla Thunderbird のプロファイルディレクトリを削除します。このステップを実行すると、プロファイルにローカルで保存されている全てのメールが削除されます。
ステップ３：作成したショートカットやシンボリックリンクを削除します。

既知の問題

以下は Mozilla Thunderbird 0.4 で判明している主な問題のリストです。これらの問題に関しては Bugzilla にバグとして報告しないでください。また、新たな問題に関してバグを報告する前に、このリストで照会してください（このリストは更新されます）。

  • *重要*： Thunderbird の前のバージョンを使用していて、テーマもしくは機能拡張をインストールしている場合は，以下の手順に従ってください。さもないと Thunderbird は正しく動作しません。まず プロファイル の場所（和訳）を 見つけてください。"chrome" と名付けられたサブディレクトリがあるはずです。そして "chrome" 内のすべてを削除します。この操作がメールのデータや設定に影響を与えることはありません。その後，0.4 と互換性のある使用したいテーマ/機能拡張の バージョンを再びインストールします。
  • デフォルトの辞書は英語のみです。英語以外の辞書を追加するにはイン ストールの説明を参照してください。
  • 学習機能つきジャンクメールフィルタを有効/無効にするオプションが全てのアカウントに適用できるように見えます (Tools / Junk Mail Controls / Adaptive Filters)。しかしこれはアカウントごとのオプションです。特 定のアカウントにこのオプションを設定するには、'Settings' パネルの 'Account:' ドロップダウンからそのアカウントを選択してか ら 'Adaptive Filters' パネルに移り、このオプションを設定します。そして必要なだけ各アカウントに対してこの作業を繰り返します。
  • 現時点では Mozilla プロファイルを Thunderbird に移行するようにはなっていません。Mozilla プロファイルを手動で（日本語版）移 行することもできますが、私たちとしてはおすすめしません。たとえ Thunderbird をプレビューとして試用してみたい場合でも新しいプロファイ ルで使用することを強くすすめます（そのプロファイルは後のリリースをプレビューする時に再び使用することが出来ます）。
  • 入力自動補完ドロップダウンフィールドのスクロールバーをクリックしてもスクロールできません。
  • Tools / Options / Attachments / で Attachment フォルダ設定が保存できません。
  • 何人かのユーザから、突然どのアカウントでもメッセージの送受信ができなくなるとの報告がありました。もし同じ状況に見舞われたら，ユーザ prefs.js ファイル（和訳）に 次の値が設定されているか確認して下さい：
    "user_pref("network.online", false);" もしこれが設定されていたらこれを削除して，これがおこった状況をフォーラムに報告してください。
  • デフォルトのテーマほぼ完成しつつありますが、まだいくつかのクラシックテーマが残っています。新しいテーマへの移行が済んでいないことに関 するバグは報告しないでください。
  • Linux：Mark All Read のキーボードショートカットが動きません。
  • Mac OS X: Thunderbird は他のアプリケーションから mailto/news URL を開くことができません。
  • Mac OS X: 特定の文字コード（韓国語等）での添付書類名が正しくレンダリングされないことがあります。
  • mailnews.reply_header_authorwrote 設定は HTML を受け付けなくなりました。これはユニコードの設定ですので、各国の文字を入力するには UTF-8 エディタを使用することができます。
  • インラインイメージ入りのメッセージを閲覧する場合に，コンテキストメニュー項目の "View Image" を選択すると小さな空のウインドウが表示されてしまいます。
