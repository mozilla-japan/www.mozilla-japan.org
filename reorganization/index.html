2005年 8月 3日、非営利ソフトウェア開発団体である Mozilla Foundation は、全額出資子会社である Mozilla Corporation を創設しました。Mozilla Corporation は課税対象となる子会社で、親会社である Mozilla Foundation が掲げる非営利で公共に資するという目標達成のために活動し、Mozilla 製品の製品開発やマーケティング、製品配布に責任を負います。

この組織再編の最新情報を追跡するには、RSS フィード 【訳注：英語による発信】 を購読してください。

全般的な質問

今日 Mozilla Foundation がアナウンスしたのは何についてですか？
Mozilla Foundation は、Mozilla オープンソースコードをベースにした Firefox、Thunderbird および関連ブランドの製品開発および配布を行うために、新しく課税対象となる全額出資子会社である Mozilla Corporation を設立します。
Mozilla Corporation の役割は何ですか？
Mozilla Corporation は、Mozilla オープンソースコードをベースにした Firefox、Thunderbird、および関連ブランド製品の製品化および配布に責任を負います。Mozilla Corporation の使命は Mozilla Foundation と同じく、インターネットに選択肢と新機軸を与えることです。
Mozilla Foundation はなぜ今回のような変更を決断したのでしょう？
Mozilla Corporation は、市場シェアを延ばしつつある Firefox の経済的な価値をてこに、Mozilla Foundation の使命である、インターネットでの選択肢の確保および新機軸を与えることです。営業子会社を設立することで、新組織による収益事業から、さまざまな Mozilla オープンソーステクノロジの開発、検証、製品化を支援するために資金を提供することができます。これにより、Firefox や Thunderbird のエンドユーザならびに開発者、そして Mozilla オープンソースコードベースをさまざまな方法で利用していと考えている人々すべてに資することができます。また、Mozilla Corporation にこうした製品に関連する収益事業を任せてしまうことで、Mozilla Foundation は税金控除の対象団体という地位を維持したまま団体の使命を達成することができるようになります。

ただし、Mozilla Corporation は一般的な収益企業ではなく、エンドユーザに可能な限り最高の体験を提供するという目的に合致した収益事業のみを担当します。

Mozilla Corporation の従業員数、そして新組織はどのように収益をあげて発展していくのでしょうか？
Mozilla Corporation はおよそ 36 名の従業員を擁することになります。Mozilla Corporation は一般的な収益企業とは異なり、ストックオプションや従業員に対するその他一般的な利益還元は行いません。
Mozilla Corporation はどこに本拠を構えるのでしょうか？
Mozilla Corporation は、カリフォルニア州マウンテンビューにある Mozilla Foundation と同じ住所に本拠を構えます。
Mozilla Corporation の業務開始はいつですか？
Mozilla Corporation はすぐに業務を開始します。
Mozilla Corporation の経営陣はどのようなものですか？
Mitchell Baker が Mozilla Corporation の代表取締役となりました。Mozilla プロジェクトの共同創設者であり長年にわたって技術面でリーダーシップを取ってきた Brendan Eich が Mozilla Corporation の技術主任に就任しました。
Mozilla Corporation の取締役には、Mitchell Baker、Christopher Blizzard、そして LinkedIn の CEO である Reid Hoffman が就任しました。Mitchell Baker が Mozilla Corporation の代表取締役です。
誰が Mozilla Corporation に異動するのですか？
Mozilla Foundation の現在の従業員のほとんどが Mozilla Corporation に異動します。Mozilla Corporation の従業員および職務の完全な一覧は Mozilla Corporation の Web サイトで見ることができます。
Mozilla Corporation からの収益 （またはその一部） は Mozilla Foundation がさらに目標を達成するための資金として還元されるのでしょうか？
Mozilla Foundation への慈善寄付であれ Mozilla Corporation による収益であれ、Mozilla Foundation および Mozilla Corporation によるすべての収入は、それが Mozilla Foundation または Mozilla Corporation によって管理されているかに関わらず、Foundation のさらなる使命遂行のために使用されます。
この件について最終決定を下したのは誰ですか？
Mozilla Foundation の理事会が Mozilla Corporation を設立するという決断を下しました。

組織について

これからの Mozilla Foundation の使命は何ですか？
Mozilla Foundation の使命はこれまでと変わることはありません。つまり、インターネットでの選択肢と新機軸の普及に努めることです。この使命は Mozilla Corporation の使命でもあります。
これは Mozilla Foundation の従業員にとってはどのような影響があるのでしょう？
ブランド製品 （Firefox および Thunderbird） およびその製品の基礎を成しているコード （例：Gecko レイアウトエンジン） の開発支援を主な職務としている Mozilla Foundation の従業員は、Mozilla Corporation へと異動します。Mozilla Foundation は Mozilla オープンソースプロジェクトの管理を続行し、Foundation の従業員は、プロジェクト運営、プロジェクトの広報およびプロジェクト全体の将来における成功を確実なものとするための活動に責任を持ちます。
誰が Mozilla Foundation に残り、活動を率いていくのでしょうか？
Mozilla Foundation の理事には、Mitch Kapor、Brian Behlendorf、Mitchell Baker、Brendan Eich、そして伊藤穣一が就任しました。Mozilla Foundation の従業員は、Mozilla Corporation とは異なり、活動方針の責任者として Frank Hecker が、加えて Gervase Markham および Zak Greant が参加しています。Mozilla Foundation の従業員とその職務についての完全な一覧は、近く Mozilla Foundation の Web サイトに掲載されます。
コードの開発・管理における Mozilla Foundation の役割はどうなるのでしょう？
Mozilla Foundation の第一の役割は、Mozilla オープンソースプロジェクトの管理、監督、促進です。Mozilla Foundation は Mozilla コードの開発には責任を負いません （ただし、Foundation の従業員の中にはパートタイムベースでそうした業務を行う者が出てくる可能性もあります）。こうした責務は、Mozilla Corporation を含む、Mozilla の開発に参加する他の個人や団体に任されます。
現在 Mozilla Foundation は何人の従業員を抱えているのですか？
Mozilla Foundation は現在 3 名の従業員を抱えています。
誰が Mozilla Corporation を運営し、誰が Mozilla Foundation を運営するのでしょうか？　またその理由は？
Mozilla Corporation は Mitchell Baker が率います。 Mitchell Baker は、長年にわたって Mozilla プロジェクトの「トカゲの世話役主任」を務めてきましたし、過去 2 年間は Mozilla Foundation の理事長を務めてきました。Mitchell は、Web に変革をもたらすために Mozilla Firefox といったエンドユーザ向けの製品の普及に多大な貢献をしてきましたので、自然と Mozilla Corporation のリーダーに選ばれました。プロジェクトの技術方針は、これまでと同様に、Brendan Eich が率いる貢献者による共同作業を経て決定されていく予定です （Brendan Eich は Mozilla Corporation に異動する予定です）。
Mozilla Foundation の活動は Frank Hecker が管理します。Frank は Mozilla プロジェクトが公式に発足する前からプロジェクトに関わってきました。彼は、Netscape Communicator のためのソースコードをリリースするという決断に貢献した、Netscape の中心的な人物でもあります。またそれ以来、彼は、Mozilla プロジェクトの管理組織である mozilla.org のスタッフを務めてきました。彼は長年にわたり、これからの Mozilla Foundation の主要なターゲットである、Mozilla プロジェクトの活動方針や運営問題に深く関わってきました。
Mozilla Corporation はどのようにして Mozilla Foundation と連携するのでしょうか？　指揮系統はどのようなものになりますか？
運営面全体では、Mozilla Corporation は独立した取締役会を持ちます。Mozilla Corporation の取締役は Mozilla Foundation の理事会によって任命されます。日々のプロジェクト関連の活動面では、Mozilla Corporation の従業員は、mozilla.org スタッフ、driver、モジュールオーナーなど、Mozilla Foundation によって確立された組織構造を通して、Mozilla Foundation および他の Mozilla プロジェクトの参加者と連携して活動します。（これらのグループのメンバーシップはあらゆるプロジェクト参加者に対して開かれている点に留意してください。）

製品について

Mozilla Foundation と Mozilla Corporation からというように、複数の Firefox が市場に登場することになるのですか？
Mozilla Foundation は独自の Firefox や Thunderbird をリリースすることはなく、こうした業務を Mozilla Corporation に任せます。ただし、Mozilla の商標ポリシーに準拠していれば、他の人が Firefox および Thunderbird をリリースすることは可能です。また、独自のブランドで Mozilla をベースにした製品を配布することも自由にできます。
Firefox のためにユーザは支払いが必要になるのでしょうか？
いいえ。Firefox および Thunderbird は、オープンソースコードをベースにした製品として今日ある状態のままです （つまり、無料です）。
Mozilla Corporation が Firefox の独占的な配布者になるのでしょうか？
いいえ。他の人も、現在運用されているのと同一の条件下で Firefox や Thunderbird を配布することができます。詳しくは、Mozilla 商標ポリシーをご覧ください。
この変更は Firefox のセキュリティにどのような影響を与えるのでしょうか？
Firefox や Thunderbird のセキュリティ問題は、これまでと同様の方法で処理されます。Firefox や Thunderbird のセキュリティ問題となり得る報告は、既定の方針に沿って Mozilla セキュリティグループによって処理され、新しいセキュリティ関連の機能の設計・開発は、Firefox および Thunderbird 内のさまざまな分野を担当する開発者のリーダーらによって管理されます。Mozilla セキュリティグループおよび開発者のリーダー （モジュールオーナー） は、個人のボランティア参加者や団体参加者の代表者、そして Mozilla Corporation および （セキュリティグループとしての） Mozilla Foundation で構成されます。

コミュニティについて

Mozilla Corporation においてコミュニティの役割とは何ですか？
コミュニティは、mozilla.rog スタッフといった Mozilla プロジェクトの既存の管理機構を通して Mozilla Corporation およびその従業員と共に作業を進めます。これまで Mozilla Foundation がコミュニティと連携してきたのと同じ要領で活動すると考えてください。
Mozilla Foundation においてコミュニティはどのような役割を果たしていくのでしょうか？
Mozilla Foundation の主要目的は、Mozilla プロジェクトに興味を持っているコミュニティメンバーと直接やり取りをし、将来の方向付けを行うことです。このプロセスの一部として、Mozilla Foundation はコミュニティと共同で、mozilla.org スタッフの役割を含め、Mozilla プロジェクトの管理機構の見直しおよび再構成を行う計画です。Mozilla Foundation そのものの正当な統轄機構は、これまでどおり、Mozilla Foundation 理事会が行います。
これからの Firefox にコミュニティはどれだけの貢献ができるのでしょうか？
コミュニティは今日行っているのとまったく同じ方法で、Firefox や Thunderbird の基礎を成している Mozilla オープンソースコードベースにコードを提供したり、Firefox や Thunderbird のビルドをテストしたり、Firefox や Thunderbird にあるバグを報告・修正したり、Firefox や Thunderbird の利用を宣伝したりして、Firefox や Thunderbird に貢献することができます。コミュニティのメンバーはまた、Mozilla Foundation に対して税控除の対象となる寄付を行うことができます （法律で定められている限り）。こうした寄付は、Mozilla プロジェクトや関連プロジェクトおよびテクノロジを支え、促進するために使用されます。
コミュニティが製品を生み出し・改良することに対して Mozilla Foundation はどのように報いるのでしょうか？
コミュニティに還元できるものとしては、これまで Mozilla プロジェクトや他のオープンソースプロジェクトでのものと同じです。個人のボランティアは多くの場合、「自分の痒いところをかく」ためにプロジェクトに参加します。例えば、自分の環境に影響のあるバグを報告したり、（場合によっては） 修正したりします。あるいは、新しい Mozilla コードや、Firefox や Thunderbird のための新しい拡張機能を開発して、みずからのニーズに合った機能を整備してもよいでしょう。そして、時が経つにつれて、プロジェクトの参加者はより本格的に Mozilla の開発に関わるようになったり、自ら Mozilla 関連の事業を興したりすることもできます。Mozilla Corporation の設立にあたっての私たちの目標の一つは、Mozilla プロジェクトや Firefox / Thunderbird プロジェクトをさらなる成功に導くことです。これが、さらにプロジェクトを取り巻く収益エコシステムを強化し、Mozilla 開発者にさらなるチャンスを提供することができるのです。
Mozilla 開発者の生活にはどのような変化がおきるのでしょうか？
Mozilla コードベースの開発を推し進めているメカニズムはこれまでと変わりありません。モジュールオーナー、driver、レビューア、そしてスーパーレビューアは、全員これまでどおりの権限を持ちますし、既存のプロセスやポリシーもそのまま運用されます。
