Mozilla Wishlist FAQ

この FAQ （よくある質問）には、wishlist メーリングリストまたは ニュースグループで頻繁にリクエストされている情報を掲載しています。このページは Robert Mohr によって管理されています。追加して欲しい項目があれば、メーリングリストかニュースグループに投稿してください。

Mozilla Firefox についてリクエストがある場合は、FAQ 【訳注：日本語版】 や ヒント & トリック 【訳注：日本語版】 を読んでからにしてください。Mozilla Thunderbird についても FAQ 【訳注：日本語版】 と ヒント & トリック 【訳注：日本語版】 のページがあります。

1 お願いする

Mozilla をより良くする上で提案がありますか？enhancement バグに報告するか、Wishlist フォーラムに投稿して、あなたの大きな夢を語ってください。下の頻繁にリクエストされている機能バグともっとも頻繁に報告されるバグのリスト確認して、重複を避けてください。

1.1 Mozilla 開発者からの返事がないようだけれど、私の投稿には重要性がないの？

あらゆるユーザーフィードバックに感謝しています。しかしながら、ユーザーからの投稿が膨大なものですから、開発者はすべての質問には応えられないのです。辛抱強く、あなたの質問に応えてくれるのを待ちましょう。もし問題にしかるべき注意が払われていないと感じた場合には、その問題についてコメントを沿えるか、誰かがその問題について Bugzilla 【訳注：日本語版】 に報告していたら、そのバグ報告に投票してください。

この文書やほかのフォーラム、または Bugzilla での無作法はやめてください。Bugzilla エチケットガイドラインやフォーラムルールを参照してください。

1.2 バグはいつ修正されるの？

たくさんの注目を集めたバグは、通常ステータスアップデートで取り上げられます。特に関心を持っているバグがあれば、自分をバグの CC リストに加えてください。あらゆるコメントやバグ報告への変更を、通知メールとして受け取ることができます。

1.3 拡張機能については？

ある特定の機能が Mozilla ビルドに実装されないかとじりじりして待っている方は、ExtensionRoom でさまざまな Mozilla アドオンを試すことができます。

2 頻繁にリクエストされている機能

これは、Wishlist フォーラムで頻繁にリクエストされている機能のリストです。もし最新の Mozilla にまだこれらの機能が入っていないなら、それはたいていの場合、誰もその機能を実装する作業に着手していないからです！ あなたも Mozilla に 参加 して、機能の追加に協力してみませんか？

2.1 アプリケーション

カスタマイズ可能なツールバー

現在、Mozilla は完全にカスタマイズ可能なツールバーを実装していません（Mozilla Firefox では利用可能です）。この機能へのリクエストは、 bugs 15144, 61710, 22056, 86455, 48926, and 51129 を参照してください。

簡易設定

画像や JavaScript の読み込みを有効／無効にするといった幾つかの設定は、ユーザによって頻繁に変更が可能です。ただし、これらの設定は PrefBar や同類の拡張がないと変更ができません。Mozilla での似たような実装は、設定ツールバー、メニュー、またはサイドバーとして扱われています。

コンポーネント・アンインストーラ

現在、Mozilla を再インストールする以外、Mozilla のアドオンを削除する簡単な方法がありません。Bug 161508 で取り組まれています。Mozilla Firefox や Mozilla Thunderbird では拡張機能を無効にすることができますが、完全に取り除くことはできません。

画像のコピー

Mozilla は現在画像をクリップボードにコピーし、直接他のアプリケーションにペーストすることはできません。この問題は bug 21747 で扱われています。

2.2 プライバシーとセキュリティ

W3C Platform for Privacy Preferences (P3P) サポート

P3P の部分的なバックエンドサポートは Mozilla 1.4 以降で利用可能です。完全実装も計画されています。

広告ブロッカー

画像ブロッカーは 外部プロジェクト として開発されています。

しかし、このソフトをインストールする前に、ページをオープンにアクセス可能にするために、広告収入に頼っている、一部の Web サイトのことを考えてみてください。広告収入を失えば、これらのサイトは会員制モデルに移行しなければならなくなります。

メール読み込みの際のサーバーへのアクセス禁止

Bug 28327 を参照してください。

埋め込みコンテンツのコントロール

Mozilla は Flash ムービー、Java アプレット、画像といった組み込みメディアをどのように処理するか、ユーザがコントロールできるようにしています。これまでよりも包括的な解決法が提案されて、作業が進められています（bug 94035, 147866, および 78104 を参照してください）。

Cookie、画像などのホワイトリスト

Bug 75915 を参照してください。

ポップアップブロッカーの実装

Mozilla には既にポップアップブロッカーが実装されています。しかし、多くの ポップアップに関する問題 が未解決のままです。

2.3 プロファイル

プロファイル保護

プロファイルのパスワード保護とプロファイルデータの暗号化については、Bug 16489, 19184 で議論されています。

カスタマイズ可能なプロファイルの保存場所

Mozilla は常に、プロファイルを、ユーザが利用しているシステムのアプリケーションデータのためのデフォルトディレクトリに保存しますが、共有、ローミングアクセス、その他の目的で、ユーザがカスタムプロファイルディレクトリを指定する方法がありません。より柔軟なプロファイル保存が提案されています（Bug 31763, 150453, 144712 を参照してください）。

プロファイルの移行とバックアップ

Mozilla がプロファイルデータとリンクしている方法が原因で、別のマシンへの Mozilla プロファイルの移行や、プロファイルデータのバックアップは、通常、手作業によるプロファイルファイルのコピーや編集が必要となります。Bug 22689, 103993, 7067, 137006, 85316, 63918, 66259 が解決すれば、ボタンを 1 回クリックするだけといった、簡単な解決策が実現するはずです。

プロファイルデータのバックアップや移動の方法についてはプロファイル FAQ を参照してください。

2.4 キーボードとマウス

タッチパッドスクロール

Mozilla はタッチパッドスクロールをサポートしていますが、古いドライバでは標準設定で無効になっています。ドライバを更新するには synaptics ダウンロードページを参照してください。

Opera のようなマウスジェスチャー

Optimoz コンポーネントをインストールすれば、マウスジェスチャーを実現できます。

キーボードショートカットのカスタマイズ

現在、キーボードショートカットをカスタマイズすることはできません。関連バグは Bug 57805 です。

2.5 ブラウザ

<blink> タグの無効化

Mozilla で <blink> タグを無効化するには、ロケーションバーに about:config と入力し、browser.blink_allowed を false としてください。

ActiveX サポート

現在、Mozilla は ActiveX をサポートしていませんが、Communicator 4.x と同様に、プラグインを利用することができます。

フルズーム機能

テキストと画像両方の拡大を実現する試みは Bug 4821 でおこなわれています。

キオスクモード

フルスクリーンモードで混乱しないようにしてください。本当のキオスクモードは、ユーザやシステム管理者が、他のユーザのブラウジングを制御するために、ブラウザの設定や機能を非表示・無効化・ロックすることができるようにするものです。キオスクモードは Bug 3341 で議論されています。

ブックマークのインライン（メニュー内）編集

Bug 19437 を参照してください。

MHTML サポート

Mozilla は MHTML （RFC 2557）ファイルの表示・エクスポートをサポートしていません（Bug 18764, 40873 を参照してください）。

キャッシュファイルの参照

ロケーションバーに「about:cache」と入力すれば、ブラウザのキャッシュを参照することができます（Bug 66299 （cacheexplorer）, 157694 も参照してください）。

タブブラウザ

Mozilla のタブブラウザは優れた機能ですが、タブの並び替え機能、そしてタブの切り離しなど、Web ページを新しいタブまたはウィンドウで 開く機能への設定追加が必要です。これらの機能について我慢できないなら、MultiZilla をインストールすれば、これらの大部分とそれ以上の機能を手に入れることができますし、Mozilla および Mozilla Firefox 向けには Tabbrowser Extensions があります。

アクセス不能なページのためのエラーページ

Bug 28586 では、ネットワークエラーダイアログを、ページを再度ロードできるようにするエラーページに置き換えようとしています。

ブックマークのチェック

ブックマークマネージャ（Ctrl+B）を使って、特定のブックマークについて内容が更新されたかどうかを定期的にチェックすることができます。これを行うには、チェックしたいブックマークを選択し、Ctrl+I を押してください。そして Schedule と Notify オプションを設定してください。リンクが壊れていたり、移動されている場合、ブックマークマネージャはブックマークを更新しません。Bug 171467 でブックマークの妥当性チェックと更新について取り上げられています。

画像の読み込み・再読み込み

Bug 47475 で、ページ内の画像のみを読み込み・再読み込みするオプションをコンテンツメニューに加えることについて取り上げられています。

ダウンロードマネージャ拡張

Bug 418004 では、Mozilla に拡張されたダウンロードマネージャを追加すする方法について扱っています。しかし、現在のビルドにより充実した拡張機能を加えるには、plugindoc にあるダウンロードマネージャのページを参照してください。

2.6 メール・ニュース

外部メールクライアントの利用

デフォルトでは、Mozilla はメールのリンクをすべて自身のメールクライアントで開きます。Mozilla に強制的にシステム標準のメールクライアントを利用させるには、ロケーションバーに about:config と入力し、network.protocol-handler.external.mailto を true にしてください（Bug 11459 を参照してください）。

メール着信通知（biff）アイコン

/Windows/<18729> では、Mail & Newsgroups が開かれ Mozilla が閉じられるまで実行されている時、新しいメールチェックがバックグラウンドで実行されます。この機能は /その他のプラットフォーム/<18730,18731,18732,79127> ではまだ利用できません。人気の高い拡張リクエストの中には、スタンドアローンなメールチェックプログラム、フィルターによって削除または既読とマークされたメールの場合は通知しない、そしてジャンクメールやその他の改良があります。bug 18730, 86553, 35892, 125328, 24577, および 71105 を参照してください。

スタンドアローンのメール・ニュース

Mozilla Thunderbird とコードネームがつけられたスタンドアローンなメールクライアントが開発中です。スタンドアローンなブラウザをビルドするプロジェクトもあります（Mozilla Firefox）。

Mail バックアップ

Bug 133095, 103993 を参照してください。関連する拡張機能については、プロファイルセクションを参照してください。

スレッド化・分類化ビュー

Bug 72493 を参照してください。

POP メッセージの失効日付

POP メッセージを一定の日数サーバに残しておくオプションは、Mozilla 1.6 および Mozilla Thunderbird に実装されています。

スマートなジャンクメールフィルター

Mozilla 1.3 以降には賢い（ベイズ式）ジャンクフィルターが備わっています。（バグを参照してください。）. 使い方についてはMozilla スパムフィルタリングをお読みください。

添付ファイルの削除

Bug 2920 を参照してください。

共通の受信箱

Outlook や Outlook Express の元ユーザの多くが、アカウントごとのフォルダ群ではなく、全てのメールアカウント共通のフォルダをリクエストしています。bug 30057 を参照してください。

複数アドレスへのメール作成

Mozilla Mail のメール作成ウィンドウでは、一つの欄に複数の電子メールアドレスを入力することができません。メールアドレスをコンマやセミコロンで区切るのではなく、リターンキーを利用してください。Mozilla Thunderbird では、メール作成ウィンドウからコンタクトサイドバー（View | Contacts Sidebar）を使って、複数の人宛てにメールを作成してください。bug 94540 および bug 94676 を参照してください。

Mozilla Thunderbird をシステムトレイに最小化する

Mozilla Thunderbird は現在、システムトレイに最小化することはできません。— これは bug 183323 です。ただし、Windows ではあらゆるアプリケーションをシステムトレイに最小化することができるアプリケーションが沢山あります。

送信メールのフィルタリング

送信メッセージのロケーションをフィルターする唯一の方法は、自分宛に BCC: し、自分が送信者となっているルールをフィルターに加えることです。Mozilla MailNews の（Edit | Mail & Newsgroup Account Settings | <アカウント名> | Copies & Folders）または Mozilla Thunderbird の（Tools | Account Settings | <アカウント名> | Copies & Folders）を参照してください。バグ 11038 および 177040 を参照してください。

ウェブメールへのアクセス

Mozilla は、Yahoo! Mail POP3 アクセス（無料ではありません）のような、オフライン用に標準的な POP3 または IMAP プロトコルを使用するウェブメールサービスへのアクセスをサポートしています。一方 Mozilla は、Hotmail、AOL Mail、Netscape ウェブメールなどで使用されているプロプライエタリなプロトコルはサポートしていません（Netscape ウェブメールへのサポートは Netscape が Mozilla に行ったプロプライエタリ拡張の一つです — これは Mozilla Foundation がアクセスできるコードではありません）。しかし、あらゆるメールクライアントからこうしたウェブメールに直接アクセスできるようにするサービスやソフトウェアがあります。自らの責任でそうしたものを試してみることもできます。

2.7 編集機能

スペルチェッカー

Mozilla 1.4.1 にはスペルチェッカーが搭載されています（オープンな話題を参照してください）。

2.8 Web 開発者からの要望

Web 開発者からの要望の多くで、熱い議論が戦わされています。Bugzilla エチケットガイドラインを読んで、罵り合いを避けてください。

ページエラーログ

Mozilla には、現在、不正な HTML、CSS、DOM など、Web ページの評価や問題をユーザが受け取る機能がありません。Bug 6211 でこの問題が議論されています。

<img> タグの alt 属性

alt 属性は、視覚障害者など、画像へのアクセスができない人のための代替表示として意図されたアクセシビリティ機能です。アクセシビリティ上の理由から、Mozilla は alt 属性の値をツールチップとしてレンダリングしません。ツールチップを表示させるには title 属性を使ってください。より詳しい解説は、Bug 25537 Comment 153 や Mozilla Web ページ作者 FAQ でご覧になれます。
【訳注：画像に alt 属性で設定された代替テキストを、NC4 以前や IE などと同様にポップアップ表示するための拡張機能もあります。詳しくは piro さんのページを参照してください。】

縦方向スクロールバーの常時表示

これは長く議論されてきたもうひとつの問題です。Bug 72540 を参照してください。

DHTML

DHTML に関してよくリクエストされる二つの機能は、getComputedStyle メソッド対応の改良と要素のスクロール位置の取得・設定機能です。

コンテキスト（右クリック）メニューをブロックする機能

Bug 86193 では、Web ページ内でコンテキストメニューを無効にするという話題が取り上げられています。
