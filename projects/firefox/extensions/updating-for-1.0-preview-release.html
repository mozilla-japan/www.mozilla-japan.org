Firefox Preview Release と Thunderbird 0.8 に向けた拡張機能とテーマのアップデート

Firefox や Thunderbird のマイナーリリースが出るたびに、アップグレードを済ませたユーザが同じ拡張機能やテーマを使い続けられるよう、作者の皆さんは次のような措置を講じてください。

やるべきこと

  1. ナイトリービルドをダウンロードして、自作拡張機能 (テーマ) をインストールしてください。拡張機能が問題なく利用でき、ブラウザが正常に動作するかどうか、テストを行って確かめてください。テーマの場合は、そのテーマを適用した状態で、すべてのウィンドウやダイアログをテストしてください。
  2. 現在のナイトリービルドは、Firefox Preview Release や Thunderbird 0.8 と見た目は同じですが、拡張機能やテーマが正しく動作しなくなる恐れのあるバグがまだリストに残っていることにくれぐれも注意してください。1.0 PR ブロッキングバグ一覧 に目を通し、これらのバグの中で話題となっているコードがあなたの拡張機能 (テーマ) と関係ありそうな場合は、1.0 に向けた修正作業を追跡する体制を整えてください。
  3. 拡張機能やテーマが正しく動作した場合、addons.mozilla.org に登録しているものであれば、Bug 255804 にそのアイテムを追加してください。その際は必ず拡張機能 (テーマ) の GUID を明記してください。addons.mozilla.org を管理している Wolf あるいは他のスタッフがそれを見て、書かれた GUID について保存されている maxVersion の値を「0.10」に引き上げます。
  4. 拡張機能やテーマが正しく動作せず、その原因が拡張機能 (テーマ) そのものにあると思われる場合、0.9 以降に新機能が追加されたことがきっかけでしょうから、ナイトリービルドで動作するよう、必要なコードやテーマ、画像の変更を行い、アイテムをアップデートしてください。それから、Bug 255806 に XPI/JAR ファイルへのリンクを追加してください。
  5. 拡張機能やテーマを addons.mozilla.org に登録しておらず、自作拡張機能 (テーマ) のインストールマニフェストでカスタムアップデート RDF ファイルを指定している場合は、その場所にあるマニフェストを編集して、アップデートに関する情報を提供することができます。各種 targetApplication 向けに maxVersion をアップデートしても現在のバージョンが互換性を保てることを知らせるか、互換性のある新しいバージョンにリンクを貼ってください。この RDF ファイルの形式については 拡張機能のバージョン管理、アップデート、互換性 を参照してください。

詳しい情報

バージョン管理システムとアップデートに関する詳しい情報は、拡張機能のバージョン管理、アップデート、互換性 を参照してください。

Firefox と Thunderbird の拡張機能 (テーマ) 管理システムに関する詳しい情報、その他の状況については、拡張機能プロジェクトのページ をご覧ください。
