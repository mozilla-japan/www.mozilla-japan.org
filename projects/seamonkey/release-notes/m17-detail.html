M17 リリースノート（詳細）


システムの必要条件

Windows 95、 Windows 98、 Windows 2000、 または Windows NT 4.0、 Pentium、 133 MHz、 32 MB のメモリ

Mac OS 8.5、 Mac OS 8.6、 または Mac OS 9、 200 MHz PowerPC 604 または G3、 32 MB のメモリ（仮想メモリ”入”）、または 48 MB のメモリ（仮想メモリ”切”）

Mac OS: QuickTimeLib ファイルが必要です。

Red Hat Linux 6.1, Pentium II, 233 MHz, 32 MB のメモリ

Linux: SuSe Linux 6.2 上で Netscape 6 を動作させるなら、ここから入手できる libjpeg.so.62 ファイルをインストールする必要があります。


新着情報


M17 の開発サイクルは、Mozilla 1.0 に予定された主要機能を追加する最後の作業を含みました。 したがって、今後の開発の焦点は、性能、安定性、 footprint（訳者注：アプリケーションを起動する際に必要とするメモリ領域の大きさのことらしいです） に向けられます。以下は、M16 からの変更点のリストの一部です。

Classic UI（訳者注：Netscape Communicator 4.7 のような従来タイプの UI）が使えます。この新しい UI を体験するには、Edit（編集）｜Preferences（設定）で Appearance（表示）｜Themes（テーマ）を開き、Themes（テーマ）パネルで Classic を選択して Switch Themes（テーマを選択）ボタンを押します。

スクロール位置がセッション履歴に記録されるようになりました。

Biff、つまり email 通知 がこの版で実装されました。この機能を有効にするには、mozilla mail window の Edit（編集）｜Mail News Account Settings（アカウントの設定）を開き、mail Server（サーバ）の設定で "Check for messages..."（"新規メールを□分ごとにチェックする"）の項目にチェックします。

アドレスバーの自動補完とセッション履歴は、より実用的になりました。

Mail News のメッセージ検索が実装されました。mozilla mail ウィンドウの Search（検索）メニューにあります。

win32 版にアルファ透過がつきました（mac 版と linux 版には以前からついています）。

MNG を扱えるようになりました。

Tooltips が実装されました。

linux 版で plug-in support が利用できるようになりました（mac 版と win32 版では以前から利用できます）。

Helper apps（ヘルパーアプリケーション）が機能します。ファイルタイプ毎に外部アプリケーションを指定することができます。

すべてのプラットフォームで mail thread pane（メールスレッド枠）は、より早くスクロールします。

M16 から、unix 向けのいくつかの server side x-remote support が利用できます。

フレーム操作が改善されました（フレームのソース表示とフレーム内の履歴／ナビゲーション修正）。

改良された cookie 処理
 


インストールする前に読んでください

AOL 4.0 かそれ以前のバージョン と Mozilla を同時に動かすことは問題になるかも知れません。 AOL 5.0 と Mozilla を動かす事による問題は報告されていません。

もし起動時にクラッシュしたならば、あなたの Mozilla のレジストリを削除してください。 Mozilla のレジストリは M9 から全てのプラットフォームで変わっています。

  • Mac OS: System Folder（システムフォルダ）内の Preferences（初期設定）フォルダ を開いて、 "Mozilla Registry" ファイルを削除してください。
  • Windows: Windows ディレクトリ を開いて、"mozregistry.dat" を削除してください
  • Linux: ~/.mozilla ディレクトリ を開いて、"registry" ファイル を削除してください。
さらに詳しく知りたい方は ここ をクリックしてください。 Win32 にインストールしている場合には インストール手順 にある起動時のクラッシュに関する追加情報も読んでください。

空のディレクトリにインストールするか、あるいはインストールする以前に、前のビルドを削除してください。以前のマイルストーンや製品に上書きでインストールすると問題を引き起こすかもしれません。 ソフトウェアが開発中である間はマイルストーン間での不整合が生じるかもしれません。

ディレクトリが書き込み可能であることを確かめてください。 Mozilla は、アプリケーションを実行する人が Mozilla をインストールするディレクトリに対して書き込みパーミッションを持っていることを必要とします。

glibc 2.0 を用いた Linux システム (RedHat 5.2 など)では、thread safe ではない dl 関数 (dlopen(), dlsym()) の問題があります。 これは M17 ビルドの起動時にランダムにクラッシュすると思われます。 もしこの問題が発生したならば、アプリケーションを何度か起動し直してください。 ftp.mozilla.org にポストされたものは RedHat 6.0(それは glibc 2.1 を使用して、この問題を持っていません)でビルドされます。より詳しい情報は Bug 8849 を参照してください。


 
 

M17 リリース・ビルドをインストールする

インストールの手順 -- Unix

  1. "mozilla" という名のディレクトリを作成して下さい。


    2. mkdir mozilla ;

  2. mozilla ディレクトリに移動して、アーカイブを展開して下さい。すると "package" というディレクトリが作成されます。


    3. cd mozilla ;
    gzip -dc ../mozilla-i686-pc-linux-gnu-M17.tar.gz | tar -xvf -

  3. "package" ディレクトリに移動して下さい。


    4. cd package

  4. mozilla を起動スクリプトで起動して下さい。


    5. ./mozilla

重要: Seamonkey を Solaris で実行するには、Mozilla on Solaris FAQ も参照してください。 ここにはビルドと実行に Solaris 2.6 パッチが必要であることが説明されています。

Unixでメールを使用する手順

既に使用している 4.x のプロファイルから移行することもできます。また、新しいプロファイルを作り、Account Setup（アカウントの設定）を使ってサーバー情報を設定することもできます。

インストールの手順 -- Win32

mozilla をインストールで推奨する方法は、インストーラー・プログラムを使用することです。
  1. mozilla-win32-installer-M17.exe （訳者注：のリンク）をクリックすると、ファイルがデフォルトのフォルダにダウンロードされます。

    2.  
  2. mozilla-win32-isntaller-M17.exe をダブルクリックして、画面の手順にしたがってください。

    3.  
  3. mozilla を実行するには スタート - プログラム - Mozilla Seamonkey - Mozilla をクリックしてください。


備考:
移行していない 4.x のプロファイルも 5.0 のプロファイルも既にあり、そして mozilla コマンドを実行しても Seamonkeyが起動しない場合は、mozilla -installer コマンドを実行してプロファイル・マネージャを立ち上げ、プロファイルを選択してください。

Win32 でメールを使用する手順

既に使用している 4.x のプロファイルから移行することもできます。また、新しいプロファイルを作ってアカウントの設定（Account Setup）を使ってサーバー情報を設定することもできます。

インストールする手順 -- Mac OS

StuffIt Expander™ がインストール済で、使用しているブラウザがそれを使うように設定されている場合:
  1. mozilla-mac.sea.bin のリンクをクリックします。ファイルがデフォルトのフォルダ（通常デスクトップ）にダウンロードされ、StufIt™ が起動して、それを mozilla-mac.sea に変換します。それから StuffIt は mozilla-mac.sea を開き、それを mozilla-mac フォルダに変換します。

    2.  
  2. mozilla-mac フォルダを開いて "mozilla" ファイルをダブルクリックしてください。（注: この段階で .bin と .sea ファイルはもう要りません。）
備考: もし手順の一つ目でブラウザーから何をするか問い合わせられた場合は、StuffIt™ の設定がされていないのが原因です。 その場合には保存(Save)をクリックして下記の 手順 3番 に進んでください。

StuffIt Expander™ がインストールされていないか、あるいはブラウザーが適切に設定されていない場合:

  1. StuffIt™ がハードディスクのどこかにあることを確認してください。

    2.  
  2. mozilla-mac.sea.bin のリンクをクリックしたままにしてください。

    3.  
  3. "Save this link as...（リンクを別名で保存...）" を選択してください


    4. ファイルダイアログが現われて、ファイルをどこに保存するか選択することができます。(例えばデスクトップ)

    注: Macのインストーラーのアクセスパスにはスラッシュ (/) を使わないでください。 スペースは使用しても構いませんが、その他の特殊な文字は避けてください。

  4. Save(保存)ボタンをクリックしてください。

    5.  
  5. 新しく作られた "mozilla-mac.sea.bin" のアイコンをダブルクリックしてください。

    6.  
  6. StuffIt™ が起動し、同時に "mozilla-mac.sea" ファイルと "mozilla-mac" という名前のフォルダが作成されます。

    7.  
  7. mozilla-mac フォルダを開いて "mozilla" ファイルをダブルクリックしてください。（注: この段階で .bin と .sea ファイルはもう要りません。）

もしもインストーラを使用して "error 340" となったならば、インストールを始めからやりなおしてください。

Macintosh でメールを使用する手順

既に使用している 4.x のプロファイルから移行することもできます。また、あたらしいプロファイルを作ってアカウントの設定（Account Setup）を使ってサーバー情報を設定することも出来ます。

一般的な問題

インターフェースのいくつかの問題(コントロールのためのいくつかの HTML for controls と XP Toolkit ウィジェット) がコンテキストメニューを呼び出すためにマウスをクリックした際に十分に応答しない。(右利き用マウスで右クリックを行う)(bug 30497)

(Linux のみ) タスクバーメニューのウインドウ名をクリックした際、コンポーネントアイコンをクリックした際、あるいはタスクバーの "Go To Window " アイコンをクリックした際に、ウインドウが全面に出ない。ウインドウ自身をクリックすると全面に出る。(bugs 8002 と 26082.)

(Windows のみ) Ctrl-Tab が現在開いているウインドウの間を廻らない。代替方法: タスクメニューを開いて、開きたいウインドウを選ぶ。(bug 30864)

キーボードショートカット

キーボードショートカットはすべてが実装されているわけではない。編集のためのいくつかのショートカット(たとえば、テキストのカットやペースト)があるダイアログボックスで働かない。ナビゲーションのためのキーボードショートカットが動作しない。複数のキーボードショートカット; メニューアイテム が動作しない。(bugs 22529,と20298.)

(Linux のみ) Keyboard の記憶 (メニューアクセスキー menu access keys) が一時的に利用不可能になる。 (bugs 22515 と 24413)

Preferance(設定)

設定ウインドウを終了した後、mozillaがアクティブなアプリケーションでなくなる。mozilla ウインドウをクリックして前面に出しなさい。(bug 6058)

フォント設定ダイアログボックス: フォントサイズを 12pt より小さくできない。 (bug 30813).

スタイルシートを off にする方法がない。(bug 32372)

メニュー

すべてを選択(Select All)がコンテキストメニューで動作しない。代わりに Edit メニューからすべてを選択を選びなさい。 (bug 28508)

(Linux と Windows のみ) コンテキストメニューから Copy を選んだとき、メニューがスクリーンに残ったままになる。メニューを取り除くには、その外側をクリックしなさい。(bug 28773)

Mousewheel のスクロール

Unix: Xサーバは適切に設定しなければならない。XFree86 では、自分の XF86Config ファイルの Pointer セクションに "ZAxisMapping 4 5" のような行を加えなさい。同時に自分のマウスに適切に Protocol の行をセットしなさい。

Windows: もし Logitech のホイールマウスを使っているのなら、Mozilla でのスクロールは動作しないかも知れない。それは Logitech の em_exec プログラムが起動している間である。この問題に対する代替手段(自分のマウスドライバをアップグレードする必要もあるかもしれない)はバグの中に文章されている。20618.

Mac OS: Mac OS のための(USBマウスの)マウスホイールのスクロールはまだ実装されていない。

プロファイル

重要: プロファイルに関係する情報は、すべてレジストリーファイルに保存されています。 レジストリーファイルは、Mozilla の初期のバージョン(AppRunner)を使った場合でも作られています。 新しいビルドをチェックアウトした時は、インストールやビルド、そして実行する前に、必ず以下の項目のものを削除するか、またはファイル名を変更してください:
  • Windows mozregistry.dat ( c:\winnt または c:\windows または同等の場所),
  • Mac OS Mozilla Registry (System Folder:Preferences),
  • Linux registry (~/.mozilla/)
プロファイルを保存するフォルダの選択は、どのプラットフォームでもうまく動きません。-- "default" フォルダがいつも使われます。 Linux と Mac OS の場合: カスタムフォルダを選択できますが、プロファイルは、依然として "default" フォルダに保存されます。 (バグ 32808).  Windows の場合: カスタムフォルダボタンを押しても、何も起こりません。プロファイルをデフォルトのディレクトリに保存するしかありません。 (バグ 34071).

デフォルトのプロファイルの場所は、Windowsでは、<exe の場所から 3 つ上のレベル>\Users50、Mac OS では、 :Documents:Mozilla:Users50:（訳者注：Mac OS 日本語版では :書類:Mozilla:Users50:） 、そして Linux では、 ~/.mozilla です。
 
 

新しいプロファイル名に空白を含めたい場合は、名前を引用符で囲んでください(例えば、"My Profile")。

コマンドラインの引数 -ProfileManager を使って、プロファイルマネージャを起動することができます。 プロファイルマネージャのユーザインタフェースが現れ、新しいプロファイルを作成したり、既存のプロファイルを削除したり、名前を変えたりできます。 作業が終ったら、Exit をクリックすることで、プロファイルマネージャを終了させることができます。 "Communicatorを開始"ボタン(訳者注: "Mozillaを開始"ボタンのことと思われる)をクリックして、選択したプロファイルを使って Mozilla を起動することができます。 また、4.5 Communicator のプロファイルから移行することができます。

プロファイルを作成したり、管理するための様々なコマンドラインオプション:

# mozilla -P <Profilename>
Mozilla を <ProfileName> を使って実行します。
# mozilla -nosplash -CreateProfile "<Profilename> <ProfileDir>"
(<Profilename>という名前で)プロファイルを作成します。そして、指定された場所<Profiledir>にディレクトリを作成し、アプリケーションを終了します。
# mozilla -nosplash -CreateProfile <Profilename>
プロファイルを作成します。そして、プロファイルのディレクトリをデフォルトの場所に作成します。
# mozilla -ProfileWizard
新しいプロファイルを作成するために、"新しいプロファイルの作成"ウィザードを起動します。 "新しいプロファイルの作成"ウィザードにより、新しいプロファイルを、デフォルトの場所またはユーザが選択した場所に作成することができます。 プロセスが終ると、プロファイルが作られ、現在のプロファイルとしてマークされます。Mozilla は、そのプロファイルを使って起動されます。 ウィザードは、現在のところ、2 画面だけです。メールとニュースに関する設定/情報は、別のアプリケーションが担当することになっています。
# mozilla -ProfileManager
プロファイルマネージャを起動します。プロファイルを選択して、"Communicatorを開始"ボタン(訳者注: "Mozillaを開始"ボタンのことと思われる)を押すと、選択したプロファイルで Mozilla を起動することができます。 現在のプロファイル(普通は、最後に使用したプロファイル)は、あらかじめ選択されています。
# mozilla -SelectProfile
"プロファイルの選択"を起動します。表示されている中からプロファイルを選択することができます。あるいは "プロファイルの管理" をクリックして、プロファイルマネージャを起動して、上記のような処理をすることができます。
# mozilla
いくつかのシナリオがあります:
 プロファイルの数 = 0 --> "新しいプロファイルの作成" ウィザードを起動します。
 プロファイルの数 = 1 --> Mozilla を直接そのプロファイルで起動します。
 プロファイルの数 > 1 --> プロファイルマネージャを起動します。
4.5 のプロファイルを移行するには、コマンドラインオプションの-installerで Mozilla を実行してください。 ユーザは、移行するプロファイルを選択することができます。そして、"Mozillaを開始"ボタンを押すことができます。 移行が成功すると、そのプロファイルはもはやアイコン付きで表示されてはいません。これは、そのプロファイルが 5.0 のプロファイルであることを示しています。 現在のところ、prefs.js ファイルとメールとニュースフォルダだけが、移行の過程でコピーされます。 ユーザは、移行したプロファイルの名前を変えたり、移行したプロファイルを使って Communicator を起動することはできません。

パスワード

（訳者注：パスワードに関するメニュー、ダイアログの項目やメッセージ、操作手順などは、 ここに書かれている内容と実際とで食い違っているようです。）
Password Manager（パスワードマネージャ） がオフになっている場合、パスワードは記憶されません。 Password Manager をオンにするには:
  1. "Edit（編集）" メニューを開き、"Preferences（設定）" を選びます。
  2. "Advanced（詳細）" カテゴリーをクリックして開きます。
  3. "Forms and Passwords（フォームとパスワード）" をクリックします。
  4. "Enable Password Manager（パスワードマネージャを有効にする）"の隣にチェックマークを付けてください。
(バグ 27994)

"Account Setup（アカウント設定）" の "Save Password（パスワードの保存）" チェックボックスをオンにしてもパスワードは保存されません。
マスターパスワードデータベースにパスワードを保存するには、サーバーにアクセスしたときに現れる、サーバーパスワードダイアログの中の "Save Password（パスワードの保存）" チェックボックスをオンにしてください。 そうすると、あなたのマスターパスワードを入力するように求めるダイアログボックス、または新しいマスターパスワードを設定するよう求めるダイアログボックスが現れます。

マスターパスワードリストからメールサーバを取り除くには、以下のどちらかを行ってください:

  • "Edit（編集）" メニューから "Preferences（設定）" を選び、 "Advanced（詳細）: Password Maneger（パスワードマネージャ）" をクリックしてください。 "View Stored Passwords（格納されたパスワードを見る）" をクリックした後、 サーバーを選んで、"Remove（削除）" をクリックしてください。

    •  
  • "Tasks（タスク）"メニューから Autofill: Display Signons を選び、 "View Saved Passwords（格納されたパスワードを見る）" をクリックします。 そして、サーバーを選択し、"Remove（削除）" をクリックします。
時々、パスワードボックスは、他のウィンドウに隠れることがあります。 そうなると、パスワードを入力して、パスワードボックスがなくならない限り、何も実行できません。 パスワードボックスが見えるように他のウィンドウを動かしてください。(バグ 28459)

フォントの問題

もしフォントがあなたの環境で小さすぎるのであれば、Mozilla の preference の設定により、多少は改善できるかもしれません。 Edit メニューの中の Preferences オプションを選んでください。 Preference ダイアログで、Fonts 項目を選んで、dpi エントリーを 120 に設定してください。

X サーバーにおける DPI 値の設定についての情報が http://www.mozilla-japan.org/unix/dpi.html にあります。

もしフォントがまだ小さすぎる(あるいは大きすぎる)場合は、DPI を正しく設定した後、フォントサイズを以下のように調整することができます:
setenv GECKO_FONT_SIZE_FACTOR 1.5

後者の方法は、フォントだけに影響することに注意してください。スクリーン解像度の設定は、絶対サイズ(例. CSS の "pt" ポイント指定)を指定するすべてのものに影響します。
