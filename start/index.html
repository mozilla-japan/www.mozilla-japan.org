おや! あなたがお使いの Mozilla は、4 週間以上前のビルドです。最新のマイルストーンビルドをお使いでない限り、日々寄せられるバグの報告によって、あなたの古いビルドはすぐに役に立たなくなってしまいます。

ですから、最新のマイルストーンを使っているのでなければ、ぜひ、最新のマイルストーンビルド か 最新のナイトリービルド のいずれかをテストして、開発にご協力いただければ助かります。(ただし、ナイトリービルドはあくまで開発中の製品ですので、お使いのコンピュータの調子を悪くしたり、頻繁にクラッシュしてあなたを苛立たせないか、といったことに関して、一切保証はありません。この点にはくれぐれもご注意ください)

Mozilla に参加するには

おめでとう! あなたは Mozilla をダウンロード (あるいはコンパイル) しました。つまり、あなたは Mozilla をテストするコミュニティの一員に名乗りを上げたわけです。(そのようなつもりがなければ、私たちの 安定化リリース を使った方が良いでしょう) そうでなくても、あなたの参加を歓迎します! 手助けに時間はかかりませんし、特別な技術も必要ありません。あなたの手助けが Mozilla の向上に役立つのです。

訳注: このページは mozilla.org の Web サイトにある Getting Involved with mozilla.org を和訳したもので、内容的に若干古くなっている部分もあります。なお、ページの最後に 日本の Mozilla コミュニティ「もじら組」のご案内 (訳者による追記) がありますので、ぜひお読みください。

何をしたらいいのでしょうか?

バグの報告
あなたはすでにビルドをダウンロードしています。これからやらなければならないことは、Mozilla を普段のブラウザ、メール／ニュース リーダーとして使うことだけです。もし Talkback 機能付きのビルドをダウンロードしたのなら、それが表示されたときには 作動させるようにしてください。Talkback レポートは、どのクラッシュがいちばん深刻で、それがどのくらいの頻度で発生するのか、といった本当に貴重なデータを私たちに提供してくれます。あなたはただ「OK」をクリックするだけでいいのです。もしバグだと思われるものを発見したら、それが 既知のバグ でないことを確認した上で、バグ登録手続き に従ってください。
(訳注: Bugzilla 日本語版 では、日本語によるバグの報告ができます。ぜひご利用下さい。)
品質保証
Mozilla 品質保証 には、手伝いに参加していただける方法をご紹介する ページ があります。これは、HTML の知識が多少あると役に立ちますが、コードの書き方を知っている必要ありません。品質保証に参加することは、Mozilla をもっとよく知りたいと思っている人には役に立ちますし、そこには強力なコミュニティがあります。特にいい方法は BugAThon に参加することです。
バグの修正
本当にあなたを悩ませるバグがありますか? 報告するのと同じように、自由にそのバグを修正してください。Mozilla のバグを修正することは、他の多くのアプリケーションの場合よりもはるかに簡単です。なぜなら、今あなたが動かしているビルドだけを使って、(たとえば、私たちの XML ベースのユーザインターフェイス言語である XUL や、CSS、JavaScript で記述されている、クロスプラットフォーム・フロントエンドに関する) バグの修正ができるからです。複雑な環境を立ち上げたり、何かをコンパイルしたりする必要はありません。私たちは、とても簡単に作業を始められるようにするツールも作りました - Patch Maker です。これで、Web ページを作成したことがあれば、あなたも Mozilla をハックできます。
ドキュメントと Web サイト
現在、この Web サイト (mozilla.org) は全面的な書き直しがおこなわれていて、どんな協力でもしていただければとても助かります。また、Mozilla はあらゆる種類のドキュメンテーションを必要としています。Mozilla ドキュメンテーションプロジェクト をご覧ください。この活動の中心は、ニュースグループ netscape.public.mozilla.documentation です。
コミュニティ
Mozilla の開発者たちは、たいてい IRC を「たまりば」にしています。もし ChatZilla をインストールしているのなら、接続は 簡単 です。または、お気に入りの IRC クライアントを立ち上げて、irc.mozilla.org:6667、#mozillazine に接続してください (これは MozillaZine によるフレンドリーなチャンネルです)。開発者向けのチャンネルは #mozilla ですが、このチャンネルでは参加者が何らかの作業をしているということに注意してください。ここでのサポートに関する質問はご遠慮ください。Mozilla コミュニティに参加する他の方法として、ニュースグループ や ステータスアップデート を読むこともできます。
(訳注: 日本語による IRC チャンネルもあります。詳しくは MozillaZine IRC のご案内 をご覧ください。)

注意: お使いのビルドが mozilla.org の公式な最終リリースではない場合、それは テスト目的に限って 公開されているもので、エンドユーザサポートはありません。もしそれが嫌なら、最終リリース を使った方が良いでしょう。
