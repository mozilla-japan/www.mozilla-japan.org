1.1. Mozilla 1.4 が対象にしているユーザーは?

Mozilla 1.4 は、Mozilla テクノロジーで何ができるかに興味を持っている人や、Mozilla をベースにした製品、パッケージを作成したい人のための、完全に機能するアプリケーションです。Mozilla は オープンソース ですので、Mozilla Public Licence を遵守する限り、どんな個人あるいは会社でも、自由に変更あるいは再頒布できます。

Mozilla 1.4 は、一般のユーザーが使用しても快適なように作成されていますが - そして、mozilla.org からリリースされた Mozilla を使用したい、という人を歓迎しますが - mozilla.org は エンドユーザーサポート の提供を開始しました。また、mozilla.org は新しい テスターとバグ・レポーター を常に求めています。

1.2. なぜ様々なバージョンの Mozilla があるのですか? どれを選べばいいのでしょうか?

Mozilla は速い開発ペースに従っており、マイルストーンリリースは 開発ロードマップ によって作成されています。2003 年現在、Mozilla は四半期ごとのスケジュールでリリースされています。

バージョンの増加は、必ずしも「アップグレード」を意味するとは限りません。通常、「アップグレード」のたびに機能の変更が行われますが、逆に安定性が落ちる可能性もあります。マイルストーンリリースは、場合によっては、安定化とセキュリティバグ修正のために、メンテナンスサイクルにシフトすることがあります。ユーザは、それらの「安定化ブランチ」をダウンロードすることが推奨されます。現在、1.0 と 1.4 のみ、安定化ブランチとなっています。

1.3. Netscape 7 と Mozilla はどう違うのですか?

Netscape 7.1 は Mozilla をベースとしています。どちらのアプリケーションも、タブブラウズ機能、カスタムキーワード、Sidebar など、ほぼ同一の機能を共有しています。例外は、Netscape WebMail や AOL Mail のサポートといった独自機能が追加されていることです。概して、Netscape 7 と Mozilla は同じように Web ページを表示し、メールを読み込みます。

1.4. Mozilla の新しいバージョンをインストールするには?

まず最初に、Mozilla 1.4.1 リリースノート であなたの OS について書かれている記載に従って、Mozilla をアンインストールした後に、新しいバージョンをインストールしてください。新しいバージョンの Mozilla を、古いバージョンに上書きしては いけません。

もしあなたの plug-ins フォルダが、Mozilla application フォルダの中にあれば、再インストールの後でリストアできるように、事前に他の場所に保存しておけます。また、プラグインフォルダは、あなたの プロファイル にも保存できます。ヘルプ｜Plug-in について (Help | About Plug-ins) を開いて、Mozilla が現在使用しているプラグインの場所を確認してください。

この方法で Mozilla をアンインストールすれば、あなたのプロファイルとユーザー設定は変更されずに残るでしょう。いくつかのアドインにはアクセスできず、再インストールの必要があるかもしれません。

もし、Mozilla の新規のインストールで問題が生じたら、トラブルシューティング と スタートガイド をご覧ください。

1.5. Mozilla のパッチはどこでダウンロードできますか?

Mozilla の安定化リリースは、安定性、安全性が高められており、通常、アップデートの必要はありません。(実際、Mozilla 1.0 は、リリース後 1 年で 2 回のメンテナンスリリースを出すにとどまりました)

mozilla.org はパッチをリリースしていません。スペルチェッカーなどの新機能が必要な場合は、アドオン をインストールするか、次の安定化マイルストーンリリースを入手してください。

1.6. Mozilla の機能に関するアドオンはどこで入手できますか?

アドオン のページを見てください。mozdev.org には、多数の開発プロジェクトがあります。

1.8. 「隠し設定」があると聞いたのですが、どのように設定したら良いのでしょう?

編集｜設定 (MacOS X では: Mozilla | Preferences) にある設定と同様に、上級ユーザーは多数の Mozilla のパラメータを精緻にコントロールできます。これらの詳細設定にアクセスし、カスタマイズするには、アドレスバーに "about:config" と入力してください。

誤った設定をすると、Mozilla が正常に動作しなくなる可能性があります。ユーザの設定はプロファイルの prefs.js に保存されています。about:config を使う前に、必ずこのファイルをバックアップしてください。また、Mozilla のカスタマイズ、メール編集隠し設定 そして アドレス帳隠し設定 をお読みください。

より使いやすいインターフェイスのアドオンをお求めなら、MozTweak や Preferential も利用できます。

1.9. Mozilla に、クラッシュ回復機構はありますか?

クラッシュ回復プロジェクトは http://recall.mozdev.org/ で開発中です。

1.10. Mozilla で、Netscape のように、AOL インスタント・メッセンジャーを直接使用できますか?

AIM は、AOL の独自の製品で、Netscape だけに統合されています。スタンドアローン 版の AIM や、(もし Java プラグインをインストールしていれば) Java 版 は使用できます。

Mozilla のための、Jabber インスタント・メッセージ・クライアントも、Jabberzilla で開発中です。

1.11. Netscape 4.x をアンインストールすると Mozilla に影響しますか?

Mozilla は、もしあれば、Netscape 4 の plug-in フォルダのプラグインを使用します (Mac OS X のネイティブビルドを除く)。Netscape 4 をアンインストールすると、これらのプラグインも削除されるかもしれません。Netscape 4 をアンインストールする前に、Netscape 4 の plug-in フォルダの中身を Mozilla の plug-in フォルダにコピーする必要があるかも知れません。ヘルプ｜Plug-in について を開いて、Mozilla が今使用しているプラグインが実際にどの場所にあるかを確認してください。

1.12. Mozilla のニュースや Mozilla を支持するサイトにはどのようなものがありますか?

Mozilla のニュースとリソースを含むサイトは:

  • MozillaZine (日本語版) - ナイトリー テストビルド の コメント もあります。
  • MozillaNews
  • BlogZilla - ニュース と tips
  • MozillaZine developer weblogs
