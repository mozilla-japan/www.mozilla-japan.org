Mozilla 1.7 RC1 新着情報

このドキュメントには、Mozilla 1.7 RC1 で実装された新機能や重要なバグの修正、その他新たな問題の一部が載っています。1.7 RC1 で行われた修正の詳細なリストは「大まかな変更ログ」をご覧ください。このリストには、1.7 Alpha と 1.7 Beta で加えられた新機能と修正がすべて載っています。

新機能と修正

ブラウザ

  • Windows 版と Linux 版の Mozilla 1.7 RC1 公式ビルドには Talkback クラッシュ報告ユーティリティ が付属しています。Mozilla 1.7 をこれまでで最も安定したリリースにするため、クラッシュレポートを送信して、バグの修正にご協力ください。
  • JavaScript を使ってブラウザのコンテキストメニューを表示させないようにする Web ページの仕掛けを無効化するオプションが追加されました。
  • Mozilla と関連付けられたファイル形式のための新しいアイコンセットが用意されました (Windows 版のみ)。
  • パスワードマネージャに、保存されたパスワードを表示する「パスワードを表示」モードが実装されました。マスターパスワードを使っている場合は、モード切り替えの際にそのパスワードを入力する必要があります。
  • 「壁紙に設定」機能を使ったときに、確認ダイアログが表示されるようになりました。
  • Linux 版 (GTK2 ビルド) では、OS テーマのサポートが向上しました。
  • Cookie 関連のダイアログが改良され、より使いやすくなりました。
  • Mac OS 版では特に、日付処理が改良されました。
  • Mozilla のポップアップブロック機能が、2 種類の設定 (dom.popup_maximum と dom.popup_allowed_events) を使ってカスタマイズできるようになりました。ただしユーザインターフェイスはまだありません。それでも、これまで以上に様々な種類のポップアップ (主にマウスオーバーによって開かれるもの) がブロックされ、一度に開くポップアップも最大で 20 個程度に制限されていることに気付くはずです。ポップアップブロック機能がアクティブになっているかどうかにかかわらず、多数のポップアップウィンドウを繰り返し開くサイトからの強力な保護を提供します。
  • ダウンロードしたファイルが、ユーザによって目的の場所が選択されると、すぐにその目的のディレクトリに移動するようになりました。これは、今まで頻繁に報告されていたバグのひとつでした (Bug 55690)。
  • スムーズスクロールの有効・無効を切り替えるユーザインターフェイスが追加されました (設定ウィンドウの Appearance (表示) パネルにあります)。

メール

  • Address Book Palm Sync に多くの改良が加えられました。
  • IMAP の IDLE コマンドをサポートしました。
  • SSPI NTLM 認証を利用した、SMTP と POP3 用の「MSN Authentication」と Secure Password Authentication をサポートしました。
  • 返信元メッセージの文字コードを使わずに「メッセージに返信する際は常にデフォルトの文字コード使用する」設定が追加されました。
  • メッセージのダウンロードと表示、保存処理のパフォーマンスが向上しました。
  • 今回の Mozilla 1.x リリースで初めて、同一のメールアカウント上で複数の ID をサポートしました。詳しくは 複数 ID サポート に関するドキュメントをご覧ください。
  • また、prefs.js で、メールフォルダに相対パスを指定できるようになりました。これによって、後で prefs.js を書き換えなくても、プロファイルを簡単に別の場所にコピーできるようになりました。
  • 「ラストネーム、ファーストネーム」の形式で書かれている名前を含むアドレスリストを編集できるようになりました。
  • メール編集時に、上下矢印キーを使って To/Cc/Bcc の入力欄を移動できるようになりました。
  • Mac OS 版では、添付ファイル名を decomposed Unicode ではなく composed Unicode に変換して表示するようになりました。
  • Mozilla の LDAP クエリが、すべてデフォルトで LDAP v3 を利用するようになりました (これまでは v2 でした)。隠し設定 .protocolVersion を使って、サーバごとに強制的に v2 を利用することも可能です。LDAP の自動設定に詳しい開発者も、自身の JavaScript から setProtocolVersion を呼び出すことで、これまでの v2 に対応できます。詳しくは Bug 198168 のコメントを追ってください。

ChatZilla (IRC クライアント)

  • キーボードショートカット (Ctrl++/-) または View (表示) メニューを使って、文字サイズを自由に変えられるようになりました。
  • 日付処理が改良され、地域設定に合った日付と時刻の形式を使用するようになりました。
  • ChatZilla が 0.9.59 にパージョンアップし、次のような改良が行われました。
    • ignore コマンドをサポートしました。
    • フォントの種類やサイズを自由に変えられるようになりました。
    • Windows 版と Linux 版では、効果音をカスタマイズできるようになりました。
    • ハーフ Op モードのユーザインターフェイスが改良されました。

バックエンド

  • 今回のリリースで、Mozilla 1.7 のサイズとパフォーマンスが劇的に向上しました。Mozilla 1.7 RC1 は、Mozilla 1.6 と比べて、起動時間が 7%、ウィンドウが開くまでの時間が 8%、ページの読み込みが 9% も速くなりました。また、バイナリサイズが 5% 小さくなりました。
  • 長年にわたる懸案事項であった、 テーブル要素の CSS バックグラウンドに関するバグが修正されました (標準モードのみ)。
  • GSSAPI を利用した Kerberos HTTP 認証をサポートしました。これは、Linux や Mac OS X など Unix ベースのプラットフォームに恩恵をもたらします。
  • gnome-vfs ライブラリを使った smb:// URL をサポートしました。この機能は Linux 版 (GTK2+XFT ビルド) でのみ有効になっています。
  • multipart/x-mixed-replace と XMLHttpRequest を使った XML ドキュメントのサーバプッシュをサポートしました。
  • Java アプレットのコードベースが異なるドメインにあった場合でも、LiveConnect が正しく機能するようになりました。
  • Mac OS 版で、幅が非常に大きい (4095px 以上の) 画像が正しく表示されるようになりました。
  • CSS3 の opacity (透過) プロパティをサポートしました。
  • onbeforeunload イベントをサポートしました (Bug 68215)。これによって、Web アプリケーション開発者は、機密情報が含まれる可能性のある Web アプリケーションを閉じたり、HTML ページを離れたりする際に、データが失われる恐れがあることを知らせる警告ダイアログを出すようなコードを追加できるようになりました。
  • 新しい SVG バックエンドが実装されました。これは mozilla.org のリリースではまだ有効になっていませんが、開発者はこの機能を有効にした独自ビルドをコンパイルすることが可能です。
  • 動的なスタイル変更処理が改良されました (Bug 15608)。
  • Network Security Services (NSS) ライブラリがバージョン 3.9 にアップグレードしました。NSS 3.9 は、NISCC の SSL/TLS テストと S/MIME テスト (不正な入力データを使った 160 万回のテストケース) を、クラッシュやメモリリークを起こすことなくすべてクリアしています。

新たな問題

ここには、前回のマイルストーン以降 既知の問題 のページに追加された事項が載っていますが、バグ自体は前からあった可能性もあります。

一般

  • Mozilla を数時間にわたって最小化しておくと、ウィンドウが元に戻るまで時間が掛かってしまうバグ (Windows 版のみ) に遭遇した場合、(about:config から) config.trim_on_minimize を false に設定することで、この問題を軽減することができます。詳しくは、Bug 76831 の コメント 0 と コメント 303 をご覧ください。
  • Mac OS 版では、IPv6 サポートがデフォルトで無効になっています。これは、IPv6 による DNS 検索を無効化する既知の方法が Mac OS X にないためで、DNS サーバとホームルータのどちらか、あるいは両方が同時にダウンした場合、接続速度が極端に低下する恐れがあります。詳しくは Bug 68796 と Bug 231607 をご覧ください。Mac OS X v10.3 では、(about:config から) network.dns.disableIPv6 の値を false に設定すれば、再び IPv6 を有効化できます。
