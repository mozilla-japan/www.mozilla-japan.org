プライバシーとセキュリティ設定 − 検証

このセクションでは、［検証］パネルを使い方を説明します。まだそのパネルを開いていない場合は、次のステップに従ってください：

  1. ［編集］を開き、［設定］を選択します。
  2. ［プライバシーとセキュリティ］カテゴリの下の［検証］をクリックします。（［プライバシーとセキュリティ］でサブカテゴリーが表示されていない場合は、［プライバシーとセキュリティ］をダブルクリックしてリストを開きます。）

証明書の検証についての情報は、 証明書検証はどのように機能するのかをご覧ください。

 

CRL

破棄証明書リスト（CRL）とは、認証局（CA）が作成し署名したもので破棄された証明書のリストです。 CRL をブラウザにダウンロードして、証明書を認証に使用する許可を出す前に、まだ証明書が有効かどうかを検証することができます。

［CRL の管理］をクリックして、証明書マネージャが利用できる CRL のリストをご覧ください。

CRLの管理 についての詳細情報は、CRL を管理するをご覧ください。

 

OCSP

オンライン証明書ステータスプロトコル（Online Certificate Status Protocol： OCSP）は、証明書が閲覧されたり利用されたりする度に、証明書マネージャが証明書の有効性をオンラインチェックできるようにしています。このプロセスには、その Web サイトによって管理されている破棄証明書リスト （CRL） に照らし合わせた証明書のチェックが含まれています。 OCSP が作動するためには、コンピュータがオンラインになっている必要があります。

証明書マネージャが OCSP をどう使うかを設定するには、 OCSP セクションの［検証の設定］で次の設定の一つを選択します：

  • 証明書の検証に OCSP を使用しない： 証明書マネージャが証明書の検証をする度に、オンラインのステータスチェックをしないようにしたい場合は、この設定を選択してください。その替わり、証明書マネージャが証明書の検証を行う場合は常に、証明書の有効期限と、証明書が認証局によって正しく署名されているかを検証することしかしません。その認証局は、その認証局自体が認証局証明書タブ （証明書マネージャのメインウィンドウ内） に加えられており、かつそのような証明書を発行する認証局が信頼できるとされていることが必要です。
  • OCSP を使って OCSP サービス URL が指定されている証明書だけを検証する： 証明書マネージャがチェックのために URL を指定する証明書を検証するたびに、オンラインステータスを実行するには、この設定を選択します。 証明書によって URL が指定されている場合、証明書マネージャは証明書が有効であると必ず表示し、有効期間と信頼性設定についてチェックを行ないます。
  • OSCP を使ってここに指定されている URL と署名者を使用しているすべての証明書を検証する： 証明書マネージャが証明書を検証するたびにオンラインステータスチェックを実行したい場合は、この設定を選択します。この設定をした場合、OCSP 応答の署名者を特定する［応答署名者］ポップアップメニューから証明書を選択しなければなりません。この設定で、証明書マネージャが認める証明書は、OCSP 応答によって検証され応答署名者証明書で（またはそれに関連した証明書を使って）署名されたものに限られます。


    • ポップアップメニューから "応答署名者証明書" を選択した場合、証明書マネージャはその署名者ためにサービス URL を （もし利用可能であれば） 自動的に入力します。サービス URL が自動的に入力されない場合は、ご自身で入力してください。詳細情報はあなたのシステム管理者にお尋ねください。

CRL マネージャ

このセクションでは、［CRL マネージャ］ダイアログボックスの使い方を解説します。ダイアログボックスを表示するには、次の手順に従ってください：

  1. ［編集］メニューを開き、［設定］を選択します。
  2. ［プライバシーとセキュリティ］カテゴリーの［検証］をクリックします。（サブカテゴリーが表示されていない場合は、［プライバシーとセキュリティ］をダブルクリックしてください。）
  3. ［CRL マネージャ］をクリックします。

このダイアログは、あなたのブラウザで利用するためにダウンロードされた CRL の一覧を表示します。多くの場合、URL をクリックすると CRL をダウンロードできます。CRL の機能については、CRL を管理するをご覧ください。

CRL を選択するには、それをクリックしてください。そして、以下の作業を行うことができます：

  • 削除： あなたのハードディスクから CRL を完全に削除します。検証用証明書として CRL が必要ないと確信ができない限り、この方法は行わないでください。確信が持てない場合は、システム管理者にお尋ねください。
  • 設定： CRL 自動更新の設定ダイアログボックスを開きます。ここで、選択された CRL の自動更新機能を有効にすることができ、更新をどのくらいの頻度で行うかを設定できます。
  • 更新： （可能な場合は）選択された CRL を直ちに更新します。

［CRL マネージャ］ダイアログボックスで、それぞれの CRL について以下の情報が得られます：

  • 組織（0）： この CRL を発行した組織の名前。
  • 部門（OU）： この CRL を発行した部門の名前（例えば、ある特定のタイプの証明書用のルート CA）。
  • 最終更新日： この CRL のブラウザ側のコピーが最後に更新された日付。
  • 次回更新日： この CRL の更新されたバージョンが、次に発行者から発行される日付。
  • 自動更新： この CRL に対して自動更新が有効になっているかを示します。自動更新をコントロールする設定画面を表示するには、CRL を選択して［設定］をクリックしてください。
  • 自動更新の状況：
    • 自動更新が有効になっていない場合、または有効になっているけれど次に予定されている更新がまだされていない場合、このフィールドは空白になります。
    • 一回でも自動更新が行われると、このフィールドに、最後の自動更新が失敗した場合には "失敗"、自動更新が成功した場合には "OK" と表示されます。

CRL インポートステータス

このセクションでは、CRL を初めてインポートしようとした時や、CRL を手動で更新した時に表示される CRL インポートステータスダイアログボックスの使い方を解説します。

このダイアログボックスは以下の情報を提供します：

  • CRL をインポートまたは更新の試みが成功したかどうか
  • どの組織が CRL を発行したのか
  • その CRL の次回の更新がいつか
  • その CRL に対して自動更新が有効になっているかどうか

自動更新が有効になっていない場合、以下のようにして有効にすることができます：

  • はい： CRL の自動更新を有効にするには、"はい" をクリックしてください。このボタンをクリックすると、CRL 自動更新の設定ダイアログボックスが表示されます。次のセクションでここでの設定の仕方を解説します。
  • いいえ： 自動更新を無効のままにしておきたい場合は、"いいえ" をクリックしてください。

CRL 自動更新の設定

このセクションでは、CRL 自動更新設定ダイアログボックスの使い方を解説します。このダイアログボックスが表示されていない場合は、次のステップに従ってください：

  1. ［編集］メニューを開き、［設定］を選択します。
  2. ［プライバシーとセキュリティ］カテゴリーの［検証］をクリックします。（サブカテゴリーが表示されていない場合は、［プライバシーとセキュリティ］をダブルクリックしてください。）
  3. ［CRL マネージャ］をクリックて、自動更新設定を表示、または変更したい CRL を選択してください。
  4. ［設定］をクリックします。

このダイアログボックスには、以下のようなオプションと情報が表示されます：

  • この CRL での自動更新を有効にする： 選択された CRL が、ここで設定したスケジュールに従って自動的に更新されるようにしたい場合、このオプションを選択してください。（CRL で次回更新日が指定されていない場合、このオプションは選択できないことに注意してください。）

    • 自動更新を有効にした場合、次のラジオボタンのいずれかを選択しなければなりません：

    • 次回更新日の X 日前に更新： 更新の頻度を、CRL 発行者が新しいバージョンの CRL を発行する頻度にあわせたい場合は、このオプションを選択してください。
    • X 日ごとに更新： CRL の次回更新日とは関係なく更新頻度を設定したい場合は、このオプションを選択してください。
  • CRL のインポート元： ブラウザがインポートした CRL の URL を示します。この設定は変更できません。違うロケーションを指定する場合には、CRL を削除して、新しいロケーションからインポートし直してください。
  • 更新失敗の連続回数： CRL の更新試行が何回連続して失敗したのかを示します。この情報には、最終更新の失敗も含まれます：
    • 最終更新が成功した場合、ここには、たとえ失敗した試行があっても "なし" と表示されます。
    • 最終更新が失敗した場合、ここには連続して失敗した試行の数と、最後の試行のエラーメッセージが表示されます。

［OK］をクリックして、あなたの設定を確定してください。
